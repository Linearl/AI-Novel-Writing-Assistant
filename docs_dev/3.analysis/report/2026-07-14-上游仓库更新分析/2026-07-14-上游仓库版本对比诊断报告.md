## 二、核心改进点分析

### 改进点 1：Book Analysis（拆书）系统 ⭐⭐⭐⭐⭐

**规模**: 全新模块，服务端 ~30 文件 + 客户端 ~28 文件 + 7 个新数据库模型 + 7 个新 Prompt（另有 1 个 RAG + 1 个 Image Prompt 属于其他改进点）

**功能**: AI 驱动的书籍分析工作台。用户导入文档后，AI 自动进行分段式文学分析（情节结构、人物系统、世界观、主题、风格、市场定位），支持：

- **两种模式**: 参考模式（分析外部书籍）+ 诊断模式（分析自己的稿件）
- **章节分割**: 规则分割 + LLM 分割双策略
- **角色提取流水线**: 候选识别 → 档案生成（4 档深度：brief/standard/deep/exhaustive）→ 12 维角色画像 → 外貌演化追踪（逐章节快照 + 术语提取）→ 肖像生成 → 提升到全局角色库
- **Token 预算管理**: 1K-10M token 预算控制 + 使用追踪
- **源文本范围筛选**: 按章节/按字符数/全文
- **发布到知识库**: 分析结果可发布为知识文档，支持 RAG 索引

**架构**: Facade + CQRS（Command/Query 分离）+ Watchdog 超时恢复 + 内存任务队列（并发控制 + 去重）

**架构重构**: v0.32 中 12 个平铺文件重构为 8 个子目录（application/caching/generation/infrastructure/publish/shared/writing/bookAnalysisCharacter），约 25 文件，提升模块化程度。

**数据库模型**:
- `BookAnalysisCharacter` — 核心角色模型，含 12 维画像 JSON
- `BookAnalysisCharacterArc` — 角色弧线（逐章节）
- `BookAnalysisCharacterScene` — 场景参与记录
- `BookAnalysisCharacterAppearance` — 外貌整合（覆盖率追踪）
- `BookAnalysisCharacterAppearanceSnapshot` — 逐章外貌快照
- `BookAnalysisCharacterAppearanceImage` — 外貌关联图片
- `BookAnalysisCharacterAppearanceTerm` — 外貌术语（置信度 + 稳定性）

---

### 改进点 2：RAG 质量提升 ⭐⭐⭐⭐⭐

**规模**: 5 个新服务文件 + 1 个新 Prompt + 1 个新数据库模型 + 2 个 Chunk 新字段

**五层 RAG 增强**:

| 层 | 服务 | 功能 |
|----|------|------|
| 结构化分面 | `chunkFacets.ts` | 7 维 facet 系统（genreTags、sellingPointTags、targetReaders、strengths、weaknesses、characterRole、chapterAnchor），支持过滤检索 |
| 上下文分块 | `RagContextualChunkService.ts` | Anthropic 风格的 contextual retrieval — 为每个 chunk 生成 LLM 上下文前缀（≤260 字符），SHA-256 内容寻址缓存 |
| 交叉编码重排 | `RagRerankerService.ts` | 外部 reranker API 集成，支持多种响应格式标准化 |
| 检索追踪 | `RagRetrievalTracer.ts` | 每次查询的完整遥志（时序、候选计数、reranker 状态、命中详情），采样率可控 |
| 追踪清理 | `RagRetrievalTraceRetention.ts` | 自动清理过期 trace（默认 6 小时周期，unref 定时器） |

**Chunk 新字段**: `facetKeys`（结构化分面索引）、`chapterAnchor`（章节锚定）

---

### 改进点 3：Prompt 模板系统 ⭐⭐⭐⭐

**规模**: 5 个新模板文件（~1841 行）+ 4 个新 workbench 文件 + 2 个新数据库模型

**核心能力**:

- **高级模板引擎**: 支持 `{{context.book_contract}}`、`{{input.novelTitle}}`、`{{slot.xxx}}` token 引用
- **版本化覆盖**: 每本小说可自定义 `novel.chapter.writer` prompt，带完整版本历史和回滚
- **编译诊断**: 检测未知 token、缺失上下文组、无效消息
- **安全约束**: 必需上下文组（10 个）不可移除，确保核心创作指导不被破坏
- **Prompt Workbench UI**: 两种编辑模式（Safe Slots + Advanced Template），带预览、诊断、版本历史

**Slot 系统增强**:
- 新增 `official_default` 模式 — 小说可显式回退到官方默认值
- 批量 `applyOfficialSlots()` 操作
- 孤立 slot 静默保留策略

---

### 改进点 4：Auto-Director 增强 ⭐⭐⭐⭐

**规模**: 客户端 ~25 个新文件 + 服务端多个新服务

**新增能力**:

- **5 步创建向导**: 想法 → 基础设置 → 世界/风格 → 模型/运行 → 方向候选（21.8KB 状态机）
- **桌面通知系统**: 浏览器 Notification API，15 秒轮询，headless watcher 全局挂载
- **待审自动提升**: 超过 14 天的低风险提案自动提交（仅限 `relation_state_update` 和 `information_disclosure`），需双重确认（复选框 + 输入确认文本）
- **散文质量检测器**: 9 种问题码（负面翻转、过多破折号/省略号、句号重复、长段落、逐字重复、截断标记、AI 自引、占位符泄漏、工程术语泄漏），中英文双语检测，上限 40 条发现
- **冲突等级曲线**: 用户锚定的冲突强度作为硬约束，不可被 AI 重写
- **待审上下文注入**: 生成章节时注入 pending_review 提案信息，AI 知晓哪些角色状态变更尚未确认
- **资源上下文重构**: `pendingReviewItems` 拆分为 `highRiskCommittedItems` + `pendingProposalItems`

---

### 改进点 5：中文本地化 ⭐⭐⭐

**规模**: 大量上下文块标签从英文切换为中文

**影响范围**:
- 章节任务块、义务契约块、本地状态标签
- 参与者/角色引导/关系阶段文本
- 32 个上下文组 ID 映射到中文显示标签
- `toListBlock()` 空兜底改为 "无"

**意义**: 中文标签能显著提升中文小说生成时 LLM 的理解准确度。

---

### 改进点 6：Comic 制作流水线 ⭐⭐⭐

**规模**: 服务端 13 个服务文件 + 2 个新数据库模型 + 客户端新组件

**新增能力**:
- **角色资产系统**: 服装/武器/道具/载具/能力，每项有独立 AI 生成图片
- **场景一致性**: Scene Bible（色板、关键元素、材质、氛围、布局）+ 建立镜头图片
- **Sprite Sheet 合成**: 三视图 + 服装 + 道具 → 512px 高度水平条带 → 作为角色参考输入图片模型
- **风格系统**: 6 种艺术风格（webtoon_color、bl_manga、shounen_bw、ink_traditional、chibi、realistic），中英双语关键词
- **性别锁定**: 显式性别约束 prompt 防止模型偏向美少年渲染

---

### 改进点 7：LLM 基础设施增强 ⭐⭐⭐

**新增能力**:
- **Token 用量追踪**: `LlmTokenUsageSnapshot` + `AsyncLocalStorage` 上下文传播 + 数据库持久化

## 四、可直接移植到我们项目的改进

### 4.1 高价值、低风险（建议立即移植）

1. **中文上下文标签本地化** — `chapterLayeredContextShared.ts` 中的 `renderBookContractText()`、`renderStoryMacroText()` 等函数，以及 32 个 context group labels 的中文化。纯标签变更，零功能风险。

2. **散文质量检测器** — `ProseQualityDetector.ts` 是独立模块，regex-based，9 种问题码，中英文双语。可直接复制使用。

3. **验收状态规范化** — `normalizeAcceptanceStatus()` 函数，将 LLM 输出别名映射到规范值。提升验收流程健壮性。

4. **角色资源上下文重构** — `highRiskCommittedItems` + `pendingProposalItems` 替代旧的 `pendingReviewItems`，更精确的资源风险评估。

5. **章节列表重试分类** — `classifyChapterListRetryIssue()` 将失败分为 5 类并给出针对性修复指令，提升章节列表生成成功率。

6. **Token 用量追踪基础设施** — `usageTracking.ts` 的 `extractLlmTokenUsage()` + `AsyncLocalStorage` 传播。

7. **请求限制器热重载** — `evictSharedLimiters()` 支持 Provider 配置热更新。

### 4.2 高价值、中等风险（需评估后移植）

8. **RAG 分面检索** — `chunkFacets.ts` 的 7 维 facet 系统。需确认 Qdrant 版本支持。

9. **RAG 上下文分块** — `RagContextualChunkService.ts` 的 Anthropic 风格 contextual retrieval。需额外 LLM 调用成本。

10. **RAG 重排序** — `RagRerankerService.ts` 需要外部 reranker API 端点。

11. **待审自动提升服务** — `PendingReviewAutoPromotionService.ts` + 策略配置。需确认业务规则适用性。

12. **冲突等级曲线** — 前端 TensionCurve 组件 + 后端 `conflict_level_source` 字段。需迁移数据库。

13. **Chapter 边界上下文块** — `buildChapterBoundaryContextBlock()` 提供排他事件、入口/结束状态、不可越线列表。

### 4.3 高价值、高风险（建议作为独立功能开发）

14. **Book Analysis 系统** — 完整的拆书功能。建议作为独立任务包开发，因其涉及全新模块、7 个数据库模型、12 个 Prompt。

15. **Prompt 模板系统** — 高级模板引擎 + 版本化覆盖。需变更 Prompt 基础设施。

16. **Auto-Director 创建向导** — 5 步向导替代现有对话框。需重新设计创建流程。

17. **Comic 制作流水线** — 全新领域，独立评估。

### 4.4 低风险基础设施改进

18. **依赖检查脚本** — `check-deps.cjs`，dev 启动前自动验证依赖完整性。
19. **pnpm allowBuilds** — 显式控制原生包构建权限。
20. **`shared/utils/`** — 新增共享工具目录，可作为我们项目的参考。

## 六、总结

上游最新版在 v0.32 基础上进行了**大规模功能扩展**，核心主题是：

1. **拆书分析** — 最大的单一新功能，从 0 到 ~60 个文件
2. **RAG 生产化** — 从基础向量检索升级为分面 + 上下文 + 重排 + 追踪的完整管线
3. **Prompt 治理** — 从固定 prompt 升级为可定制、版本化、可视化的模板系统
4. **Auto-Director 成熟化** — 向导、通知、自动提升、质量检测的全面增强
5. **中文优先** — 大量英文标签中文化，提升中文小说生成质量
6. **Comic 新领域** — 完整的 AI 漫画制作流水线

**对我们项目最具价值的改进**（按投入产出比排序）：
1. 中文本地化（零成本、高收益）
2. 散文质量检测器（独立模块、即插即用）
3. RAG 质量提升（中等成本、高收益）
4. Auto-Director 增强（中等成本、高收益）
5. Book Analysis 系统（高成本、高收益、建议独立任务包）
