## 4. 改进措施

### 4.1 新增测试文件

| 文件                                     | 测试用例数 | 覆盖内容                           |
| ---------------------------------------- | ---------- | ---------------------------------- |
| `internalNavigationEnhanced.test.js`     | 10         | null/undefined/空字符串/各种 URL scheme |
| `novelWorkflowTaskUi.test.js`            | 62         | 所有导出函数的完整覆盖             |

### 4.2 测试设计策略

**采用 AAA 模式** (Arrange-Act-Assert):
```javascript
test("getWorkflowBadge returns paused label for failed task", () => {
  // Arrange
  const task = { status: "failed", checkpointType: "chapter_batch_ready" };

  // Act
  const result = getWorkflowBadge(task);

  // Assert
  assert.equal(result.label, "第 1-10 章自动执行已暂停");
  assert.equal(result.variant, "destructive");
});
```

**覆盖维度**:
1. ✅ 正常路径 (Happy Path)
2. ✅ 错误处理 (null/undefined 输入)
3. ✅ 边界条件 (空字符串、特殊值)
4. ✅ 分支覆盖 (每个 if/else、switch case)
