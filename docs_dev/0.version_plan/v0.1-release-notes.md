## 1. 版本概述

v0.1.00 是本 fork 相对上游 v0.32.0 的首个正式定版。上游已提供完整的 AI 小说创作工作台基础架构（自动导演、Creative Hub、写法引擎、知识库、世界观等），本版本在此基础上新增 **8 个功能模块** + **多项工程化修复**。

### 与上游的关系

- 上游 v0.32.0 提供了完整的单书工作流（从开书到章节执行）
- 本 fork 专注于**审校质量提升**、**角色系统增强**、**运维可观测性**和**开发体验改善**
- 漫画/短剧模块已在本 fork 中移除（与小说创作方向无关）

## 2. 本 fork 新增功能（8 个）

### 2.1 审校上下文增强（REQ-2049）

补齐审校 prompt 缺失的全局字段，使审校 LLM 能看到更完整的上下文。

- `audit.prompts.ts`：light/full prompt 新增 book_contract、story_macro、timeline_context、character_dynamics、payoff_directives 5 组 context requirements
- `chapterLayeredContextBlocks.ts`：review 模式强制注入上述 context block，recent_chapters 扩展到 review 模式
- 影响范围：所有轻审校和完整审校调用

### 2.2 全局审校 + 跨章节回灌（REQ-2050）

从全书视角检测逐章审校无法发现的跨章节问题，并回灌到逐章审校形成修复闭环。

- 新增 `GlobalReviewIssue` Prisma model（severity/category/status 全生命周期）
- 新增 `audit.global.prompts.ts`（5 个审校维度：角色一致性/情节连贯/伏笔/节奏/世界观）
- 新增 `GlobalReviewService.ts`（scope 解析 + 320K token budget 自动裁剪 + 输出解析写入）
- 新增 `auditContextBuilder.ts`（全局层 8K + 章节层组装）
- 新增 `POST /api/novels/:id/global-review` 端点
- 逐章审校时自动注入 `global_review_feedback` context block（priority 105，最多 10 条）

### 2.3 角色重要度分级（REQ-2051）

新增 CharacterTier 四级分类，贯穿数据层到前端，影响章节上下文详略。

| 层级 | 变更 |
|------|------|
| Prisma | `CharacterTier` enum（lead/major/named/extra）+ `Character.tier` 字段 |
| Shared types | `CharacterTier` 类型 + Character/CastOptionMember/SupplementalCandidate 接口加 tier |
| Prompt | Zod schema 加 tier 校验、8 个 prompt 加分级指令、lead 有且仅有 1 个约束 |
| Service | lead 数量校验、cast apply 写入 tier、supplemental 透传 targetTier |
| 前端 | tier 选择器、侧边栏按 tier 分组/筛选、tier 徽章、阵容成员标签 |
| 下游 | 章节上下文按 tier 决定角色 profile 详略度 |

### 2.4 批量润色（REQ-3014）

对多个章节同时执行风格检测和修复，支持后台任务、进度追踪和取消。

- 新增 `novelBatchStyleRoutes.ts`（4 个端点：检测/修复/进度/取消）
- 后台任务机制：AbortController 取销、进度轮询、自动清理
- 前端：`useBatchPolish` hook + `BatchPolishButton`/`BatchPolishProgress`/`BatchPolishResult` 组件

### 2.5 反馈 → GitHub Issue 生成器（REQ-3019）

将用户反馈 + 自动收集的上下文通过 AI 生成结构化 GitHub Issue。

- 新增 `feedbackContextCollector.ts`（环形缓冲区收集 console/异常/网络/路由，<50KB）
- 重写 `FeedbackDialog.tsx`（FAB 浮动按钮 + 弹窗 + Ctrl+V 粘贴截图 + 预览标签页）
- 新增 `issueGenerator.ts` + `issueGeneration.prompts.ts`（PromptAsset 注册）
- 新增 `POST /api/feedback/generate` 端点
- 前端：Markdown 预览 + 复制到剪贴板 + 跳转 GitHub Issues 页面

### 2.6 小说预览增强（REQ-3011）

预览页面新增控制面板，支持键盘快捷键翻页。

### 2.7 任务中心批量操作（REQ-3012/3013）

任务中心支持全选、批量归档、批量清理导演跟进任务列表。

### 2.8 日志中心（REQ-3016）

新增服务端日志查询 API 和日志中心页面（`/logs`），支持运行时排查。
