## 1. 版本概述

v0.1.00 是项目从仓库初始化到首个正式定版的完整版本。涵盖基础设施搭建、核心功能开发、质量加固和 v0.1 任务包群的全部交付。

### 核心交付

1. **项目基础设施**：Monorepo 搭建、Prisma schema、开发脚本、.gitignore 治理
2. **任务包功能群（5 个）**：审校增强、全局审校、角色分级、批量润色、反馈 Issue 生成
3. **历史功能归档（8+ 个）**：预览增强、批量操作、日志系统、世界同步等
4. **工程化治理**：超大文件拆分、类型错误修复、测试通过率保障、git 追踪清理

### 产品级验收标准

> **核心验收目标**：用户能够利用系统独立完成一个完整的小说项目，成文 AI 味不重。

**验收指标**：

| 维度 | 验收标准 | 当前状态 |
|------|----------|----------|
| **独立性** | 用户无需外部辅助，仅用系统即可完成小说创作 | ✅ 功能已覆盖 |
| **AI 味控制** | 输出文本的 AI 味在可接受范围内 | ⏸️ 待验证 |
| **完整性** | 完成的小说达到出版基本要求 | ⏸️ 待验证 |
| **一致性** | 小说内部逻辑一致，无明显矛盾 | ✅ 已通过 |

**参考案例**：《玫瑰窃贼》— 基于柳爽同名歌曲衍生的都市文艺小说（18 万字，30 章）

## 2. v0.1.00 任务包群（本版本核心交付）

### 2.1 审校上下文增强（REQ-2049）

补齐审校缺失的全局字段——book_contract、story_macro、timeline_context、character_dynamics、payoff_directives。

- `audit.prompts.ts`：light/full prompt 新增 context requirements
- `chapterLayeredContextBlocks.ts`：review 模式强制注入 timeline/character_dynamics/payoff_directives，recent_chapters 扩展到 review 模式

### 2.2 全局审校 + 跨章节回灌（REQ-2050）

从全书视角检测跨章节问题（角色一致性、伏笔呼应、情节连贯性、节奏、设定自洽），输出结构化问题清单并回灌到逐章审校。

- 新增 `GlobalReviewIssue` Prisma model
- 新增 `audit.global.prompts.ts` 全局审校 prompt（PromptAsset 注册）
- 新增 `GlobalReviewService.ts`（scope 解析 + 320K token budget 裁剪 + 输出解析）
- 新增 `auditContextBuilder.ts`（全局层 + 章节层上下文组装）
- 新增 `POST /api/novels/:id/global-review` 端点
- 逐章审校时注入 `global_review_feedback` context block（priority 105）

### 2.3 角色重要度分级（REQ-2051）

新增 CharacterTier 四级分类（lead/major/named/extra），贯穿数据层→Prompt→Service→前端→下游。

- Prisma：`CharacterTier` enum + `Character.tier` 字段
- Shared types：`CharacterTier` 类型 + 3 个接口加 tier
- Prompt：Zod schema 加 tier 校验、8 个 prompt 加分级指令
- Service：lead 数量校验、cast apply 写入 tier、supplemental 透传
- 前端：tier 选择器、侧边栏分组/筛选、徽章、阵容标签、补充角色目标 tier
- 下游：章节上下文按 tier 详略（extra 最小 / named 基础 / lead-major 完整）

### 2.4 批量润色（REQ-3014）

批量风格检测和修复，允许对多个章节同时执行风格检测和润色。

- 新增 `novelBatchStyleRoutes.ts`（4 个端点：检测/修复/进度/取消）
- 后台任务机制（AbortController 取销、进度轮询）
- 前端：`useBatchPolish` hook + `BatchPolishButton`/`BatchPolishProgress`/`BatchPolishResult` 组件

### 2.5 反馈 → GitHub Issue 生成器（REQ-3019）

FAB 浮动按钮 + 前端上下文收集 + AI Issue 生成 + 预览复制跳转。

- 新增 `feedbackContextCollector.ts`（环形缓冲区收集 console/异常/网络/路由）
- 重写 `FeedbackDialog.tsx`（FAB + 弹窗 + Ctrl+V 粘贴截图 + 预览标签页）
- 新增 `issueGenerator.ts` + `issueGeneration.prompts.ts`（PromptAsset 注册）
- 新增 `POST /api/feedback/generate` 端点
- 前端：Markdown 预览 + 复制 + 跳转 GitHub Issues
