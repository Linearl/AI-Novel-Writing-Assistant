## 1. 版本概述

v0.1 是项目的第一个正式版本，专注于**修复原版项目中的架构问题**和**提升易用性**。

### 核心目标

1. **修复架构缺陷**: 解决 13 个架构摩擦点（services/modules/routes 三层并存、依赖注入、测试重构等）
2. **处理健康检查问题**: 解决 2026-06-28 健康检查报告中的所有 P0/P1/P2 问题
3. **易用性改进**: 新增 24 个用户导向的功能（文笔资料库、氛围写作参考卡、创作助手等）
4. **安全加固**: 修复 128 个 npm 漏洞中的关键问题，加固安全防线

### 产品级验收标准

> **核心验收目标**：用户能够利用系统独立完成一个完整的小说项目，成文 AI 味不重。

**验收指标**：

| 维度 | 验收标准 | 量化目标 | 当前状态 |
|------|----------|----------|----------|
| **独立性** | 用户无需外部辅助，仅用系统即可完成小说创作 | 100% 功能覆盖创作全流程 | ✅ 功能已覆盖 |
| **AI 味控制** | 输出文本的 AI 味在可接受范围内 | 人类盲测通过率 > 70% | ⏸️ 待验证 |
| **完整性** | 完成的小说达到出版基本要求 | 字数 10-20 万字，结构完整 | ⏸️ 待验证 |
| **一致性** | 小说内部逻辑一致，无明显矛盾 | 一致性检查通过率 100% | ✅ 已通过 |

**参考案例**：《玫瑰窃贼》— 基于柳爽同名歌曲衍生的都市文艺小说（18 万字，30 章）

**定稿标准**：《玫瑰窃贼》全流程跑通 + AI 味盲测通过 = v0.1 正式定稿

详见 [v0.1-acceptance-criteria.md](./v0.1-acceptance-criteria.md)

## 7. 升级指南

从原版项目升级到 v0.1：

```bash
# 1. 拉取最新代码
git pull origin master

# 2. 安装依赖
pnpm install

# 3. 运行数据库迁移
pnpm db:migrate

# 4. 构建所有包
pnpm build

# 5. 运行测试验证
pnpm test:all
```

**注意事项**:
- v0.1 包含 breaking changes（API 路由重构）
- 需要重新生成 Prisma 客户端：`pnpm prisma:generate`
- Desktop 包需要重新构建：`pnpm build:desktop:all`
