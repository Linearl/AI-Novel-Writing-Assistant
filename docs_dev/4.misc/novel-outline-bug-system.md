## 二、主线架构

### 主线冲突

技术理想主义与资本现实、行业巨头垄断之间的长期对抗。主角凭借系统和自身实力，必须面对商业世界的复杂规则、巨头的围剿，以及从"技术执行者"到"战略决策者"的身份转变与能力挑战。

### 主角路径

从一个技术能力顶尖但只会埋头苦干、不善向上管理的"技术隐士"，逐渐成长为能统筹技术、商业和团队的"技术领袖"。核心变化在于学会展示自己的价值，并用系统改变行业格局，完成从"做事"到"做局"的蜕变。

### 主钩子

开篇以"裁员裁到大动脉"的戏剧性场景开场：主角作为公司核心技术骨干，因不善向上管理、不会给领导画饼而被误裁，导致项目崩溃。系统觉醒后，他利用能力快速定位并修复前公司无法解决的致命漏洞，实现初次逆袭打脸。前30章密集展示系统在技术场景中的神操作，并在创业启动时引入强敌（如巨头公司的技术封锁），形成强烈追读钩子。

### 推进循环

```
系统升级解锁新能力
    → 应用新能力解决更复杂的商业/技术难题
    → 推动公司进入新阶段
    → 触发系统新任务或新升级条件
```

循环核心：**"能力应用范围"与"事业规模"的同步扩张**，同时穿插对前公司及行业旧势力的持续打脸。

### 结局方向

主角的技术理想得以实现，其创立的公司成为国产AI领域的重要一极，改变了行业生态。系统或与主角深度融合，或揭示了更宏大的起源，但主角已不再单纯依赖它，成为了真正的行业领袖。

## 三、支线设计

### 支线一：人才收集/成长线（核心支线）

> **定位**：随着创业推进，主角不断发现、吸引、培养各领域人才，组建一支"各有短板但互补"的核心团队。每位成员的加入都是一个独立小故事弧，成长线与主线剧情交织。

#### 设计原则

- **每引入一位核心成员 = 一个小故事弧**（相遇 → 考验 → 信任建立 → 关键发挥）
- 成员能力有明确上限，不是万能工具人，各有需要克服的弱点
- 团队扩大带来管理挑战，倒逼主角从"独狼"进化为"领袖"
- 成员间的化学反应本身就是故事素材（摩擦、互补、默契）

#### 人才收集节奏建议

| 阶段 | 约章节 | 团队规模 | 人才来源 | 叙事功能 |
|------|--------|----------|----------|----------|
| 觉醒期 | 1-10 | 独狼 | — | 展示个人能力，建立系统设定 |
| 萌芽期 | 11-30 | 2-3人 | 前同事/发小/偶遇 | 信任建立，创业起步 |
| 成长期 | 31-80 | 5-8人 | 技术社区/竞争对手叛逃/高校 | 产品迭代，团队磨合 |
| 壮大期 | 81-150 | 10-20人 | 行业挖角/海外归国/校招培养 | 公司扩张，管理挑战 |
| 巅峰期 | 150+ | 核心+外围 | 行业联盟/开源社区 | 生态构建，格局升级 |

#### 已有核心成员（五人初始团队）

| 角色 | 定位 | 核心能力 | 短板 | 成长方向 |
|------|------|----------|------|----------|
| **林逸** | 创始人/CTO | 系统金手指 + 顶级技术直觉 | 社交表达、商业判断 | 从技术执行者到战略决策者 |
| **赵阳** | 联合创始人/产品 | 用户感知、团队润滑 | 自我怀疑、技术深度有限 | 从"气氛担当"到真正的产品领袖 |
| **陈子轩** | 投资人/外部资源 | 人脉、资金、商业视野 | 家族牵绊、独立决策力 | 从"靠爹的富二代"到独立投资人 |
| **苏晴** | 投资人/商业引路人 | 资本运作、战略判断 | 理想与现实的拉扯 | 在LP压力与创始愿景间找到平衡 |
| **李明** | 表面导师/隐藏对手 | 学术资源、行业影响力 | 技术偏执、控制欲 | 从诱导者到被觉醒者识破 |

#### 可扩展人才方向（待细化）

| 方向 | 建议角色 | 加入时机 | 故事素材 |
|------|----------|----------|----------|
| **算法大牛** | 被巨头冷落的天才研究员 | 成长期（31-80章） | 与林逸的技术理念碰撞，代码审查中的高手过招 |
| **安全专家** | 白帽黑客/前安全公司离职 | 成长期 | 保护系统秘密 + 应对巨头技术渗透的双重压力 |
| **硬件/芯片方向** | 海归芯片工程师 | 壮大期 | 国产算力需求，软硬结合的技术挑战 |
| **商业运营** | 被裁员的资深BD | 成长期 | 帮林逸补上商业短板，职场打脸旧东家 |
| **实习生/应届生** | 天才少年/竞赛选手 | 萌芽期→长期 | 主角亲自培养，师徒关系，能力从零到独当一面 |
| **开源社区大佬** | 远程协作的技术网红 | 壮大期 | 线上到线下的信任建立，社区资源引入 |

### 支线二：旧东家恩怨线

- 旧东家项目持续崩溃 → 前同事求助 → 主角有限度帮忙（建立行业声望）
- 王浩窃取成果的秘密逐渐浮出水面
- 旧东家内部改革派与保守派的分裂，给主角可乘之机
- 最终走向：合作 or 彻底碾压，取决于王浩的选择

### 支线三：开源社区/行业联盟线

- 主角在技术社区匿名贡献 → 逐渐建立声望
- 从单打独斗到吸引开源联盟关注
- 联合对抗巨头垄断，推动行业标准制定
- 最终：逸码科技成为开源生态的核心节点

### 支线四：系统秘密探索线

- 系统偶尔给出超出"Bug分析"范畴的提示 → 暗示更大来历
- 李明早年绝密项目的线索逐渐浮现
- 主角在利用系统与理解系统之间摇摆
- 最终揭示：系统来源与主角的技术理想之间存在深层关联

> **不加感情线**：林逸与苏晴、赵阳等的关系定位为纯粹的事业伙伴/战友，不发展恋爱关系。主角的情感投入全部放在技术理想和团队成长上。

## 四、核心设定

### 系统能力与代价

| 规则 | 能力 | 代价 | 边界 |
|------|------|------|------|
| **精神消耗** | 分析Bug或窥见思维 | 精神力消耗，影响日常判断和健康 | 每日使用次数有限，复杂分析消耗更大 |
| **暴露风险** | 频繁使用系统能力 | 引起他人注意，招来外部调查或打压 | 公共场合或敏感区域使用需谨慎 |
| **升级条件** | 解决更复杂Bug或完成特定任务 | 时间和努力投入，需承担失败风险 | 升级不可逆，必须满足条件才能进阶 |
| **道德边界** | 窥见他人思维 | 心理负担和道德压力 | 不能无故窥视，否则系统警告或能力暂时禁用 |

### 系统升级路径建议

| 等级 | 解锁条件 | 新能力 | 对应事业阶段 |
|------|----------|--------|-------------|
| Lv.1 | 初始觉醒（被裁时） | 本地代码Bug快速定位 | 失业期自救 |
| Lv.2 | 修复前公司致命漏洞 | 可分析远程/文档化的Bug | 初步逆袭打脸 |
| Lv.3 | 首个创业产品上线 | 窥见他人思维碎片（模糊） | 创业萌芽 |
| Lv.4 | 击败一次巨头技术封锁 | 系统级架构分析 | 公司站稳脚跟 |
| Lv.5 | 推动行业标准制定 | AI模型训练过程可视化 | 行业影响力 |
| Lv.6+ | 待设计 | 系统与主角融合深化 | 终极对决 |

- 共同代价：过度使用系统导致精神崩溃或健康恶化
- 共同代价：暴露系统引来巨头围剿或行业封杀

## 七、主要角色详设

### 1. 林逸（主角）

| 维度 | 内容 |
|------|------|
| **身份** | 前星云科技高级工程师，现逸码科技创始人兼CTO |
| **外在形象** | 沉默寡言、眼神专注，习惯用右手食指无意识敲击桌面或激光笔按钮 |
| **性格内核** | 技术痴迷、逻辑至上，内心正直但社交笨拙，怀有强烈的技术报国情怀。"技术隐士"——代码是他的母语，人际关系是他的外语 |
| **背景** | 国内顶尖大学计算机系毕业→星云科技核心项目隐形支柱→因不善向上管理被王浩主导裁掉。核心贡献长期被忽视，技术成果被他人窃取 |
| **核心矛盾** | 坚信"技术够强就不需要在意人际关系"vs 现实反复证明这个信念的错误。必须从"只做事"进化到"会做局" |
| **金手指** | 被裁时觉醒"Bug分析系统"，快速定位代码漏洞 + 窥见他人部分思维碎片 |
| **道德底线** | 绝不利用系统谋取私利或伤害无辜，技术应服务于人而非控制人 |
| **成长弧线** | 技术隐士 → 学习商业规则 → 团队领袖 → 行业领袖。终极目标：打造世界一流的国产大模型 |

| 维度 | 内容 |
|------|------|

| **故事功能** | 代表"旧秩序"与"资本短视"的压力源，"碾压与反扑"关系 |

## 八、创作注意事项

### 写作红线

1. **系统是绝对秘密**——暴露系统是不可触碰的红线
2. **不加感情线**——主角的情感投入全部放在技术理想和团队成长上
3. **AI-First原则**——故事中的技术决策必须以AI结构化理解为主，不用关键词匹配等低级方式
4. **避免"技术万能"**——系统有明确代价和上限，不能解决所有问题

### 爽点设计节奏

- 每3-5章一个小打脸/技术碾压场景
- 每10-15章一个系统升级节点
- 每20-30章一个事业里程碑 + 对应的大冲突升级
- 人才收集穿插其中，每引入新人带来新的故事动力

### 技术细节原则

- Bug描述要具体可信，避免空泛的"系统检测到致命漏洞"
- 代码审查、技术评审等场景要有行业代入感
- AI创业的融资、产品、竞争等环节要有真实感
- 系统能力的展示要"克制中带爽感"，不搞一上来就无敌
