## 1. 背景与问题

`pnpm test` 批量运行时，4 个 auto-director 测试失败（测试通过率 79%），但单独运行每个测试文件全部通过。根因是测试间共享的 prisma mock 对象被前序测试污染，后续测试的 mock 断言与实际调用不匹配。

失败测试：
1. `auto director approval preferences expose defaults and persist concrete approval points`
2. `auto director auto-approval audit records the event, appends a milestone, and prunes older rows`
3. `auto director replan notice audit records a reminder instead of an auto-approved wording`
4. `auto director auto-approval audit loads the latest 10 records per novel`

不修复的影响：CI 始终报红，无法区分真实回归与已知 flaky，测试失去门控作用。

---

## 2. 目标与范围

### 2.1 目标

1. 批量模式下 19/19 测试全部通过
2. 每个测试文件的 prisma mock 在 teardown 时完整恢复

### 2.2 In Scope

**后端测试**：
- `server/tests/autoDirectorAutoApprovalAudit.test.js`
- `server/tests/autoDirectorApprovalPreferenceContract.test.js`
- `server/tests/autoDirectorApprovalPreferenceRoutes.test.js`
- `server/tests/autoDirectorChannelCallbacks.test.js`
- 其他共享 prisma mock 的 auto-director 测试文件

### 2.3 Out of Scope

- 测试框架迁移（保持 Node.js 内置 test runner）
- 非 auto-director 测试的隔离问题

---

## 3. 需求详情

### 3.1 根因分析

批量运行时 `run-tests.cjs fast` 将多个测试文件在同一进程中顺序执行。auto-director 测试文件直接修改 `prisma.model.method = async () => ...`，部分测试的 `finally` 块未完整恢复所有被 mock 的方法，导致后续测试读到脏 mock。

### 3.2 修复策略

- 在每个测试的 `finally` 或 `after` 中恢复所有被替换的 prisma 方法
- 考虑抽取共享的 mock helper（`createPrismaMock` + `restorePrismaMock`）
