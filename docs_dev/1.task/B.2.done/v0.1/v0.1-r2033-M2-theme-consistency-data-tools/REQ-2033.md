## 1. 背景与问题

主题相关数据散布在 8 个不同位置，全是自由文本或 JSON blob，没有工具将它们聚合。具体痛点：

- **伏笔失控**：`PayoffLedgerItem` 有 overdue/failed 状态，但没有汇总工具，用户无法快速知道"有多少伏笔逾期未兑现"
- **卷主题无覆盖检查**：`VolumePlan.mainPromise` 声明了卷的主题承诺，但没有工具检查是否有章节 `purpose` 实际支撑这个承诺
- **主题层级不可见**：书级承诺 → 卷级承诺 → 章节功能的三层关系没有聚合展示

---

## 2. 目标与范围

### 2.1 目标

1. 新增 3 个 `inspect` 类别 agent tool，让 narrative_advisor 和其他 agent 能查询主题一致性数据
2. 纯规则检查，不调用 LLM，确定性输出
3. 复用现有 Prisma 查询，不新增 service

### 2.2 In Scope

**后端**：
- `server/src/agents/tools/novelReadTools.ts`（或新建 `themeConsistencyTools.ts`）：3 个工具定义
- `server/src/agents/toolRegistry.ts`：注册
- `server/src/agents/approvalPolicy.ts`：权限

### 2.3 Out of Scope

- 不做 LLM 语义分析（REQ-2034 承接）
- 不修改数据库 schema
- 不新增前端可视化

## 3. 需求详情

### 3.1 audit_payoff_health

扫描 `PayoffLedgerItem`，统计伏笔健康状态。

**输入**：`{ novelId: string }`
**输出**：
```json
{
  "total": 25,
  "byStatus": {
    "setup": 5,
    "hinted": 8,
    "pending_payoff": 6,
    "paid_off": 4,
    "overdue": 1,
    "failed": 1
  },
  "risks": [
    {
      "ledgerKey": "主角身世之谜",
      "status": "overdue",
      "targetEndChapterOrder": 30,
      "lastTouchedChapterOrder": 15,
      "daysStale": 15,
      "riskLevel": "high"
    }
  ],
  "summary": "25 个伏笔中 1 个逾期、1 个失败，建议优先处理"
}
```

**风险等级规则**：
- `high`：status 为 overdue 或 failed
- `medium`：status 为 pending_payoff 且距 targetEndChapterOrder < 5 章
- `low`：其他

### 3.2 audit_volume_theme_coverage

检查每卷 `mainPromise` 是否有章节 `purpose` 实际支撑。

**输入**：`{ novelId: string }`
**输出**：
```json
{
  "volumes": [
    {
      "volumeId": "...",
      "volumeSortOrder": 1,
      "mainPromise": "主角从凡人到修士的蜕变",
      "chapterCount": 30,
      "chaptersWithPurpose": 28,
      "coverageRate": 0.93,
      "uncoveredChapters": [5, 12]
    }
  ],
  "overallCoverage": 0.89,
  "gaps": [
    { "volumeSortOrder": 2, "message": "第 2 卷无 mainPromise 设定" }
  ]
}
```

**覆盖率计算**：`chaptersWithPurpose / chapterCount`，其中 `chaptersWithPurpose` 为 `purpose` 字段非空的章节数。

### 3.3 get_theme_hierarchy

聚合三层主题结构。

**输入**：`{ novelId: string }`
**输出**：
```json
{
  "book": {
    "mainPromise": "一个普通人逆天改命的热血故事",
    "readingPromise": "跟随主角从底层崛起，体验热血与感动"
  },
  "volumes": [
    {
      "volumeSortOrder": 1,
      "title": "初入仙门",
      "mainPromise": "从凡人到修士的蜕变",
      "chapters": [
        { "order": 1, "title": "第一章", "purpose": "引出世界观和主角" },
        { "order": 2, "title": "第二章", "purpose": null }
      ]
    }
  ],
  "coverageStats": {
    "totalChapters": 100,
    "withPurpose": 85,
    "overallCoverage": 0.85
  }
}
```

## 4. 验收标准

- [ ] 3 个工具均可通过 `getAgentToolDefinition()` 获取
- [ ] `audit_payoff_health` 能正确统计伏笔状态和风险
- [ ] `audit_volume_theme_coverage` 能正确计算覆盖率
- [ ] `get_theme_hierarchy` 能正确聚合三层结构
- [ ] 所有工具 category 为 `"inspect"`，riskLevel 为 `"low"`
- [ ] 类型检查 + 测试通过

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| PayoffLedgerItem 可能为空 | 返回空统计，不报错 |
| VolumePlan.mainPromise 全部为 null | coverageRate 为 0，gaps 提示"所有卷均无主题承诺" |
