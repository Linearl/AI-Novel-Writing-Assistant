## 1. 背景与问题

### 1.1 现状

步骤6（章节执行区）使用三栏布局：

```
┌──────────┬──────────────────────┬────────────┐
│ 左侧队列  │     正文内容区        │ 右侧边栏    │
│  300px    │   minmax(0, 1fr)     │   332px    │
└──────────┴──────────────────────┴────────────┘
```

在 1366px 或 1440px 等低分辨率屏幕上：
- 左侧 300px + 右侧 332px = 632px 被固定列占用
- 1366px 屏幕下正文区仅剩 ~734px
- 1440px 屏幕下正文区仅剩 ~808px
- 用户阅读和编辑正文时空间不足

### 1.2 用户场景

1. 低分辨率笔记本用户需要更多正文空间
2. 用户在专注写作时不需要右侧边栏
3. 用户需要频繁切换正文阅读和 AI 执行台操作

### 1.3 当前布局代码

`client/src/pages/novels/components/ChapterManagementTab.tsx` (line 177):

```tsx
<div className="flex flex-col gap-4 xl:grid xl:h-[calc(100dvh-8rem)] xl:grid-cols-[300px_minmax(0,1fr)_332px] xl:items-stretch">
```

右侧边栏（lines 214-331）是内联 `<div>`，包含三个 Tab：动态栏、资料诊断、AI 执行台。

## 2. 目标与范围

### 2.1 目标

1. 右侧边栏支持折叠/展开
2. 折叠按钮放在 Tab 栏最左侧
3. 折叠后正文区获得更多空间
4. 折叠状态持久化（localStorage）

### 2.2 In Scope

**前端：**
- 右侧边栏折叠/展开按钮
- 折叠状态下的布局调整
- 折叠状态持久化

### 2.3 Out of Scope

- 右侧边栏宽度拖拽调整
- 左侧队列折叠
- 移动端适配（移动端已是单栏堆叠）

---

## 3. 需求详情

### 3.1 折叠按钮位置

WHEN 用户查看步骤6右侧边栏
THE SYSTEM SHALL 在 Tab 栏（动态栏/资料诊断/AI 执行台）的最左侧显示折叠按钮：

```
展开状态：[>] 动态栏 | 资料诊断 | AI 执行台
折叠状态：[<]
```

- 按钮使用 `>` 图标表示"折叠"（收起右侧）
- 按钮使用 `<` 图标表示"展开"（展开右侧）
- 按钮宽度固定，与 Tab 栏等高

### 3.2 折叠行为

WHEN 用户点击 `>` 按钮
THE SYSTEM SHALL：
- 右侧边栏内容区域隐藏
- Tab 栏收缩为仅显示 `<` 按钮
- 正文区扩展占满剩余空间
- 布局从 `grid-cols-[300px_minmax(0,1fr)_332px]` 变为 `grid-cols-[300px_minmax(0,1fr)_40px]`

### 3.3 展开行为

WHEN 用户点击 `<` 按钮
THE SYSTEM SHALL：
- 右侧边栏内容区域恢复显示
- Tab 栏恢复为完整的三个 Tab
- 正文区恢复原有宽度
- 布局恢复为 `grid-cols-[300px_minmax(0,1fr)_332px]`

### 3.4 状态持久化

WHEN 用户折叠/展开右侧边栏
THE SYSTEM SHALL 将状态保存到 localStorage：

```
Key: ai-novel.chapter-execution.right-rail.collapsed
Value: "true" | "false"
Default: "false"（展开）
```

页面加载时读取该值恢复状态。

### 3.5 动画

折叠/展开过程使用 CSS transition 平滑过渡：

```css
transition: grid-template-columns 200ms ease-in-out;
```

- [ ] 折叠后不影响左侧队列的显示

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 折叠后用户找不到展开按钮 | 按钮始终可见（在折叠的窄条中） |
| Grid 列变化可能导致布局抖动 | 使用 CSS transition 平滑过渡 |
| 移动端不需要此功能 | 仅在 xl 断点以上渲染折叠按钮 |
