# Decision Log - 世界同步手动对比功能

## 决策记录

### 2026-07-13 添加手动对比功能

**问题**：自动对比失效，无法检测到World和NovelWorld的差异

**决策**：添加手动对比API，复用共享的compareStructures函数

**原因**：
- 自动对比的syncDiffQuery可能因为缓存或条件问题没有触发
- 手动对比可以让用户主动触发对比，确保差异被检测到

**结果**：成功添加手动对比功能，用户可以手动触发对比

### 2026-07-13 合并差异显示面板

**问题**：自动对比和手动对比显示在两个独立面板，导致重复

**决策**：移除手动对比的独立面板，统一使用自动对比面板显示差异

**原因**：
- 避免用户困惑
- 减少UI复杂度

**结果**：成功合并差异显示面板

### 2026-07-13 忽略metadata.lastGeneratedAt字段

**问题**：同步后metadata.lastGeneratedAt字段会更新，导致手动对比仍然显示差异

**决策**：在对比逻辑中忽略metadata.lastGeneratedAt字段

**原因**：
- 同步操作后，NovelWorld的metadata.lastGeneratedAt会被更新为当前时间
- 这是正常行为，不需要显示为差异

**结果**：同步后手动对比和自动对比都显示一致
