# REQ-7019 决策日志

## D1: 选工厂函数手动注入而非 tsyringe

**日期**：2026-07-10
**决策**：使用工厂函数手动注入模式，不引入 tsyringe 或其他 DI 容器
**原因**：
1. 当前只需改造 3-5 个 service，手动注入复杂度可控
2. 零外部依赖，不改变构建流程
3. 依赖关系显式可见，调试友好
4. 未来需要时可平滑迁移到 tsyringe（接口兼容）
**影响**：需手动管理依赖图，但随着改造范围扩大可能需引入轻量工厂

## D2: 接口定义放在 `server/src/platform/di/`

**日期**：2026-07-10
**决策**：依赖接口和适配器放在 `server/src/platform/di/` 而非 `server/src/services/`
**原因**：`platform/` 是基础设施层，DI 接口属于平台能力，不应与业务 service 混放
**影响**：新增 `platform/di/` 目录

## D3: 构造函数参数提供默认值保持向后兼容

**日期**：2026-07-10
**决策**：改造后的 service 构造函数使用默认参数（真实依赖），测试时显式传入 mock
**原因**：避免改造时修改所有调用方，降低迁移风险
**影响**：生产代码无需改动构造函数调用，测试代码显式注入
