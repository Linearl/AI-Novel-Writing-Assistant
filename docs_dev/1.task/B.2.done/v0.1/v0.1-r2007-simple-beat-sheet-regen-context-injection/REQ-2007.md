## 1. 背景与问题

### 1.1 现状

步骤5（结构化大纲工作台）中点击"重新生成节奏板"时，AI 收到的上下文**不包含**：

| 缺失数据 | 影响 |
| ---------- | ---- |
| 之前的节奏板内容 | AI 无法知道此前的 beat 结构，可能生成完全不同的节奏划分 |
| 已有章节细化结果（taskSheet/sceneCards/purpose/boundary） | AI 无法感知已细化的章节，可能生成与已有细化冲突的 beat |

首次生成和重新生成走**完全相同的代码路径**，零差异。

### 1.2 根因

`volumeBeatSheetGeneration.ts` 中的 `generateBeatSheet()` 构建 `promptInput` 时，虽然 `workspace` 对象包含 `beatSheets` 和 `targetVolume.chapters[]`，但 `buildVolumeBeatSheetContextBlocks()` 函数没有将这些数据渲染为上下文块。

### 1.3 关键文件

| 文件 | 行号 | 作用 |
| ---- | ---- | ---- |
| `server/src/services/novel/volume/volumeBeatSheetGeneration.ts` | 72-80 | 构建 promptInput |
| `server/src/prompting/prompts/novel/volume/contextBlocks.ts` | 150-198 | 构建上下文块（缺失 existing_beat_sheet 和 chapter_details） |
| `server/src/prompting/prompts/novel/volume/beatSheet.prompts.ts` | — | Prompt 模板 |
| `server/src/prompting/prompts/novel/volume/shared.ts` | 42-50 | VolumeBeatSheetPromptInput 类型 |
| `server/src/services/novel/volume/volumeGenerationHelpers.ts` | 355-380 | mergeBeatSheet 替换逻辑 |

## 2. 目标与范围

### 2.1 目标

1. 重新生成节奏板时，默认注入此前的节奏板和章节细化结果供 AI 参考
2. 提供勾选项让用户选择是否参考（默认勾选）
3. 参考数据作为"约束参考"而非"必须保留"，AI 仍有创作自由

### 2.2 In Scope

**后端：**
- `contextBlocks.ts` 新增 `existing_beat_sheet` 和 `existing_chapter_details` 上下文块
- `beatSheet.prompts.ts` Prompt 模板中新增参考数据插槽
- `volumeBeatSheetGeneration.ts` 区分首次生成和重新生成

**前端：**
- 重新生成确认对话框中新增"参考此前生成结果"勾选项（默认勾选）
- 将勾选项传递到后端

### 2.3 Out of Scope

- 强制 AI 保留此前的 beat 结构（仅作为参考）
- 章节细化结果的自动合并（重新生成后章节细化仍保留，但可能与新 beat 不匹配）

## 3. 需求详情

### 3.1 勾选项

WHEN 用户点击"重新生成当前卷节奏板"
THE SYSTEM SHALL 在确认对话框中显示勾选项：

```
☑ 参考此前节奏板和章节细化结果（推荐）
  注入已有的 beat 结构和章节任务单供 AI 参考，保持节奏连续性
```

默认勾选。取消勾选则等同于当前行为（从零生成）。

### 3.2 上下文注入 — 节奏板

WHEN 重新生成且勾选"参考此前结果"
THE SYSTEM SHALL 在 AI prompt 中注入此前的节奏板内容：

```
## 参考：此前节奏板（仅供参考，可调整）
beat 1: {beatTitle} — 章节 1-3 — {beatSummary}
beat 2: {beatTitle} — 章节 4-6 — {beatSummary}
...
```

数据来源：`workspace.beatSheets` 中当前卷的 beat sheet。

### 3.3 上下文注入 — 章节细化

WHEN 重新生成且勾选"参考此前结果"
THE SYSTEM SHALL 在 AI prompt 中注入当前卷已细化章节的摘要：

```
## 参考：已有章节细化结果（仅供参考）
章节 1「{title}」: purpose={purpose}, 任务单摘要={taskSheet前100字}
章节 2「{title}」: purpose={purpose}, 任务单摘要={taskSheet前100字}
...
```

数据来源：`targetVolume.chapters[]` 中有 `taskSheet` 或 `purpose` 的章节。

### 3.4 Prompt 模板

`beatSheet.prompts.ts` 的 human message 模板中新增可选段落：

```
{existingBeatSheet ? `
## 参考信息
以下是此前的节奏板和已有章节细化结果，请在此基础上优化，保持节奏连续性。
如果需要大幅调整结构，也可以重新组织，但请尽量保留已有的好设计。

${existingBeatSheet}

${existingChapterDetails}
` : ''}
```

## 4. 验收标准

- [ ] 重新生成节奏板时，确认对话框显示"参考此前生成结果"勾选项
- [ ] 默认勾选
- [ ] 勾选后 AI prompt 中包含此前节奏板内容
- [ ] 勾选后 AI prompt 中包含已细化章节摘要
- [ ] 取消勾选时 AI prompt 中不包含参考数据（与当前行为一致）
- [ ] 首次生成时无勾选项（无历史可参考）
- [ ] 参考数据不影响 AI 的创作自由（仅为参考，非强制约束）

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 注入过多上下文导致 token 超限 | 节奏板摘要截断，章节细化只注入 purpose + taskSheet 前 100 字 |
| AI 过度依赖参考数据导致创新不足 | Prompt 中明确"仅供参考，可调整" |
