# REQ-2027: 章节编辑器编辑后资产沉淀

## 1. 问题背景

章节编辑器支持用户手动编辑 AI 生成的正文，但编辑行为只更新了当前章节内容，没有从用户的修改中沉淀任何可复用资产。用户的修改隐含了两层价值：

1. **反 AI 偏好**：用户改掉的往往是"AI 味道"重的表达，这些修改可以提炼为反 AI 规则，防止后续生成再犯同类问题。
2. **风格偏好**：用户的修改方向体现了对当前风格画像的不满或调整意图，可以 fork 一份新画像，保留用户的偏好修正。

当前系统中已有完整的风格提取（StyleExtractionTaskService）和反 AI 规则管理（AntiAiRuleService）能力，但它们都是独立的全局操作入口（写法引擎页面），没有与章节编辑器的编辑行为打通。

## 2. 目标

在章节编辑器主编辑区的保存按钮附近新增两个操作按钮：

1. **提取反 AI 规则**：对比修改前后的文本，识别用户意图，自动提取若干条反 AI 规则，用户确认后保存（默认应用到当前项目）。
2. **提取风格画像**：对比修改前后的文本，基于当前项目绑定的风格画像 fork 生成一套新画像，用户确认后切换项目到新画像。

**关键约束**：提取在保存前和保存后均可触发。保存时自动缓存编辑前的原始内容（preEditContent），确保保存后仍可提取 diff。

## 3. 范围

### 包含
- UI 层：章节编辑器主编辑区保存按钮附近新增 2 个按钮
- UI 层：提取结果的确认/编辑弹窗（反 AI 规则预览 + 风格画像预览）
- 前端层：保存时自动缓存编辑前原始内容（preEditContent），支持保存后提取
- 后端层：基于 diff 视图（含修改前后完整章节内容）的反 AI 规则提取 API（新增）
- 后端层：基于 diff 视图的风格画像 fork + 生成 API（新增）
- 数据层：反 AI 规则保存并绑定到当前小说项目的风格画像
- 数据层：新风格画像创建 + 风格绑定切换

### 不包含
- 修改现有风格提取流程（StyleExtractionTaskService）
- 批量跨章节的风格分析
- 自动触发（仅用户手动点击触发）

## 4. 非目标

- 不修改现有章节编辑器的保存逻辑
- 不修改现有写法引擎页面的功能
- 不做自动触发，必须用户主动点击
- 不支持批量操作（一次只处理当前章节的单次编辑 diff）

## 5. EARS 验收条目

| ID | 验收条件 | 优先级 |
|----|----------|--------|
| AC-1 | 章节编辑器主编辑区保存按钮附近显示"提取反 AI 规则"和"提取风格画像"两个按钮 | Must |
| AC-2 | 保存时自动缓存编辑前的原始内容（preEditContent），保存后按钮仍可用 | Must |
| AC-3 | 保存前也可提取（此时用 savedContent 作为 before，contentDraft 作为 after） | Must |
| AC-4 | 点击"提取反 AI 规则"后，将修改前后的完整章节内容作为 diff 视图提交给后端 | Must |
| AC-5 | 提取结果以弹窗展示，用户可逐条查看、编辑、启用/禁用，确认后保存 | Must |
| AC-6 | 保存的反 AI 规则自动关联到当前小说绑定的风格画像（style profile） | Must |
| AC-7 | 点击"提取风格画像"后，读取当前项目绑定的风格画像，fork 一份新的，结合 diff 视图生成规则调整 | Must |
| AC-8 | 新风格画像命名规则为"{原画像名}-{小说名}-v{版本号}"，支持用户手动修改 | Must |
| AC-9 | 新风格画像创建后，自动将当前小说的风格绑定切换到新画像 | Must |
| AC-10 | 提取过程有 loading 状态反馈，失败有 toast 错误提示 | Must |
| AC-11 | diff 过小时（如修改少于 10 个字），给出提示但仍允许用户操作 | Should |

## 6. 风险与未决项

| 类型 | 描述 | 状态 |
|------|------|------|
| 风险 | diff 太小导致提取质量低 | 低修改量时给用户提示 |
| 风险 | 风格画像 fork 频率过高导致画像膨胀 | 版本号递增 + 用户手动命名控制 |
| 风险 | 反 AI 规则与现有规则重复 | 提取时传入现有规则列表，让 LLM 去重 |
| 风险 | preEditContent 缓存占用内存 | 仅缓存一章内容，切换章节或保存新版本时清除 |
| 未决 | 版本号从哪里读取（全局递增还是按原画像递增） | 已决策：按原画像名维度递增 |
