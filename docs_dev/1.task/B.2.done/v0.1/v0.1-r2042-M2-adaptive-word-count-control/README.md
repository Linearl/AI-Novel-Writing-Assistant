## 1. 任务概述

### 1.1 需求来源

竞品分析讨论 + 用户反馈 — 章节字数膨胀（过度描写、水文）与核心章节字数不足的双重问题。

### 1.2 核心内容

1. 自适应字数控制：根据章节在卷中的角色（普通/过渡 vs 高潮/转折）自动分配不同字数范围
2. 字数系数机制：基准 3000~4000 字，普通章 ×(1~1.25)，高潮章 ×(1~2.5)
3. 生成后检测：实际字数 vs 目标范围，超限自动触发 compress，不足触发 expand
4. 水文检测：检测无效描写密度（与推进剧情无关的段落比例），而非单纯看字数
5. 章节角色由 auto-director 在 volume planning / beat sheet 阶段自动标注

### 1.3 前置条件

无
