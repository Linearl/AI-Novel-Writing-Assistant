# REQ-7026 决策留痕

## 决策记录

### D-01：研究型任务包，先行调研后决策是否迁移

| 项目 | 内容 |
| ---- | ---- |
| 决策日期 | 2026-07-10 |
| 决策者 | AI 建议 |
| 决策内容 | 本任务包的交付物是评估报告和迁移计划，不预设"一定要迁移到 tsx" |
| 决策理由 | 模块系统迁移涉及全 monorepo 构建链，风险高。没有充分的数据支持，盲目迁移可能导致生产环境性能退化或依赖不兼容。先调研再决策可降低风险 |
| 备选方案 | 直接实施迁移 — 风险不可控，可能引入性能退化或依赖兼容性问题 |

### D-02：实验在独立分支进行，不污染主分支

| 项目 | 内容 |
| ---- | ---- |
| 决策日期 | 2026-07-10 |
| 决策者 | AI 建议 |
| 决策内容 | tsx 原型和 Project References 实验在独立 git 分支创建，不在主分支修改任何构建配置 |
| 决策理由 | 实验过程可能多次修改 tsconfig/package.json，在主分支操作会影响其他开发者的构建环境。独立分支保证实验失败后可直接丢弃 |
| 备选方案 | 在主分支创建实验性配置 — 可能破坏其他开发者的构建环境 |

### D-03：基准测试用稳态 RSS 而非峰值 RSS

| 项目 | 内容 |
| ---- | ---- |
| 决策日期 | 2026-07-10 |
| 决策者 | AI 建议 |
| 决策内容 | 使用运行 30 分钟后 5 分钟采样平均值作为内存对比指标 |
| 决策理由 | Node.js 启动初期 V8 堆和 GC 行为不稳定，峰值 RSS 不反映长期运行的真实内存需求。稳态值才是生产环境的关键指标 |
| 备选方案 | 用启动后的峰值 RSS — 受 V8 预热和 GC 时机影响，数据不稳定 |

### D-05：prisma:generate prebuild hook 是独立于 tsx 迁移的低风险改进

| 项目 | 内容 |
| ---- | ---- |
| 决策日期 | 2026-07-10 |
| 决策者 | AI 建议 |
| 决策内容 | prisma:generate 加入 prebuild hook 作为独立任务（T5），无论 tsx 迁移是否 go 都应该执行 |
| 决策理由 | 这是一个低风险、高价值的改进，无需等待 tsx 迁移决策。当前已被多个开发者忘记执行 prisma generate 导致首次 build 失败 |
| 备选方案 | 等待 tsx 迁移时一起做 — 延迟了一个独立的低风险改进 |
