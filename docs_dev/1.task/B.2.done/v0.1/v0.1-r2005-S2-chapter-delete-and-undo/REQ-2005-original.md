# REQ-2005 步骤6新建章节删除与误操作恢复

## 1. 背景与问题

### 1.1 现状

步骤6（章节执行区）的"新建章节"按钮创建一个空壳章节（无 taskSheet、无 sceneCards、无卷规划映射）。创建后章节无法删除，误操作后无法恢复。

### 1.2 根因

- `Chapter` 表的删除只在 `VolumeChapterSyncService` 的 `applyDeletes: true` 路径中存在
- 前端没有章节删除 API 调用入口
- 没有软删除字段（`deletedAt`）或 undo 机制

### 1.3 影响

- 用户误点"新建章节"后无法清理
- 步骤6章节队列中积累无用空章节
- 用户体验受损，缺乏操作安全感

---

## 2. 目标与范围

### 2.1 目标

1. 步骤6中选中章节后提供"删除章节"按钮
2. 删除前需确认，有内容章节显示更强警告
3. 删除后提供 10 秒 undo 窗口

### 2.2 In Scope

**后端：** 软删除 API + 恢复 API
**前端：** 删除按钮 + 确认对话框 + undo toast

### 2.3 Out of Scope

- 物理删除、批量删除、删除审计日志 UI

---

## 3. 需求详情

### 3.1 删除按钮位置

步骤6选中章节后，右侧 AI 执行台区域显示"删除章节"按钮（位于主操作按钮下方或更多操作菜单中）。

### 3.2 确认对话框

- 空章节：简单确认
- 有内容章节：显示字数警告 + undo 提示

### 3.3 软删除

设置 `deletedAt` 字段，不物理删除。章节从默认列表中隐藏。

### 3.4 Undo Toast

删除后显示 10 秒 toast，点击"撤销"调用恢复 API。

---

## 4. 验收标准

- [ ] 选中章节后显示"删除章节"按钮
- [ ] 空章节和有内容章节分别显示不同确认对话框
- [ ] 删除后章节从队列消失，自动选中相邻章节
- [ ] 删除后显示 undo toast（10 秒）
- [ ] 点击撤销后章节恢复
- [ ] 恢复 API 正确清除 deletedAt
