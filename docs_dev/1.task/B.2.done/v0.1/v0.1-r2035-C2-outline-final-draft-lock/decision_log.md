# REQ-2035 决策日志

## D1: 锁定粒度为章节级

| 字段 | 内容 |
| ---- | ---- |
| 日期 | 2026-07-03 |
| 决策人 | AI |
| 决策 | 锁定粒度为章节级（非字段级），简化实现 |
| 原因 | 章节级锁定覆盖 80% 场景——用户确认一个章节的整体内容后，不希望 auto-director 修改其中任何部分。字段级锁定（锁定章节标题但允许修改正文等）需求不明确，且实现复杂度显著更高。 |
| 影响 | Chapter 模型增加一个 Boolean 字段 `locked`，auto-director 各阶段在操作前过滤 locked 章节。 |
| 备选方案 | 字段级锁定（锁定 title / summary / outline 等独立字段）— 粒度更细但实现复杂度高，且用户场景中极少需要"锁定标题但允许改正文"这种组合 |
