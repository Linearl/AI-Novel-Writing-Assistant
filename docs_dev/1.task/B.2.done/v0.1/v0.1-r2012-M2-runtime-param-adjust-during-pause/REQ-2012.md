## 1. 背景与问题

自动导演任务在创建时设定的多项高级参数（是否自动审校、是否自动修复、审批点配置、artifactSyncMode）一旦确认即锁定于 `seedPayload` 中。任务暂停后，用户只能通过 `policy_update` 调整推进模式、内容覆盖权限、深度审校开关、模型档次等运行时策略参数，无法修正创建时设置不当的高级参数。

**痛点**：
1. 用户在创建任务时对参数影响认知不足，设错后无法纠正
2. 模型档次（modelTier）后端已支持 `policy_update` 但前端未暴露
3. artifactSyncMode 不仅暂停后不可调，连创建时前端都没暴露
4. 暂停后如需"关掉自动修复改为手动审校"或"放宽审批点减少暂停次数"，无法操作

**不改会怎样**：用户设错 autoReview/autoRepair/审批点后只能取消任务重新创建，浪费已生成的章节和时间；或者被迫接受不符合预期的自动化行为。

---

## 2. 目标与范围

### 2.1 目标

1. 任务暂停后，在任务中心提供"编辑高级设置"入口，弹出面板复用创建时的 `DirectorAutoExecutionPlanFields` + `AutoDirectorApprovalStrategyPanel`
2. 用户修改后通过新增/扩展 API 提交，服务端允许在暂停态覆写 `autoExecutionPlan` 字段和审批点配置
3. 前端暴露 modelTier 选择器（纳入 `TaskCenterRuntimePolicyCard` 或"编辑高级设置"面板）
4. 前端暴露 artifactSyncMode 选择器（创建时 + 暂停后均可配），纳入 `DirectorAutoExecutionPlanFields`

### 2.2 In Scope

**后端**：
- 新增或扩展 API 端点，允许暂停后部分覆写 `autoExecutionPlan`（autoReview / autoRepair / artifactSyncMode）
- 新增或扩展 API 端点，允许暂停后更新审批点配置
- `policy_update` 已支持 `modelTier`，无需后端改动

**前端**：
- 创建时：`DirectorAutoExecutionPlanFields` 增加 artifactSyncMode 下拉选择器
- 暂停后：任务中心详情页增加"编辑高级设置"按钮，弹出模态面板
- 暂停后面板内容：autoReview 开关 / autoRepair 开关 / artifactSyncMode 选择器 / 审批点配置（复用 `AutoDirectorApprovalStrategyPanel`）
- `TaskCenterRuntimePolicyCard` 增加 modelTier 下拉选择器

### 2.3 Out of Scope

- 执行范围（startOrder / endOrder / volumeOrder / mode）的暂停后修改（涉及更复杂的执行状态重算，单独需求）
- 审批点配置的运行中"实时生效"（修改后下次 checkpoint 生效即可，不要求立即触发或撤销当前 checkpoint）
- 端到端测试（手动验证即可）

## 3. 需求详情

### 3.1 暂停后调整 autoReview / autoRepair / artifactSyncMode

**WHEN** 自动导演任务处于暂停状态（status 为 `checkpoint_reached` 或 `failed`）
**THE SYSTEM SHALL** 允许用户修改 `autoExecutionPlan` 中的 `autoReview`、`autoRepair`、`artifactSyncMode` 字段
**IF** 修改提交成功
**THE SYSTEM SHALL** 将修改后的值写入 task 的 `seedPayload.autoExecutionPlan`，下次 `continue` / `retry` 时以新值生效

### 3.2 暂停后调整审批点配置

**WHEN** 用户打开"编辑高级设置"面板
**THE SYSTEM SHALL** 加载当前任务关联的审批点授权码（`autoApproval.pointCodes`）
**WHEN** 用户修改审批点并保存
**THE SYSTEM SHALL** 将新的审批点配置写入任务，下次 `continue` 时按新配置判断 checkpoint 是否自动跳过

### 3.3 前端暴露 modelTier

**WHEN** 用户打开运行时策略卡片（`TaskCenterRuntimePolicyCard`）或"编辑高级设置"面板
**THE SYSTEM SHALL** 显示模型档次下拉选择器，选项为 `cheap_fast`（快速廉价）、`balanced`（均衡）、`high_quality`（高质量）
**WHEN** 用户选择新档次并保存
**THE SYSTEM SHALL** 调用已有的 `policy_update` 命令提交 `modelTier`

### 3.4 前端暴露 artifactSyncMode

**WHEN** 用户创建自动导演任务或编辑高级设置
**THE SYSTEM SHALL** 显示 artifactSyncMode 下拉选择器，选项为 `adaptive`（自适应，默认）、`deferred`（延迟/快速产文）、`strict`（严格同步）
**IF** 用户未显式选择
**THE SYSTEM SHALL** 默认使用 `adaptive`

## 4. 验收标准

- [ ] 任务中心详情页出现"编辑高级设置"按钮（仅自动导演任务 + 暂停/失败状态可见）
- [ ] 点击后弹出模态面板，包含 autoReview 开关、autoRepair 开关、artifactSyncMode 下拉、审批策略配置
- [ ] 修改保存后发送新 API 请求，返回成功后提示用户
- [ ] 创建任务时 `DirectorAutoExecutionPlanFields` 包含 artifactSyncMode 选择器
- [ ] `TaskCenterRuntimePolicyCard` 包含 modelTier 下拉选择器
- [ ] 暂停后修改 autoReview=false + continue，下一批章节不自动审校
- [ ] 暂停后修改审批点为全量自动批准 + continue，后续 checkpoint 自动跳过
- [ ] `pnpm typecheck` 通过
- [ ] `pnpm test` 通过

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| seedPayload 直接修改可能影响任务一致性 | 仅允许修改 autoReview / autoRepair / artifactSyncMode 三个字段，不涉及执行范围和已生成章节 |
| 审批点修改后与已通过的 checkpoint 状态不一致 | 审批点仅影响"后续"checkpoint，已通过的不回退 |
| 前后端类型同步 | 修改 shared/types/ 后先 build shared，再改 server/client |

---

## 6. 关联与边界

- 与 REQ-2006（步骤 4 每卷章节数手动调整）：同为暂停后参数调整，但 REQ-2006 侧重执行范围，本需求侧重审校策略和审批策略
- 不改动 `DirectorPolicyEngine` 核心逻辑，仅在暂停态允许 payload 覆写
