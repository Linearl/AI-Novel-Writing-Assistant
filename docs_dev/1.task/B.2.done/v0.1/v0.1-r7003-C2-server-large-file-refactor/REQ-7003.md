> 状态：⏳ 进行中

| ---- | ---- |
| 需求编号 | REQ-7003 |

| 关联需求 | REQ-7002（client 侧拆分） |

---

## 1. 背景与问题

server/src/ 共 18 个 TypeScript 文件超过 800 行（项目约束上限），最大达 1311 行。这些文件集中在以下功能域：

| 功能域 | 文件数 | 最大行数 |
|--------|--------|----------|
| world（世界观） | 3 | 1311 |
| prompting（提示词） | 4 | 1296 |
| novel/director（导演） | 5 | 1047 |
| rag | 1 | 1072 |
| styleEngine | 1 | 911 |
| planner | 1 | 978 |
| novel/core | 1 | 1046 |
| novel/runtime | 1 | 905 |
| novel/volume | 1 | 803 |

不改的影响：
- AI 辅助开发时上下文利用率低
- 同一文件多人修改冲突率高
- 代码审查难以聚焦

---

## 2. 目标与范围

### 2.1 目标

1. 所有 18 个文件拆分后 ≤ 600 行
2. 拆分按功能域分批进行，每批独立可验证
3. 功能零回归

### 2.2 In Scope

**后端**：18 个超大文件（详见附录）

### 2.3 Out of Scope

- client 侧文件（由 REQ-7002 负责）
- shared/ 包
- 业务逻辑变更（纯结构重构）

---

## 3. 需求详情

### 3.1 分批策略

按功能域分为 5 个批次，按优先级排序：

| 批次 | 功能域 | 文件数 | 优先级 | 理由 |
|------|--------|--------|--------|------|
| Batch 1 | world | 3 | P2 | 最大文件集中域 |
| Batch 2 | prompting | 4 | P2 | 提示词系统，职责边界清晰 |
| Batch 3 | novel/director | 5 | P2 | 导演核心，变更频率高 |
| Batch 4 | rag + planner + styleEngine | 3 | P3 | 孤立大文件 |
| Batch 5 | novel/core + runtime + volume | 3 | P3 | 小说核心管线 |

### 3.2 约束

- 纯结构重构，不改业务逻辑
- 每批拆分后 `pnpm typecheck` + `pnpm test` 必须通过
- 保持现有的导入路径（通过 facade / index.ts 重导出）

---

## 4. 验收标准

- [ ] 所有 18 个文件拆分后 ≤ 600 行
- [ ] `pnpm typecheck` 0 错误
- [ ] `pnpm test` 全部通过
- [ ] `pnpm build` 成功
- [ ] 无新增 `any` 类型

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 拆分破坏内部引用链 | 每步 typecheck，通过 facade 重导出保持公共 API 不变 |
| 循环依赖引入 | 拆分前分析依赖图，避免反向导入 |
