# REQ-7022 决策日志

## D1: 拆分行数阈值设为 680 行（而非 700 行绝对上限）

**日期**：2026-07-10
**决策**：先拆分 8 个 >680 行的文件，目标降至 <600 行
**原因**：项目约束 >700 行必须重构，但 680-700 区间的文件在下一次修改后就会触发强制重构线。提前处理 680+ 的文件减少未来返工。
**影响**：本次拆分范围从"仅 700+ 行文件"扩大到"680+ 行文件"，多了几个文件。

## D2: facade index.ts 维持导入兼容

**日期**：2026-07-10
**决策**：拆分后通过 `index.ts` 重导出，维持原导入路径可用
**原因**：避免拆分导致大量调用方需要同步修改导入路径。facade 提供一段缓冲期，后续各调用方可逐步迁移到新路径。
**影响**：拆分模块的 index.ts 会有 barrel export 负担，作为过渡方案可接受。

## D3: debug/ + commands/ 合并到 operations/ 而非新建目录

**日期**：2026-07-10
**决策**：不新建第四个目录，将少量文件收敛到已有 operations/ 下
**原因**：debug/ 2 文件、commands/ 3 文件，单独各自成目录过碎。operations/ 语义上能容纳二者（debug 操作 + command 操作）。新建目录会增加导航负担。
**影响**：operations/ 目录文件数从 ~N 增加到 ~N+5。

## D4: State 去重优先合并，次要重命名

**日期**：2026-07-10
**决策**：若 director/StateXxx.ts 与 director/state/Xxx.ts 定义同一 state → 合并到 director/state/；若职责不同 → 重命名以消除歧义
**原因**：同名歧义是最大的可读性问题，解决歧义优先于追求完全合并。
**影响**：可能保留少量独立 state 文件，但至少命名已清晰。
