# REQ-2025 决策日志

## D1：阈值粒度选择

- **决策点**：重试阈值按 issue signature 还是全局计数
- **选项**：A) 按 issue signature 独立计数  B) 全局计数
- **选择**：A — 按 issue signature
- **理由**：同一类问题重试 N 次无果说明该问题无法通过自动修复解决，应停下来；但不同类型的问题应独立计数，避免一个顽固问题阻断其他可修复问题的自动消除
- **日期**：2026-06-29

## D2：默认策略

- **决策点**：高风险默认策略是人工审核还是自动消除
- **选项**：A) 人工审核  B) 自动消除
- **选择**：A — 人工审核
- **理由**：向后兼容，用户主动选择自动消除才启用，避免意外行为变化
- **日期**：2026-06-29

## D3：Budget 倍率

- **决策点**：高风险自动消除时的 budget 放大倍数
- **选项**：A) 1.5 倍  B) 2 倍  C) 3 倍
- **选择**：B — 2 倍
- **理由**：1.5 倍提升不明显（patchRepair 从 2 变 3），3 倍 token 消耗过大；2 倍是合理平衡
- **日期**：2026-06-29

## D4：是否支持"忽略"策略

- **决策点**：高风险是否允许用户选择"忽略"
- **选项**：A) 支持（通过/忽略/自动消除三选一）  B) 不支持（人工审核/自动消除二选一）
- **选择**：B — 不支持忽略
- **理由**：高风险意味着章节质量存在严重问题（invalid 信号），忽略会导致质量问题累积到后续章节，违反质量门原则
- **日期**：2026-06-29

## D5：阈值耗尽行为

- **决策点**：重试阈值耗尽后的行为
- **选项**：A) 直接跳过（defer_and_continue）  B) 阻断等人工  C) 按全书自动驾驶模式决定
- **选择**：C — 复用现有断路器逻辑
- **理由**：全书自动驾驶模式下走 defer_and_continue（跳过继续），普通模式下走阻断（等人工），与现有行为一致，不需要新增分支
- **日期**：2026-06-29

## D6：卡片显示条件

- **决策点**：风险控制卡片在哪些模式下显示
- **选项**：A) 所有模式  B) 仅 full_book_autopilot  C) auto_to_execution + full_book_autopilot
- **选择**：B — 仅 full_book_autopilot
- **理由**：只有全书自动驾驶模式会走 qualityLoop budget 链，其他模式的高风险处理由用户手动触发，不需要此配置
- **日期**：2026-06-29
