# REQ-2011 决策日志

## D1: 使用 next-themes 而非手动实现

| 字段 | 内容 |
| ---- | ---- |
| 日期 | 2026-06-27 |
| 决策人 | AI |
| 决策 | 安装 `next-themes` 库实现主题管理 |
| 原因 | next-themes 是业界标准方案，内置系统主题检测、localStorage 持久化、SSR 安全、闪烁防护。手动实现需要处理同样的边界情况，且容易出错。 |
| 影响 | 新增一个轻量依赖（~3KB gzipped） |

## D2: 默认跟随系统主题

| 字段 | 内容 |
| ---- | ---- |
| 日期 | 2026-06-27 |
| 决策人 | AI |
| 决策 | `defaultTheme="system"`，首次访问时跟随用户系统偏好 |
| 原因 | 尊重用户已有的系统设置，减少首次配置步骤。用户可手动切换到固定亮色或暗色。 |
| 影响 | 系统设置为暗色的用户首次访问即为暗色主题 |

## D3: 导航栏快捷切换而非仅设置页

| 字段 | 内容 |
| ---- | ---- |
| 日期 | 2026-06-27 |
| 决策人 | AI |
| 决策 | 在导航栏增加三态切换按钮（☀️/🌙/💻），同时在设置页保留配置入口 |
| 原因 | 主题切换是高频操作，放在导航栏可一键触达。设置页作为备选入口，适合不熟悉快捷按钮的用户。 |
| 影响 | Navbar 右侧区域新增一个按钮 |
