# REQ-3013 决策日志

## D1：复用已有 batch-archive API

- 决策：不新建 API，复用 `POST /api/tasks/batch-archive`
- 理由：该端点已存在且支持批量归档，前端只需调用即可
- 影响：无后端改动

## D2：全选仅选当前页

- 决策：全选 checkbox 只选当前页的 items，不跨页全选
- 理由：跨页全选需要额外 API 支持，当前分页数据已在前端
- 影响：用户需要翻页后再次全选

## D3：批量归档按钮独立于批量执行动作

- 决策：归档按钮与"执行批量动作"按钮并列，不混入同一逻辑
- 理由：归档是清理操作，执行动作是推进操作，语义不同
- 影响：批量栏布局调整
