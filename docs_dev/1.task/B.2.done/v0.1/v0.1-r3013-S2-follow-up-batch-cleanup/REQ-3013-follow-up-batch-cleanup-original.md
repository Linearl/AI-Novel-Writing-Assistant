# REQ-3013：导演跟进任务列表全选与批量清理

## 背景

导演跟进任务列表（`/auto-director/follow-ups`）展示自动导演产生的待处理、异常、已替代等任务。当前支持：
- 单个 checkbox 勾选
- 批量执行动作（仅 `continue_auto_execution` 和 `retry_with_task_model`）

缺少的能力：
- 全选当前页/全部任务
- 批量清理已完成/已取消的过期任务

## 需求描述

### 功能 1：全选

- 列表表头增加"全选" checkbox
- 点击全选 → 勾选当前页所有可选任务
- 再次点击 → 取消全选
- 全选状态下，新增/移除筛选条件时自动重新全选
- 显示"已选 N / 共 M 项"

### 功能 2：批量清理

- 批量操作栏增加"批量归档"按钮
- 仅当所选任务全部为终态（`succeeded` / `failed` / `cancelled`）时可用
- 点击后调用 `POST /api/tasks/batch-archive` 批量归档
- 归档成功后刷新列表，toast 提示"已归档 N 项任务"
- 归档过程中显示 loading 状态

### 非功能要求

- 全选操作不应有额外 API 请求（基于当前页数据）
- 批量归档单次上限 50 条
- 归档为幂等操作（已归档的跳过）

## 验收标准

1. [ ] 表头有全选 checkbox，点击可全选/取消全选
2. [ ] 全选后底部批量栏显示"已选 N 项"
3. [ ] 批量归档按钮仅在所选任务全为终态时可点击
4. [ ] 点击批量归档后任务从列表消失，toast 提示成功
5. [ ] 非终态任务被选中时，批量归档按钮置灰并提示原因
