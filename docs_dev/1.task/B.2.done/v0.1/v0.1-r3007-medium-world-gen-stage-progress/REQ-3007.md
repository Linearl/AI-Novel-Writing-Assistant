## 1. 背景与问题

世界生成向导的步骤 2（骨架生成）当前使用 `POST /worlds/skeleton/generate` 的普通 HTTP 请求，前端通过 `useMutation.isPending` 仅知道"是否在加载"，按钮显示通用的 disabled 态。

对比步骤 1（灵感分析）已有 SSE 流式进度（`analyzeStream.latestRun?.message`），步骤 2 的体验明显落后。用户点击"生成骨架"后面对一个无反馈的等待，不知道系统在做什么、还要等多久。

---

## 2. 目标与范围

### 2.1 目标

1. 骨架生成过程中，按钮文案实时显示当前阶段名称和序号（如"正在生成核心规则... 2/7"）
2. 阶段名称对用户友好，反映实际生成内容
3. 生成完成后自动恢复按钮原始状态

### 2.2 In Scope

**后端**：
- 将 `generateWorldSkeleton` 从单次 LLM 调用改为多阶段执行
- 新增 `POST /worlds/skeleton/generate/stream` SSE 端点（或改造现有端点支持流式）
- 定义阶段枚举和进度事件格式

**前端**：
- `WorldGeneratorStepTwo` 的生成按钮消费 SSE 进度事件
- 按钮文案格式：`正在{阶段名}... {当前}/{总计}`
- 复用现有 `useSSE` hook 或类似模式

### 2.3 Out of Scope

- 旧版 `worldBuildingGraph` 的改造
- 步骤 1（灵感分析）的进度优化（已实现）
- 步骤 4（保存）的进度优化（保存很快，无需阶段进度）
- 进度条/百分比可视化（仅做文案进度）

---

## 3. 需求详情

### 3.1 阶段定义

骨架生成拆分为以下阶段（最终数量以 design.md 为准，这里给出预期方案）：

| 序号 | 阶段 ID | 用户可见标签 | 生成内容 |
| ---- | ------- | ------------ | -------- |
| 1 | concept | 正在构思世界概念 | 世界名称、一句话描述、类型定位 |
| 2 | rules | 正在生成核心规则 | 魔法体系、科技水平、物理法则 |
| 3 | factions | 正在构建势力阵营 | 主要势力、政治格局、种族分布 |
| 4 | geography | 正在绘制地理版图 | 地理环境、重要地点、区域划分 |
| 5 | conflicts | 正在编织冲突脉络 | 核心冲突、势力矛盾、叙事张力 |
| 6 | entries | 正在规划故事入口 | 故事入口建议、叙事切入点 |
| 7 | assessment | 正在校验完整度 | 一致性检查、完整度评分 |

### 3.2 SSE 进度事件格式

```
event: progress
data: {"stage":"rules","stageIndex":2,"totalStages":7,"label":"正在生成核心规则..."}

event: done
data: {"concept":{...},"structuredData":{...},"assessment":{...}}
```

### 3.3 前端按钮行为

- 未生成时：显示"生成骨架"
- 生成中：显示 `正在{label}... {stageIndex}/{totalStages}`，按钮 disabled
- 生成完成：恢复"生成骨架"，自动进入下一步
- 生成失败：显示错误信息，按钮恢复可点击

---

## 4. 验收标准

- [ ] 点击"生成骨架"后，按钮文案每 5-15 秒更新一次（反映阶段切换）
- [ ] 按钮文案格式为"正在XX... N/7"
- [ ] 所有 7 个阶段均有对应文案显示
- [ ] 生成完成后按钮恢复原始状态
- [ ] 生成失败时显示错误并恢复按钮
- [ ] 类型检查通过（`pnpm typecheck`）
- [ ] 无回归：生成结果质量与改造前一致

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 多阶段 LLM 调用总耗时可能增加 | 各阶段 prompt 精简，总 token 量不显著增加；并行无依赖的阶段 |
| 某阶段失败导致整体中断 | 支持部分结果返回 + 失败阶段重试 |
| 现有 `runStructuredPrompt` 不支持流式 | 封装新的多阶段执行器，不修改通用基础设施 |

---

## 6. 关联与边界

- 与步骤 1 的 SSE 流式进度（`useSSE`）：前端复用相同模式，保持一致性
- 与旧版 `worldBuildingGraph`：不复用，但可参考其节点定义作为阶段划分参考
