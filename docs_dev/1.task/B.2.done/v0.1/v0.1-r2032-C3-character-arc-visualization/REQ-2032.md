## 1. 背景与问题

REQ-2031 提供了 4 个后端查询工具，但用户仍无法直观地看到：
- 某角色的弧光发展轨迹（从起点到终点的事件+情绪变化）
- 角色之间的关系网络（谁信任谁、谁与谁冲突）
- 全书角色动态全景图

现有 UI 只有文字列表，缺乏可视化图表。

---

## 2. 目标与范围

### 2.1 目标

1. 角色弧光时间线图：展示某角色从第 1 章到最后 1 章的事件节点和情绪变化曲线
2. 角色关系网络图：展示核心角色之间的关系强度网络
3. 支持选择角色查看其弧光详情

### 2.2 In Scope

**后端**：
- 新增 `GET /api/novels/:novelId/character-arcs` — 返回所有角色的弧光摘要
- 新增 `GET /api/novels/:novelId/character-relations` — 返回角色关系网络数据

**前端**：
- `CharacterArcTimeline` 组件：水平时间线图，X 轴章节序号，节点=事件，底线=情绪曲线
- `CharacterRelationNetwork` 组件：力导向图或简单网络图，节点=角色，边=四维关系分数
- 角色选择器：下拉选择角色，切换查看其弧光
- 集成到角色管理页面或新增"角色弧光"标签页

### 2.3 Out of Scope

- 不做角色弧光的 AI 分析建议（后续迭代）
- 不做弧光编辑（只读展示）
- 不做动画或 3D 效果

---

## 3. 需求详情

### 3.1 角色弧光数据 API

`GET /api/novels/:novelId/character-arcs` 返回：

```json
{
  "characters": [
    {
      "characterId": "...",
      "name": "林墨",
      "role": "protagonist",
      "arc": { "arcStart": "...", "arcMidpoint": "...", "arcClimax": "...", "arcEnd": "..." },
      "timeline": [
        { "chapterOrder": 1, "event": "偶遇仙人", "emotion": "好奇", "stressLevel": 20 }
      ]
    }
  ]
}
```

### 3.2 角色关系网络 API

`GET /api/novels/:novelId/character-relations` 返回：

```json
{
  "nodes": [
    { "characterId": "...", "name": "林墨", "role": "protagonist", "isCore": true }
  ],
  "edges": [
    {
      "from": "id_A", "to": "id_B",
      "trust": 60, "intimacy": 30, "conflict": 20, "dependency": 40,
      "label": "同门师兄妹"
    }
  ]
}
```

### 3.3 时间线图

- X 轴：章节序号（1, 2, 3...），按卷分组
- Y 轴（底部折线）：stressLevel（0-100），红色渐变
- 事件节点：圆形标记，hover 显示事件标题 + 情绪
- 弧光阶段标注：用背景色块区分 arcStart/Midpoint/Climax/End 区间

### 3.4 关系网络图

- 节点大小：按角色重要性（core=true 更大）
- 节点颜色：按 castRole（protagonist=蓝，antagonist=红，ally=绿，等）
- 边粗细：按 trust + intimacy 综合分数
- 边颜色：trust 高=绿色，conflict 高=红色，中间=灰色
- hover 边：显示四维分数和关系标签

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| d3-force 引入增加包体积 | 用 visx 的力导向图，或纯 SVG 简单布局 |
| 角色过多时网络图拥挤 | 只显示 core=true 的角色，提供筛选 |
| 时间线数据量大 | 分卷加载或虚拟滚动 |
