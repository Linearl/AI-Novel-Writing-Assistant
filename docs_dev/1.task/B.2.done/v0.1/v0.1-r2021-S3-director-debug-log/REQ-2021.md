# REQ-2021 — 质量修复阶段调试日志保存

## 1. 目标

在 auto-director 断路器触发时，将完整的调试上下文保存为磁盘 JSON 文件，使开发者能够事后重现问题场景、定位根因，降低调试成本。

## 2. 范围

### 2.1 包含（Scope In）

1. **调试日志保存服务**: 实现一个纯函数/工具模块，接收调试上下文并序列化为 JSON 写入磁盘
2. **触发点集成**: 在 `stopAutoExecutionForCircuitBreaker` 中调用日志保存
3. **配置开关**: 新增 `DIRECTOR_DEBUG_LOG_ENABLED` 环境变量控制功能启停
4. **自动清理**: 保留最近 100 个日志文件，超出时删除最旧文件
5. **日志内容**: 时间戳、taskId、novelId、chapterId、autoExecution 快照、circuitBreaker 快照、最近 LLM 使用记录、错误堆栈（如有）

### 2.2 不包含（Scope Out）

1. 不实现实时日志流式推送（WebSocket/SSE）
2. 不实现日志的 Web UI 查看界面
3. 不修改现有的数据库事件日志（`DirectorAutomationLedgerEventService`）行为
4. 不修改断路器状态机逻辑或阈值

## 3. 非目标（Non-Goals）

- 这不是通用的日志框架（logging framework），仅服务于 director debug 场景
- 这不是审计合规日志，不需要防篡改或签名
- 不替代已有的 ledger event 数据库持久化

## 4. EARS 验收条目

### E1. 调试日志保存（Event-driven）

E1.1. **当** `stopAutoExecutionForCircuitBreaker` 被调用时，系统应将完整的调试上下文序列化为 JSON 文件并写入 `server/logs/director-debug/` 目录。
E1.2. **当** 保存调试日志时，文件名应包含时间戳和 taskId，格式为 `{ISO-timestamp}_{taskId}.json`，确保唯一且可排序。
E1.3. **当** 日志内容被序列化时，应包含以下字段：`timestamp`、`taskId`、`novelId`、`chapterId`、`autoExecution`（完整状态快照）、`circuitBreaker`（完整状态快照）、`recentLlmUsage`（最近 10 条 LLM 使用记录，如可获取）、`errorStack`（如有）、`config`（当前断路器阈值配置）。

### E2. 配置开关（Ubiquitous）

E2.1. **当** 环境变量 `DIRECTOR_DEBUG_LOG_ENABLED` 未设置或设为 `true` 时，调试日志功能应启用。
E2.2. **当** 环境变量 `DIRECTOR_DEBUG_LOG_ENABLED` 设为 `false` 时，`stopAutoExecutionForCircuitBreaker` 中应跳过日志保存，不产生磁盘写入。

### E3. 自动清理（Periodic）

E3.1. **当** 每次保存新调试日志后，系统应检查 `server/logs/director-debug/` 目录中的文件数量。
E3.2. **当** 文件数量超过 100 个时，系统应按文件名时间戳排序并删除最旧的文件，保留最近 100 个。

### E4. 错误处理（Ubiquitous）

E4.1. **当** 日志写入失败时（磁盘满、权限不足等），系统应静默忽略错误，不中断断路器停止流程。
E4.2. **当** `server/logs/director-debug/` 目录不存在时，系统应在首次写入时自动创建。

## 5. 验收标准

- [ ] 调用 `stopAutoExecutionForCircuitBreaker` 后，`server/logs/director-debug/` 目录下生成包含完整上下文的 JSON 文件
- [ ] JSON 文件包含 timestamp、taskId、novelId、chapterId、autoExecution、circuitBreaker、recentLlmUsage、errorStack、config 字段
- [ ] 设置 `DIRECTOR_DEBUG_LOG_ENABLED=false` 后，断路器触发不产生日志文件
- [ ] 目录超过 100 个文件时，新写入自动清理最旧文件
- [ ] 日志写入失败不中断断路器停止流程
- [ ] 单元测试覆盖日志保存函数的正常路径、开关关闭路径、清理路径

## 6. 风险与约束

| 风险 | 缓解措施 |
|------|----------|
| 高频断路器触发导致磁盘空间占用 | 100 文件上限 + JSON 格式体积可控（单文件 <100KB） |
| 日志写入 I/O 影响断路器停止延迟 | 异步写入（fire-and-forget），不 await 写入完成 |
| 日志文件包含敏感小说内容 | 明确为开发调试用途，仅本地存储，不上传；后续可加 .gitignore |

## 7. 关联与边界

- 与 `DirectorAutomationLedgerEventService` 的边界：ledger 为数据库持久化事件，本次为磁盘 JSON 快照，两者互补不冲突
- 依赖的外部条件：`stopAutoExecutionForCircuitBreaker` 函数签名保持稳定（当前已有 4 个参数输入）
