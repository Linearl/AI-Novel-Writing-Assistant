# REQ-2030 节奏曲线可视化与调整 (Pace Curve Visualization)

## 1. 背景与问题

现有 Prisma schema 中，`VolumeChapterPlan` 和 `Chapter` 都存储了 `conflictLevel`（0-100）和 `revealLevel`（0-100），`Chapter` 还有 `pacingScore`。但系统中没有任何地方以图表形式展示这些数据：

- 用户无法直观看到全书的节奏起伏曲线
- 无法判断冲突是否在正确的位置达到高潮
- 无法对比已写章节的实际节奏与规划节奏
- 未开始撰写的章节，节奏参数只能通过 JSON 编辑，无可视化调整入口

`StructuredBeatSheetCard.tsx` 只有文字版的 beat 分组展示，没有折线图或任何数值可视化。

---

## 2. 目标与范围

### 2.1 目标

1. 提供一个可交互的节奏曲线图表，展示全书各章节的 conflictLevel 和 revealLevel 走势
2. 按卷分组着色，清晰展示卷间节奏变化
3. 支持对未开始撰写的章节（`Chapter` 记录不存在或 `status` 为未开始）直接调整 conflictLevel / revealLevel
4. 已完成章节的节奏数据只读展示，不可编辑

### 2.2 In Scope

**后端**：
- 新增 `GET /api/novels/:novelId/pace-curve` 端点，返回各卷各章节的节奏数据聚合
- 数据来源：`VolumeChapterPlan`（规划值）+ `Chapter`（实际值，如已有）

**前端**：
- 新增 `PaceCurveChart` 组件（基于 recharts 或类似轻量图表库）
- 横轴：章节序号（按卷分组，卷间有分隔线）
- 纵轴：0-100 分数
- 双线：conflictLevel（红色系）+ revealLevel（蓝色系）
- 已写章节用实心点，未写章节用空心点
- 未写章节点击空心点可弹出滑块调整 conflictLevel / revealLevel
- 调整后实时保存到 `VolumeChapterPlan`

**基础设施**：
- 可能需要安装 recharts（如项目中尚未引入）

### 2.3 Out of Scope

- 不新增 pacingScore 的独立曲线（pacingScore 在已写章节上可后期叠加）
- 不做 AI 自动建议节奏（后续迭代）
- 不做节奏曲线的导出/分享功能
- 不改变 beat sheet 生成流程

---

## 3. 需求详情

### 3.1 节奏数据 API

THE SYSTEM SHALL 提供 `GET /api/novels/:novelId/pace-curve` 端点，返回：

```json
{
  "volumes": [
    {
      "volumeId": "...",
      "volumeSortOrder": 1,
      "title": "第一卷",
      "chapters": [
        {
          "chapterId": "...",
          "order": 1,
          "title": "第一章",
          "conflictLevel": 45,
          "revealLevel": 20,
          "pacingScore": null,
          "status": "planned",
          "isWritten": false
        }
      ]
    }
  ]
}
```

- `isWritten: true` — 对应 `Chapter` 记录存在且有内容，数据来自 `Chapter` 表
- `isWritten: false` — 仅有 `VolumeChapterPlan`，数据来自规划值
- `status` 取值：`"planned"` | `"writing"` | `"written"` | `"audited"`

### 3.2 节奏曲线图表

THE SYSTEM SHALL 渲染一个折线图：
- X 轴：章节序号（1, 2, 3...），按卷分组，卷边界有垂直虚线分隔
- Y 轴：0-100
- 两条折线：`conflictLevel`（红色 `#ef4444`）和 `revealLevel`（蓝色 `#3b82f6`）
- 已写章节（`isWritten: true`）：实心圆点
- 未写章节（`isWritten: false`）：空心圆点，可点击
- 图例可切换显示/隐藏各条线
- hover 时显示章节标题 + 具体数值 tooltip

### 3.3 节奏参数调整

WHEN 用户点击未写章节的空心点，THE SYSTEM SHALL 弹出调整面板：
- 显示章节标题
- conflictLevel 滑块（0-100）
- revealLevel 滑块（0-100）
- 实时预览：拖动滑块时折线图同步更新
- 保存按钮：调用 `PATCH /api/novels/:novelId/volumes/:volumeId/chapter-plans/:order` 更新 `VolumeChapterPlan`

WHEN 章节已写（`isWritten: true`），THE SYSTEM SHALL 仅展示数值，不显示编辑控件。

### 3.4 响应式布局

图表宽度自适应容器，最小宽度 600px，横向可滚动。在窄屏（< 768px）下只显示 conflictLevel 单线。

## 4. 验收标准

- [ ] 打开节奏曲线页面，能看到全书各章节的 conflictLevel 和 revealLevel 折线图
- [ ] 卷间有清晰的分隔标识，每卷着色可区分
- [ ] 已写章节显示实心点，未写章节显示空心点
- [ ] 点击未写章节空心点，弹出滑块可调整 conflictLevel 和 revealLevel
- [ ] 调整后保存，刷新页面后数据持久化
- [ ] 已写章节不可编辑，只显示数值
- [ ] hover 显示章节标题和具体数值
- [ ] 类型检查通过（`pnpm typecheck`）
- [ ] 前端测试通过（`pnpm test:client`）

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| recharts 可能与现有依赖冲突 | 先检查 `package.json`，如有冲突改用 visx 或纯 SVG 实现 |
| 章节数量多时图表性能 | 虚拟滚动或分卷加载，首屏只渲染当前卷 |
| `VolumeChapterPlan` 可能不存在（未进入规划阶段） | API 返回空 volumes 数组，前端显示"暂无节奏数据"占位 |

---

## 6. 关联与边界

- 与 REQ-2029 的关系：narrative_advisor 的 `summarize_chapter_range` 工具返回章节数据，但不返回节奏分数；本需求提供专门的节奏曲线视图
- 与现有 beat sheet 的关系：beat sheet 是文字版的节奏规划（`StructuredBeatSheetCard.tsx`），本需求是数值版的可视化图表，两者互补
