# REQ-7018 决策日志

## D1: 路由注册函数命名规范 `register<Domain>Routes`

**日期**：2026-07-10
**决策**：统一使用 `register<Domain>Routes(app: Express)` 命名规范
**原因**：与已有模块（如 `registerNovelRoutes`）保持一致，清晰表达意图
**影响**：所有新迁移路由文件遵循此规范

## D2: 分批迁移，每批后验证

**日期**：2026-07-10
**决策**：按依赖程度分三批迁移，每批后立即跑路由测试
**原因**：一次性迁移 30 个文件风险过高，分批可快速定位问题
**影响**：迁移周期可能较长，但风险可控

## D3: 保留 `routes/` 目录为 `_legacy/` 而非直接删除

**日期**：2026-07-10
**决策**：迁移完成后将 `routes/` 重命名为 `_legacy/` 或 `_archived_routes/`，保留一段时间再删除
**原因**：预留回滚空间，防止迁移遗漏导致线上问题
**影响**：短期内多一个归档目录，待稳定后删除
