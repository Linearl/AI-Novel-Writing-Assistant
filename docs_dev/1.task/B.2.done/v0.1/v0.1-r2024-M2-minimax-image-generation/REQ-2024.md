# REQ-2024 支持 MiniMax 图像生成

> 状态：✅ 已完成

---

## 1. 背景与问题

### 1.1 现状

当前图像生成仅支持 OpenAI 兼容的 API 格式。用户使用 MiniMax 作为 LLM Provider，但 MiniMax 的图像生成 API 使用不同的端点和请求格式：

**OpenAI 格式**:
- 端点: `/images/generations`
- 参数: `model`, `prompt`, `n`, `size`

**MiniMax 格式**:
- 端点: `/v1/image_generation`
- 参数: `model`, `prompt`, `n`, `aspect_ratio`, `response_format`

### 1.2 关键文件

| 文件 | 作用 |
| ---- | ---- |
| `server/src/services/image/provider.ts` | 图像生成核心：请求构建、API 调用、响应解析 |

---

## 2. 目标与范围

### 2.1 目标

1. 支持 MiniMax 图像生成 API 格式（请求体、端点、响应解析）
2. 支持 MiniMax 特有参数（`aspect_ratio`、`style`、`prompt_optimizer`）
3. 不影响现有 OpenAI 兼容格式

### 2.2 In Scope

- 修改 `buildImageGenerationRequestBody` 支持 MiniMax 格式
- 修改 `generateImagesByProvider` 支持 MiniMax 端点
- 修改 `parseImagesFromPayload` 支持 MiniMax 响应格式
- MiniMax 特有错误码处理

### 2.3 Out of Scope

- MiniMax 其他功能（视频、语音、音乐）
- 图像编辑功能（仅文生图）

---

## 3. 需求详情

### 3.1 请求体构建

WHEN 用户选择 MiniMax Provider 生成图像
THE SYSTEM SHALL 使用 MiniMax 格式构建请求体：
- `aspect_ratio` 替代 `size`
- 添加 `response_format`（默认 `"url"`）
- 添加 `prompt_optimizer`（默认 `false`）

### 3.2 API 调用

WHEN Provider 为 MiniMax
THE SYSTEM SHALL 使用以下配置：
- baseURL: `https://api.minimaxi.com`
- endpoint: `/v1/image_generation`
- 认证: Bearer token（复用现有 resolveProviderSecret）

### 3.3 响应解析

WHEN 响应来自 MiniMax API
THE SYSTEM SHALL 从 `data.image_urls[]` 提取图像 URL，处理 `base_resp.status_code` 错误。

## 4. 验收标准（EARS）

| ID | 验收条件 | 优先级 |
|----|----------|--------|
| AC-1 | 用户选择 MiniMax Provider 时，使用 `/v1/image_generation` 端点 | Must |
| AC-2 | 请求体包含 MiniMax 特有的 `aspect_ratio` 参数 | Must |
| AC-3 | 响应解析支持 MiniMax 的 `data.image_urls` 格式 | Must |
| AC-4 | 支持 MiniMax 的 `style` 参数（画风设置） | Should |
| AC-5 | 支持 MiniMax 的 `prompt_optimizer` 参数 | Could |

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| MiniMax API 可能有内容审核限制 | 记录错误码 1026，提示用户修改 prompt |
| MiniMax 错误码体系与 OpenAI 不同 | 专用错误处理，精确提示用户 |
