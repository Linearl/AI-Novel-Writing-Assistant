# REQ-2020: 自动导演流水线执行模式

## 1. 需求概述

### 1.1 背景

当前自动导演的章节执行流程是**批量模式**：
1. **结构化大纲阶段**：批量完成所有章节的细化（task sheet 生成）
2. **章节执行阶段**：批量完成所有章节的写作

这种模式的问题：
- 当章节细化完成但写作尚未开始时，存在等待浪费
- 用户无法更早看到已细化章节的内容

### 1.2 目标

引入**流水线执行模式**，实现章节细化与写作的交错执行：
- 章节 A 细化完成后，立即开始写作
- 同时章节 B 继续细化
- 以此类推，提高整体执行效率

### 1.3 用户价值

1. **时间节省**：预计可节省 30-40% 的总执行时间
2. **更早看到内容**：用户可以更早看到已完成的章节内容
3. **资源利用率**：CPU 和 API 调用更均衡

---

## 2. 功能需求

### 2.1 自动导演高级设置

**FR-2020-01：流水线模式开关**

在自动导演的高级设置中新增选项：
- **选项名称**：使用改进流水线
- **选项类型**：复选框（Checkbox）
- **默认值**：不勾选（保持当前批量模式）
- **说明文案**：启用后，章节细化与写作将交错执行，可提高整体执行速度

**FR-2020-02：暂停后可修改配置**

当自动导演暂停时，允许在任务中心中修改流水线模式配置：
- 暂停状态下，高级设置面板中的流水线模式开关可编辑
- 修改后继续执行时，使用新的配置

### 2.2 流水线执行逻辑

**FR-2020-03：细化与写作交错执行**

流水线模式下的执行流程：
```
第1章细化完成 → 第1章写作 + 第2章细化
                 ↓
第2章细化完成 → 第2章写作 + 第3章细化
                 ↓
...
```

**FR-2020-04：人工审核阻塞机制**

当某一章的写作完成后：
1. 执行自动审校
2. 如果审校未通过且用户配置需要人工审核：
   - **阻塞正文写作**：暂停后续章节的写作
   - **细化继续**：章节细化流水线继续执行
   - **等待人工处理**：用户审核修复后，恢复正文写作

状态流转：
```
章节A写作完成 → 审校未通过 → 需要人工审核
  ↓
阻塞正文写作（细化继续）
  ↓
人工审核修复完成
  ↓
恢复正文写作
```

**FR-2020-05：前置检查**

在开始某一章的写作前，必须确保：
- 该章节的细化已完成（task sheet 已生成）
- 上一章节的写作已完成（或已跳过）

### 2.3 状态管理

**FR-2020-06：流水线状态追踪**

新增状态字段，用于追踪流水线执行状态：
- `pipelineMode`: "batch" | "pipeline"
- `refinementProgress`: 细化进度
- `writingProgress`: 写作进度
- `blockedChapterId`: 被阻塞的章节 ID（如有）
- `blockingReason`: 阻塞原因

---

## 3. 非功能需求

### 3.1 性能

- 流水线模式不应显著增加 API 调用次数
- 细化与写作的交错不应导致资源竞争

### 3.2 可靠性

- 流水线模式下的错误处理应与批量模式一致
- 暂停/恢复功能应正常工作

### 3.3 兼容性

- 流水线模式应与现有的懒规划（JIT）模式兼容
- 不应影响现有的自动导演功能

## 4. 验收标准

### AC-2020-01：高级设置开关
- [ ] 自动导演高级设置中显示"使用改进流水线"复选框
- [ ] 默认不勾选
- [ ] 勾选后保存配置

### AC-2020-02：暂停后可修改
- [ ] 自动导演暂停时，流水线模式开关可编辑
- [ ] 修改后继续执行，使用新配置

### AC-2020-03：流水线执行
- [ ] 启用流水线模式后，章节细化与写作交错执行
- [ ] 章节 A 细化完成后立即开始写作
- [ ] 同时章节 B 继续细化

### AC-2020-04：人工审核阻塞
- [ ] 写作完成后审校未通过且需要人工审核时，阻塞后续写作
- [ ] 细化流水线继续执行
- [ ] 人工审核修复后，恢复写作

### AC-2020-05：状态追踪
- [ ] 流水线状态正确显示在 UI 中
- [ ] 阻塞状态有明确提示

---

## 5. 技术约束

1. 必须与现有的懒规划（JIT）模式兼容
2. 必须与现有的质量闭环机制兼容
3. 必须支持暂停/恢复功能
4. 必须支持取消功能
