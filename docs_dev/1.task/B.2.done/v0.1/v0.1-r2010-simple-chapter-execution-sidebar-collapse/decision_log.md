# REQ-2010 决策日志

## D1: Grid 列从 332px 缩到 40px 而非 0px

| 字段 | 内容 |
| ---- | ---- |
| 日期 | 2026-06-27 |
| 决策人 | AI |
| 决策 | 折叠时右侧列从 332px 缩到 40px（仅显示折叠按钮），不完全移除列 |
| 原因 | 完全移除列会导致 grid 模板列数变化（3列→2列），布局跳变较大。保留 40px 窄列让折叠按钮始终可见且位置稳定。 |
| 影响 | 折叠后仍有 40px 的窄条，正文区不会完全占满 |
| 备选方案 | 完全移除右侧列（2列布局）— 布局跳变大，展开按钮需要放在其他位置 |

## D2: 折叠按钮放在 Tab 栏最左侧

| 字段 | 内容 |
| ---- | ---- |
| 日期 | 2026-06-27 |
| 决策人 | AI |
| 决策 | 折叠按钮放在 TabsList 的最左侧，与 Tab 按钮同行 |
| 原因 | 用户要求"折叠按钮放（动态栏/资料诊断/AI执行台）卡片最左位置"。放在 Tab 栏内而非卡片外部，保持视觉一致性。 |
| 影响 | TabsList 从 `grid-cols-3` 变为 `grid-cols-4`（1按钮+3Tab） |
