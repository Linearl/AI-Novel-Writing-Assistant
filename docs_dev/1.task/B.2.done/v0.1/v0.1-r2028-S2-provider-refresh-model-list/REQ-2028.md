## 1. 背景与问题

### 1.1 现状

当前配置厂商弹窗（ProviderConfigDialog）中，"获取模型列表"按钮仅在新增自定义厂商时显示。内置厂商（如 MiniMax、DeepSeek 等）的弹窗内没有手动拉取模型列表的入口。

用户修改 API Key 或 Base URL 后，需要关闭弹窗、回到厂商卡片点击"刷新模型"才能更新列表，操作路径断裂。

### 1.2 根因

现有的"刷新模型"能力已完整：后端 `POST /settings/api-keys/:provider/refresh-models` API 和前端 `refreshModelsMutation` 均已就绪，只差弹窗内的触发入口。

### 1.3 关键文件

| 文件 | 作用 |
| ---- | ---- |
| `client/src/pages/settings/components/ProviderConfigDialog.tsx` | 厂商配置弹窗，需新增按钮 |
| `client/src/pages/settings/SettingsPage.tsx` | 设置页面，需传递 mutation 到弹窗 |

---

## 2. 目标与范围

### 2.1 目标

1. 在配置厂商弹窗的模型选择区域新增"拉取模型列表"按钮
2. 与"测试连接"按钮同级，支持内置厂商和自定义厂商

### 2.2 In Scope

- UI：ProviderConfigDialog 中模型选择区域新增"拉取模型列表"按钮
- 按钮显示条件：已配置 API Key 或 Base URL（有足够信息发起请求）
- 点击后调用已有的 `refreshProviderModelList` API，更新 `selectableModels`

### 2.3 Out of Scope

- 新增后端 API（已有）
- 修改厂商卡片上的"刷新模型"按钮逻辑

---

## 3. 需求详情

### 3.1 按钮位置

WHEN 用户打开配置厂商弹窗
THE SYSTEM SHALL 在模型选择区域显示"拉取模型列表"按钮，位于"测试连接"按钮同级。

### 3.2 触发行为

WHEN 用户点击"拉取模型列表"按钮
THE SYSTEM SHALL 调用 `refreshProviderModelList` API，显示 loading 状态，成功后更新 `selectableModels`，下拉列表刷新。

### 3.3 错误处理

WHEN API 调用失败
THE SYSTEM SHALL 显示错误提示，不关闭弹窗。

### 3.4 兼容性

WHEN 用户打开自定义厂商弹窗
THE SYSTEM SHALL 保持现有"获取模型列表"按钮行为不变。
