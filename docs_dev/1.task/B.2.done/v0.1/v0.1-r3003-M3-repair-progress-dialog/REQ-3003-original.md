# REQ-3003 章节修复进度感知弹窗 — 原始需求

> 本文件为需求冻结副本。创建于 2026-06-26，此后不再修改。

## 背景

目前审校发现问题后，用户在"AI 执行台"点击"自动修复问题"按钮，章节修复过程中用户没有进度感知。需要新增一个查看详情按钮，放在修复按钮下方，点击后弹窗显示修复的过程。

## 用户痛点

1. 点击"自动修复问题"后，按钮变为"正在自动修复..."，除此之外没有任何进度反馈
2. 修复过程的 SSE 流式输出藏在"资料诊断 → 修复"子标签里，用户需要在右侧面板切标签才能看到
3. 用户不知道修复是否在进行、进度如何、还要等多久

## 期望行为

- 修复进行中时，在"自动修复问题"按钮下方出现"查看详情"按钮
- 点击"查看详情"弹出 Dialog 弹窗
- 弹窗内显示：修复过程的流式文本输出 + 当前状态（阶段名 + 消息）
- 弹窗支持实时跟随流式输出滚动
- 修复完成后弹窗保持可查看（显示完整修复内容），用户可以关闭
