# REQ-2036: 用户反馈系统

## 1. 需求概述

### 1.1 背景

AI 小说创作助手需要一个用户反馈系统，让用户能够：
- 提交使用过程中遇到的问题
- 上传截图作为问题证据
- 查看反馈处理进度

### 1.2 目标

构建一个完整的用户反馈系统，包括：
- 后端 API（反馈提交、列表、详情、归档、评论）
- 文件系统存储（每条反馈独立文件夹）
- 前端 UI（反馈入口、列表、详情、评论）

### 1.3 非目标

- 不实现 AI 自动分析反馈（可后续迭代）
- 不实现自动修复功能
- 不实现邮件通知

## 2. 功能需求

### 2.1 反馈提交

**用户故事**：作为用户，我希望能提交问题反馈，以便开发团队了解并修复问题。

**验收标准**：
- [ ] 用户可填写标题（必填，1-200 字符）
- [ ] 用户可填写描述（必填，1-5000 字符）
- [ ] 用户可选择严重程度（P1/P2/P3/P4）
- [ ] 用户可上传附件（图片，最多 5 张，每张最大 5MB）
- [ ] 提交成功后显示确认信息
- [ ] 速率限制：每小时最多 10 次提交

### 2.2 反馈存储

**存储结构**：
```
server/feedback/
├── 1_20260702_093000/          # {userId}_{timestamp}
│   ├── issue.md                # 反馈内容（含 frontmatter）
│   └── attachments/            # 附件目录
│       └── image.png
├── 2_20260702_100000/
│   └── issue.md
└── fixed/                      # 已归档反馈
    └── 1_20260601_120000/
        └── issue.md
```

**issue.md 格式**：
```markdown
---
title: "反馈标题"
severity: P2
submitted_at: "2026-07-02T01:30:00.000Z"
user_id: "1"
---

# 反馈标题

## 描述

反馈详细描述...

## 附件

![image.png](attachments/image.png)
```

### 2.3 反馈列表

**用户故事**：作为管理员，我希望能查看所有反馈列表，以便了解用户问题。

**验收标准**：
- [ ] 显示所有未归档反馈
- [ ] 每条显示：标题、严重程度、提交时间、用户 ID
- [ ] 支持按严重程度筛选
- [ ] 支持按时间排序
- [ ] 显示是否有附件、是否有分析结果

### 2.4 反馈详情

**用户故事**：作为管理员，我希望能查看反馈详情，以便了解问题全貌。

**验收标准**：
- [ ] 显示反馈完整内容
- [ ] 显示附件（图片可预览）
- [ ] 显示评论列表
- [ ] 提供归档操作入口

### 2.5 反馈归档

**用户故事**：作为管理员，我希望能归档已处理的反馈。

**验收标准**：
- [ ] 归档后反馈移至 `fixed/` 目录
- [ ] 归档操作需确认
- [ ] 归档后不再显示在默认列表

### 2.6 评论系统

**用户故事**：作为管理员，我希望能对反馈添加评论。

**验收标准**：
- [ ] 管理员可添加评论
- [ ] 评论按时间排序
- [ ] 评论内容支持 Markdown

## 3. 技术需求

### 3.1 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/feedback | 提交反馈 |
| GET | /api/feedback/admin/reviews | 获取反馈列表 |
| GET | /api/feedback/admin/reviews/:folderName | 获取反馈详情 |
| POST | /api/feedback/admin/reviews/:folderName/archive | 归档反馈 |
| DELETE | /api/feedback/admin/reviews/:folderName | 删除反馈 |
| GET | /api/feedback/:folderName/comments | 获取评论列表 |
| POST | /api/feedback/:folderName/comments | 添加评论 |
| GET | /api/feedback/:folderName/attachments/:fileName | 获取附件 |

### 3.2 安全要求

- 反馈提交需速率限制
- 附件上传需限制文件类型和大小
- 管理员接口需认证

### 3.3 性能要求

- 反馈列表加载 < 1s
- 附件上传支持断点续传（可选）

## 4. 验收标准

1. 所有 API 端点正常工作
2. 文件系统存储结构正确
3. 前端 UI 可正常提交和查看反馈
4. 速率限制生效
5. 类型检查通过
6. 单元测试通过
