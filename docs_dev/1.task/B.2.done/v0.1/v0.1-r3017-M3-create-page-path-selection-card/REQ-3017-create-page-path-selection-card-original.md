# REQ-3017 创建页面路径选择卡片

## 任务描述

在创建小说页面顶部新增路径选择卡片，统一AI自动导演开书、素材导入、手动填写三种路径，简化用户选择流程。

## 需求要点

1. **页面布局调整**
   - 快速预览卡片保持独立显示（顶部）
   - 新增路径选择卡片（快速预览下方）
   - 根据路径选择动态显示对应内容区

2. **路径选择卡片**
   - 路径A：我有初步想法，AI自动导演开书
   - 路径B：我有完善想法，从素材导入
   - 路径C：都不需要，我要手动填写

3. **路径A - AI自动导演开书**
   - 点击后直接打开 NovelAutoDirectorDialog 对话框
   - 用户输入想法 → AI生成候选 → 选择 → 自动填充所有字段
   - 对话框完成后显示表单，用户可继续微调

4. **路径B - 素材导入**
   - 点击后在下方显示素材导入框
   - 用户导入/粘贴长文本 → AI解析 → 显示预览 → 用户确认 → 填充字段
   - 导入完成后用户可继续编辑

5. **路径C - 手动填写**
   - 点击后在下方显示字段表单
   - 提示文字："可以开始手动填写"
   - 用户逐个填写字段

6. **统一目标**
   - 无论走哪条路径，最终都填充 Novel 表的同一组字段
   - 用户在第一步就选择适合自己的创作方式

## 复杂度评估

- **涉及文件**：3-4个文件（主要是 NovelCreate.tsx 和子组件）
- **跨模块范围**：单模块（client/src/pages/novels）
- **架构影响**：局部重构（UI层面）
- **测试难度**：需新增单元测试
- **回滚成本**：直接revert
- **依赖链**：无外部依赖

→ **复杂度**：M（中等）
→ **优先级**：P3（改进）
→ **Tag**：M3
