# REQ-3017 决策记录

## 决策1：快速预览卡片保持独立

**背景**：快速预览卡片是一个独立功能，允许用户输入一句话灵感，AI生成3个候选方案。

**决策**：快速预览卡片保持独立显示，放在路径选择卡片的上方。

**理由**：
- 快速预览是独立功能，不应该被路径选择限制
- 用户可能想先生成灵感，再决定走哪条路径
- 保持功能独立性，便于后续扩展

**影响**：无负面影响，快速预览功能保持不变。

---

## 决策2：路径A直接打开对话框

**背景**：路径A是"AI自动导演开书"，需要用户输入想法。

**决策**：点击路径A后直接打开 NovelAutoDirectorDialog 对话框。

**理由**：
- 用户已经选择了AI模式，没必要再显示额外的引导
- 直接打开对话框，用户体验更流畅
- 对话框本身已经有完整的引导流程

**影响**：路径A的流程更简洁，用户直接进入AI交互。

---

## 决策3：统一填充Novel表字段

**背景**：无论走哪条路径，最终都填充同一组字段。

**决策**：三种路径都填充 Novel 表的同一组字段（description、worldSetting、characters、outline等）。

**理由**：
- 底层数据模型统一，便于维护
- 用户不需要关心不同路径的差异
- 最终目标一致：创建小说项目

**影响**：路径选择只是UI层面的改变，不影响底层数据流。

---

## 决策4：路径选择卡片位置

**背景**：需要决定路径选择卡片在页面中的位置。

**决策**：放在快速预览卡片的下方，创建小说表单的上方。

**理由**：
- 用户先看到快速预览（可选），再选择路径
- 路径选择是必选步骤，放在表单上方
- 页面布局从上到下：快速预览 → 路径选择 → 动态内容区

**影响**：页面布局更清晰，用户流程更顺畅。

---

## 决策5：路径C的提示文字

**背景**：路径C是"手动填写"，用户直接填写表单。

**决策**：在路径C的动态内容区显示提示文字："可以开始手动填写"。

**理由**：
- 给用户明确的引导
- 说明表单中的"AI帮我填"按钮可以使用
- 降低用户的认知负担

**影响**：用户体验更好，更清楚如何操作。
