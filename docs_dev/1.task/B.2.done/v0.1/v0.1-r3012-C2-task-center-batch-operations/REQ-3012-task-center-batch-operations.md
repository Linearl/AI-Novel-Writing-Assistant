# REQ-3012: 任务中心批量操作功能

## 背景

任务中心当前仅支持单个任务操作，当用户需要批量处理任务（如批量取消排队中的任务）时，必须逐个点击，效率低下。

## 目标

为任务中心任务列表添加多选、全选和批量取消功能，提升任务管理效率。

## 范围

### 功能需求

1. **多选支持**
   - 每个任务项左侧添加复选框
   - 支持点击复选框选择/取消单个任务
   - 支持点击任务项本身不触发选择（保持查看详情功能）

2. **全选功能**
   - 任务列表顶部添加全选复选框
   - 全选框状态：
     - 未选中：没有任务被选中
     - 部分选中：部分任务被选中
     - 全选中：所有任务被选中
   - 点击全选框切换选中状态

3. **批量取消**
   - 仅当有任务被选中时显示批量操作栏
   - 显示已选中任务数量
   - 批量取消按钮：点击后取消所有选中的可取消任务
   - 不可取消的任务（如正在运行的）自动跳过并提示

4. **状态管理**
   - 选中状态本地管理（useState）
   - 切换筛选条件或排序时清除选中状态
   - 任务列表刷新时清除选中状态

### 技术需求

- 保持现有任务列表功能不变
- UI 符合项目设计规范
- 选中状态不持久化（页面刷新后清除）
- 类型安全，无 TypeScript 错误

## 验收标准

- [ ] 复选框正确显示在每个任务项左侧
- [ ] 全选框状态正确反映当前选择情况
- [ ] 批量取消仅在有选中任务时显示
- [ ] 批量取消按钮调用正确的 API
- [ ] 不可取消的任务被自动跳过
- [ ] 选中状态在筛选/排序/刷新时正确清除
- [ ] 类型检查通过
- [ ] 无控制台错误
