# REQ-7004 Decision Log

## D1：边表 vs 图数据库

- **决策**：使用 Prisma 边表，不引入图数据库
- **理由**：当前无多跳遍历需求，边表 + FK 约束足以解决三层问题；图数据库作为独立后续需求（REQ-7xxx）
- **日期**：2026-06-30
- **来源**：图数据库能力诊断报告建议

## D2：迁移策略选择

- **决策**：渐进式迁移（新增 → 双写 → 适配 → 移除），非一次性替换
- **理由**：降低风险，每个阶段独立可验证可回滚
- **日期**：2026-06-30

## D3：不迁移 stateChangesJson 和 issuesJson

- **决策**：这两个字段存储的是描述性对象（非实体 ID 引用），不在本次迁移范围
- **理由**：它们不是图边，是结构化日志数据
- **日期**：2026-06-30

## D4：双向边存储策略

- **决策**：互逆关系存储两条独立边，在写入时通过事务保证一致性
- **备选**：只存一条边，读取时双向查询
- **理由**：两条边虽然冗余，但查询更简单（只需单向），且与 DirectorArtifactDependency 现有模式一致
- **日期**：2026-06-30

## D5：World.structureJson 过渡期双写

- **决策**：过渡期同时写新表和旧 JSON，最终只保留新表
- **理由**：worldVisualization.ts 和其他消费方可能依赖 structureJson，需要过渡期验证
- **日期**：2026-06-30
