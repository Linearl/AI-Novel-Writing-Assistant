# REQ-7004 JSON 软引用 FK 缺失修复

> 创建日期：2026-06-30
> 最后更新：2026-06-30
> 复杂度：medium
> 分类：7xxx（技术债务和重构）
> 优先级：P1

---

## 1. 目标

将 Prisma schema 中 19 个 `*IdsJson` JSON 软引用字段迁移为独立 FK 边表，消除三层问题（悬空引用、双向不一致、无法索引查询），同时保持现有 API 接口不变。

## 2. 范围

### 2.1 P0 — 时间线领域（核心图结构）

**受影响模型**：StoryTimelineEvent、ChapterTimeAnchor、TimelineHook、TimelineConstraint

**新增边表**：
- `TimelineEventEdge` — 事件间因果/禁止边（prerequisite, consequence, forbidden）
- `TimelineEventParticipant` — 事件参与者边（event ↔ character）
- `TimelineEventFaction` — 事件势力关联边（event ↔ faction）
- `TimelineAnchorEventLink` — 锚点-事件关联边（anchor ↔ event，含 linkType: startsAfter/plannedEvent/endedWith）
- `TimelineHookEventLink` — Hook-事件关联边（hook ↔ event）
- `TimelineHookParticipant` — Hook 参与者边（hook ↔ character）
- `TimelineConstraintLink` — 约束关联边（constraint ↔ event/hook/character，多态引用）

### 2.2 P1 — 世界设定领域

**受影响模型**：World.structureJson（JSON Blob）

**变更**：将 `WorldRelations`（forceRelations, locationControls, locationConnections）从 JSON Blob 提取为独立 Prisma 表：
- `WorldForceRelation` 表
- `WorldLocationControl` 表
- `WorldLocationConnection` 表

### 2.3 P2 — 其他零散字段

**受影响模型**：OpenConflict、CharacterResource、StoryPlan、CanonicalStateVersion

- `OpenConflict.affectedCharacterIdsJson` → `OpenConflictCharacter` 边表
- `CharacterResource.knownByCharacterIdsJson` → `CharacterResourceKnownBy` 边表
- `StoryPlan.sourceIssueIdsJson` → `StoryPlanIssue` 边表
- `CanonicalStateVersion.acceptedProposalIdsJson` → `StateVersionProposal` 边表

## 3. 非目标

- 不引入图数据库（KuzuDB 等作为后续独立需求）
- 不修改前端可视化层（worldVisualization.ts 的邻接表构建逻辑适配新表即可）
- 不改变 API 请求/响应格式

## 4. EARS 验收条目

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| E1 | 删除被引用实体后，边表自动级联删除（无悬空引用） | 单元测试：删除 event → 验证 edge 表无残留 |
| E2 | 写入互逆边时双向一致（事务保证） | 单元测试：写入 prerequisite → 验证 consequence 自动创建 |
| E3 | 边表支持按 source/target 索引查询 | 集成测试：按 targetId 查询 → 走索引 |
| E4 | 存量数据迁移后记录数一致 | 迁移脚本校验：JSON 数组元素总数 == 边表行数 |
| E5 | 现有 API 响应格式不变 | 回归测试：对比迁移前后 API 响应 |
| E6 | prisma migrate dev 通过 | 迁移脚本无报错 |
| E7 | typecheck + test + build 全绿 | CI 验证 |

## 5. 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 存量数据迁移丢失 | 高 | 迁移前备份 + 迁移后校验 |
| 性能回退（边表 JOIN 替代 JSON 解析） | 低 | 边表有索引，JOIN 应优于全表扫描 + JSON 解析 |
| World.structureJson 提取影响可视化 | 中 | worldVisualization.ts 适配新数据源 |

## 6. 假设

- 现有 JSON 字段中的 ID 全部指向有效实体（无历史悬空引用）
- 时间线领域是当前使用最频繁的图结构，优先修复
