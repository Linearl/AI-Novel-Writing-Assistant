# REQ-2019 重复工具调用循环检测

> 创建日期：2026-06-27
> 目标版本：v0.1
> 状态：✅ 完成

---

## 1. 任务概述

### 1.1 需求来源

REQ-2002（流式死循环检测）的补充。REQ-2002 解决了**文本级**死循环（LLM 输出重复内容），但无法覆盖**结构级**死循环——即 LLM 反复输出相同参数的工具调用 JSON，导致同一工具被反复调用。

### 1.2 核心内容

1. 在 `RunExecutionService.runActionPlan` 中新增工具调用历史追踪
2. 当同一工具 + 相同参数被调用达到 3 次时，中断当前 action 执行
3. 返回 `LOOP_DETECTED` 错误码，附带循环工具名称和参数摘要
4. 错误消息引导模型换策略（"该工具已用相同参数调用 3 次，请换一种方式完成任务"）

### 1.3 前置条件

- 无（独立功能，仅修改 `RunExecutionService`）

## 3. 验收标准

- [ ] 同一工具 + 相同参数调用 3 次 → 打断执行
- [ ] 返回 `LOOP_DETECTED` 错误码 + 中文提示
- [ ] 不同参数的同一工具调用不触发检测
- [ ] 不同工具的调用不触发检测
- [ ] 单元测试覆盖：正常调用、相同参数触发、不同参数不触发、重置后不累积
