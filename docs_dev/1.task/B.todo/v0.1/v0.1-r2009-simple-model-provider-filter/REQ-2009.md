## 1. 背景与问题

### 1.1 现状

模型路由设置页（`ModelRoutesPage`）的厂商下拉框显示**全部 11 个内置厂商**，无论用户是否配置了 API Key：

| 厂商 | 是否显示 | 是否已配置 |
| ---- | ---- | ---- |
| DeepSeek | ✓ | ✓ 已配置 |
| SiliconFlow | ✓ | ✓ 已配置 |
| OpenAI | ✓ | ✗ 未配置 |
| Anthropic | ✓ | ✗ 未配置 |
| Grok | ✓ | ✗ 未配置 |
| Kimi | ✓ | ✗ 未配置 |
| MiniMax | ✓ | ✗ 未配置 |
| GLM | ✓ | ✗ 未配置 |
| Qwen | ✓ | ✗ 未配置 |
| Gemini | ✓ | ✗ 未配置 |
| Ollama | ✓ | ✗ 未配置 |

### 1.2 问题

- 用户只配置了 2-3 个厂商，但每次都要在 11 个选项中找到目标
- 未配置的厂商选中后无法使用，容易造成困惑
- 与 `LLMSelector` 组件的行为不一致（后者已做过滤）

### 1.3 已有过滤逻辑

`LLMSelector` 组件已经使用 `isRunnableProviderConfig()` 过滤：

```typescript
// client/src/lib/llmSelection.ts (line 36-38)
export function isRunnableProviderConfig(config: APIKeyStatus): boolean {
  return config.isConfigured && config.isActive && getProviderSelectionModels(config).length > 0;
}
```

**但 `ModelRoutesPage` 没有使用这个过滤函数。**

## 2. 目标与范围

### 2.1 目标

1. 模型路由设置页的厂商下拉框仅显示已配置且可用的厂商
2. 未配置的厂商不显示在下拉框中
3. 已有路由引用的厂商即使后来被禁用，编辑时仍可见（带禁用标记）

### 2.2 In Scope

**前端：**
- `ModelRoutesPage` 应用 `isRunnableProviderConfig` 过滤
- 已有路由引用的未配置厂商仍可见（灰色标记）
- 空状态提示（"暂无可用厂商，请先在 API Key 设置中配置"）

### 2.3 Out of Scope

- 服务端新增 API 端点（复用现有 `/settings/api-keys`）
- 厂商配置入口改造
- 厂商排序逻辑变更

## 3. 需求详情

### 3.1 厂商下拉框过滤

WHEN 用户在模型路由设置页编辑厂商下拉框
THE SYSTEM SHALL 仅显示满足以下条件的厂商：

1. `isConfigured === true`（已配置 API Key 或环境变量）
2. `isActive === true`（未被禁用）
3. 至少有一个可用模型

使用已有的 `isRunnableProviderConfig()` 函数，不新增过滤逻辑。

### 3.2 已有路由的厂商保护

WHEN 用户编辑一个已有路由，且该路由引用的厂商当前不可用（如 API Key 被删除）
THE SYSTEM SHALL：

- 在下拉框中仍然显示该厂商（作为额外选项）
- 该厂商选项显示为灰色 + 标签"（未配置）"
- 用户可以切换到其他可用厂商，但不能保存为未配置厂商

### 3.3 空状态

WHEN 没有任何可用厂商
THE SYSTEM SHALL 在下拉框位置显示提示：

```
暂无可用厂商，请先在「API Key 设置」中配置至少一个厂商。
```

并提供跳转链接到 API Key 设置页。

---

## 4. 验收标准

- [ ] 模型路由设置页厂商下拉框仅显示已配置可用的厂商
- [ ] 未配置的厂商不出现在下拉框中
- [ ] 已有路由引用的未配置厂商仍可见（灰色 + "未配置"标签）
- [ ] 无可用厂商时显示空状态提示和跳转链接
- [ ] `LLMSelector` 组件行为不受影响（已过滤）
- [ ] 新建路由时只能选择可用厂商
- [ ] 编辑已有路由时，当前厂商始终可见（即使不可用）

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 用户删除 API Key 后已有路由无法编辑 | 已有路由的当前厂商始终可见（保护机制） |
| 过滤后下拉框可能为空 | 空状态提示引导用户去配置 API Key |

---

## 6. 关联与边界

- 与 `LLMSelector` 的关系：复用同一过滤函数 `isRunnableProviderConfig()`
- 与 `/settings/api-keys` 的关系：不修改 API，客户端过滤
- 与 API Key 设置页的关系：空状态提示跳转到该页面
