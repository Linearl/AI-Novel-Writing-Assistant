# REQ-2006 步骤4每卷章节数手动调整

## 1. 背景与问题

### 1.1 现状

当前章节分配是全自动的：

1. 用户设置全书预计章节数（`estimatedChapterCount`，最低12）
2. 用户设置卷数（`OutlineTab` 中的"自定义卷数"开关）
3. 系统通过 `allocateChapterBudgets()` 自动分配每卷章节数：
   - 有已有章节：按比例加权分配
   - 无已有章节：均匀分配（最少3章/卷）

用户**无法**手动调整单卷的章节数。

### 1.2 用户场景

1. 用户希望第一卷多写几章来铺垫世界观
2. 用户希望最后一卷章节数少一些，节奏更紧凑
3. 用户对自动分配结果不满意，想要微调
4. 用户在生成节奏板前想预设每卷的大致章节数

### 1.3 当前代码

| 文件 | 函数 | 作用 |
| ---- | ---- | ---- |
| `server/src/services/novel/volume/volumeChapterBudgetAllocation.ts` | `deriveChapterBudget()` | 推导全书总章节数预算 |
| `server/src/services/novel/volume/volumeChapterBudgetAllocation.ts` | `allocateChapterBudgets()` | 将总预算分配到各卷 |
| `server/src/services/novel/volume/volumeBeatSheetGeneration.ts` | `resolveBeatSheetTargetChapterCount()` | 确定单卷目标章节数 |
| `client/src/pages/novels/components/OutlineTab.tsx` | — | 卷数配置 UI |

## 2. 目标与范围

### 2.1 目标

1. 用户在步骤4（卷战略/卷骨架工作台）中可以手动设置每卷的目标章节数
2. 手动设置优先于自动分配
3. 设置后影响节奏板生成和章节列表

### 2.2 In Scope

**共享类型：**
- `VolumePlan` 接口新增 `targetChapterCount?: number | null`

**后端：**
- `allocateChapterBudgets()` 尊重每卷的手动覆盖
- `resolveBeatSheetTargetChapterCount()` 优先使用手动值

**前端：**
- 步骤4卷卡片中新增章节数输入控件
- 修改章节数后实时更新显示

### 2.3 Out of Scope

- 全书总章节数自动重算（手动调整单卷后不自动调整其他卷）
- 章节数上限/下限强制校验（仅做提示）
- 步骤5的章节细化自动重做

---

## 3. 需求详情

### 3.1 数据模型

`VolumePlan` 接口新增字段：

```typescript
interface VolumePlan {
  // ... existing fields
  targetChapterCount?: number | null;  // 手动设置的目标章节数，null 表示使用自动分配
}
```

### 3.2 UI：卷卡片章节数输入

WHEN 用户在步骤4查看卷卡片
THE SYSTEM SHALL 在每卷卡片中显示章节数配置：

- 标签："目标章节数"
- 输入框：数字输入，最小值 3，最大值 50
- 占位文本：显示自动分配值（如"自动: 8章"）
- 清除按钮：恢复为自动分配（设为 null）

### 3.3 后端分配逻辑

WHEN `allocateChapterBudgets()` 执行分配
THE SYSTEM SHALL：

1. 先检查每卷是否有 `targetChapterCount` 手动值
2. 有手动值的卷使用手动值
3. 无手动值的卷从剩余预算中自动分配
4. 若手动值总和超过总预算，按比例缩减自动分配的卷（最少3章/卷）

### 3.4 节奏板生成

WHEN `resolveBeatSheetTargetChapterCount()` 确定单卷章节数
THE SYSTEM SHALL 优先使用 `volume.targetChapterCount`（如有），否则使用自动分配值。

## 4. 验收标准

- [ ] 步骤4卷卡片中显示"目标章节数"输入框
- [ ] 输入数字后保存到 `VolumePlan.targetChapterCount`
- [ ] 清除输入后恢复为自动分配
- [ ] 节奏板生成时使用手动设置的章节数
- [ ] 自动分配逻辑正确处理部分手动覆盖的情况
- [ ] 手动值总和超过总预算时给出提示

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 手动值不合理导致节奏板质量下降 | 设置合理的 min/max 范围，超出时提示但不阻断 |
| 手动调整后已有的节奏板失效 | 提示用户需要重新生成节奏板 |
