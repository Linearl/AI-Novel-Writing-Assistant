## 1. 任务概述

### 1.1 需求来源

用户需求：右上角通知缺乏具体错误信息，日志无持久化，无日志查看入口。

### 1.2 核心内容

1. **后端日志基础设施**：引入 winston 结构化日志，持久化到文件，全局错误中间件增强
2. **日志查询 API**：支持按等级、时间范围筛选，分页返回
3. **前端日志中心页面**：侧边栏入口 + 日志列表 + 筛选 + 详情抽屉
4. **toast 通知增强**：具体错误信息 + "查看详情"跳转

### 1.3 前置条件

- 无阻塞前置条件
- 现有 morgan HTTP 日志保持不变
