## 1. 背景与问题

当前系统存在以下日志和通知问题：

1. **通知信息不足**：右上角弹出的 toast 通知（sonner）在 5xx 错误时仅显示"服务器错误，请稍后重试"等泛化消息，用户无法获取具体错误原因。
2. **日志无持久化**：服务端仅有 morgan 做 HTTP 请求日志，无结构化应用日志，错误上下文随进程消失。
3. **无日志查看入口**：前端没有日志查看界面，用户和开发者无法追溯历史错误。

---

## 2. 目标与范围

### 2.1 目标

1. 在左侧导航栏"系统设置"下方新增"日志中心"入口
2. 右上角通知必须携带具体错误信息，日志中心有详细记录
3. 日志持久化保存到服务端 log 文件
4. 日志中心支持按时间范围、日志等级筛选

### 2.2 In Scope

**后端**：
- 引入 winston 结构化日志库
- 日志持久化到 `server/logs/` 目录
- 全局错误处理中间件增强，详细错误写入日志
- 日志查询 API（`GET /api/logs`），支持 level / startTime / endTime / page / pageSize

**前端**：
- 侧边栏"系统"分组新增"日志中心"导航项
- 日志中心页面：日志列表、时间范围筛选、等级筛选、分页
- toast 通知增强：显示更具体的错误信息

### 2.3 Out of Scope

- 日志实时推送（WebSocket / SSE）
- 日志导出功能
- 日志分析 / 统计图表
- 移动端日志中心适配（后续迭代）

---

## 3. 需求详情

### 3.1 导航入口

WHEN 用户查看左侧导航栏
THE SYSTEM SHALL 在"系统"分组的"系统设置"下方显示"日志中心"入口，路由为 `/logs`

### 3.2 日志持久化

WHEN 服务端发生任何 warn / error 级别事件
THE SYSTEM SHALL 将结构化日志（时间戳、等级、消息、堆栈、请求上下文）写入 `server/logs/` 目录下的文件

IF 日志文件超过 10MB
THEN THE SYSTEM SHALL 自动轮转（按日期归档）

### 3.3 错误通知增强

WHEN API 返回 4xx / 5xx 错误
THE SYSTEM SHALL 在 toast 通知中显示具体的错误消息（非泛化文案），并提供"查看详情"入口

### 3.4 日志查询 API

WHEN 前端请求 `GET /api/logs`
THE SYSTEM SHALL 返回日志列表，支持以下筛选参数：
- `level`: debug / info / warn / error
- `startTime`: ISO 时间戳
- `endTime`: ISO 时间戳
- `page` / `pageSize`: 分页

### 3.5 日志中心页面

WHEN 用户访问 `/logs`
THE SYSTEM SHALL 展示日志列表页面，包含：
- 等级筛选下拉
- 时间范围选择器
- 日志条目：时间戳、等级标签、来源模块、消息、可展开的详情（堆栈 / 请求上下文）
- 分页控件

## 4. 验收标准

- [ ] 侧边栏"系统设置"下方显示"日志中心"入口
- [ ] 点击"日志中心"可正确打开日志列表页
- [ ] 服务端 warn / error 日志持久化到 `server/logs/` 文件
- [ ] API 错误时 toast 显示具体错误信息而非泛化文案
- [ ] 日志中心支持按等级筛选
- [ ] 日志中心支持按时间范围筛选
- [ ] 日志中心支持分页浏览
- [ ] 日志条目可展开查看详情（堆栈 / 上下文）

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 日志文件过大影响磁盘 | winston daily rotate file，保留 14 天 |
| 日志查询 API 性能 | 限制分页大小，按日期倒序，仅返回最近 30 天 |
| 敏感信息泄露 | 日志 API 需鉴权，日志中不记录密钥/token |

- 与 REQ-2021 / REQ-2022（导演调试日志）的边界：REQ-2021/2022 是 auto-director 专用调试日志，本需求是全局应用级日志中心
