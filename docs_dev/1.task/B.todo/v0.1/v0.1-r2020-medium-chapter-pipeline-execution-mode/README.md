# REQ-2020: 自动导演流水线执行模式

## 概述

引入**流水线执行模式**，实现章节细化与写作的交错执行，提高自动导演的执行效率。

## 核心价值

1. **时间节省**：预计可节省 30-40% 的总执行时间
2. **更早看到内容**：用户可以更早看到已完成的章节内容
3. **资源利用率**：CPU 和 API 调用更均衡

## 功能特性

### 1. 自动导演高级设置
- 新增"使用改进流水线"复选框
- 默认不勾选（保持当前批量模式）
- 暂停状态下可修改配置

### 2. 流水线执行逻辑
- 细化与写作交错执行
- 人工审核阻塞机制
- 前置检查确保 task sheet 已生成

### 3. 状态管理
- 流水线状态追踪
- 阻塞状态显示
- 暂停/恢复支持

## 技术栈

- **后端**：TypeScript, Express, Prisma
- **前端**：React, TypeScript, Vite
- **数据库**：SQLite (dev) / PostgreSQL (prod)

## 验收标准

- [ ] 自动导演高级设置中显示"使用改进流水线"复选框
- [ ] 默认不勾选
- [ ] 启用流水线模式后，章节细化与写作交错执行
- [ ] 写作完成后审校未通过且需要人工审核时，阻塞后续写作
- [ ] 细化流水线继续执行
- [ ] 人工审核修复后，恢复写作
- [ ] 流水线状态正确显示在 UI 中

## 预估工时

5 天

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 细化未完成就开始写作 | 写作失败 | 添加前置检查 |
| 并行执行导致资源竞争 | 性能下降 | 限制并行度 |
| 全局状态不一致 | 连续性问题 | 使用数据库事务 |
| 人工审核阻塞逻辑复杂 | 调试困难 | 添加详细日志 |

## 备注

- 此需求来源于用户对自动导演执行效率的优化需求
- 需要与现有的懒规划（JIT）模式兼容
- 需要支持暂停/恢复/取消功能
