# REQ-2021 决策日志

## DEC-001: 日志存储介质选择

- **决策**: 使用磁盘 JSON 文件存储调试日志，不使用数据库
- **替代方案**: 扩展 `DirectorAutomationLedgerEventService` 在数据库中增加详细调试字段
- **理由**: 调试日志体积大（完整状态快照）、写入频率低（仅断路器触发时）、生命周期短（保留 100 个），磁盘文件更适合这种模式。数据库方案会增加迁移成本且不适合存储大 JSON 文档
- **影响**: 新增 `server/logs/director-debug/` 目录，需加入 `.gitignore`
- **日期**: 2026-06-28

## DEC-002: 集成方式 — fire-and-forget

- **决策**: 日志保存调用不 await，使用 `.catch(() => {})` 兜底
- **替代方案**: await 写入完成后再返回
- **理由**: `stopAutoExecutionForCircuitBreaker` 是关键路径，其延迟直接影响用户体验。日志写入是辅助功能，不应增加关键路径延迟。写入失败不影响断路器核心逻辑
- **影响**: 日志写入可能在进程退出时丢失，但断路器触发场景下进程通常不会立即退出
- **日期**: 2026-06-28

## DEC-003: 配置默认值

- **决策**: `DIRECTOR_DEBUG_LOG_ENABLED` 默认开启（未设值 = 启用）
- **替代方案**: 默认关闭，需显式开启
- **理由**: 调试日志主要在开发环境使用，默认开启减少配置步骤。生产环境可通过环境变量显式关闭
- **影响**: 生产环境如需关闭需显式配置 `DIRECTOR_DEBUG_LOG_ENABLED=false`
- **日期**: 2026-06-28

## DEC-004: 日志文件保留策略

- **决策**: 固定保留最近 100 个文件，写入后立即检查并清理
- **替代方案**: 基于文件总大小限制（如 10MB）；定时清理
- **理由**: 文件数量限制最简单可靠，无需计算文件大小或维护定时器。100 个文件在磁盘上占用 <10MB，对开发机无压力
- **影响**: 如果断路器频繁触发，最旧的日志可能在 100 次触发后被清理
- **日期**: 2026-06-28

## DEC-005: recentLlmUsage 字段处理

- **决策**: 首版传空数组 `[]`，预留接口字段，后续迭代补充
- **替代方案**: 从 autoExecution 上下文中查找可用的 LLM 使用记录
- **理由**: 当前 `DirectorAutoExecutionState` 类型中无 LLM 使用记录字段。要获取此数据需扩展运行时上下文传递，超出本任务范围
- **影响**: 首版调试日志缺少 LLM 使用信息，但核心的 autoExecution/circuitBreaker 快照已足够定位大部分问题
- **日期**: 2026-06-28
