# REQ-2047 决策日志

## 决策记录

### D1: 实体粒度 — 单条技法

- **日期**: 2026-07-09
- **决策**: 每条文笔手法 = 一个独立的可开关写作技法（类似 AntiAiRule 粒度）
- **备选**: A) 单条技法 B) 技法集合（小号 StyleProfile）
- **原因**: 用户要求开关控制，粒度越细越精确；AI 筛选时单条更容易做相关性匹配

### D2: 绑定模型 — 三级池子

- **日期**: 2026-07-09
- **决策**: 全局 ∪ 画像 ∪ 小说三级池子
- **备选**: A) 完全独立 B) 绑定画像 C) 只绑定小说
- **原因**: 用户明确要求。部分技法特别适用于特定画像，需要画像级绑定

### D3: 内容存储 — MD 文件

- **日期**: 2026-07-09
- **决策**: 技法内容以 .md 文件存储，frontmatter (name/description/category) + 全文 body
- **备选**: A) 三层字段 B) 两层 MD C) DB 存储
- **原因**: 用户明确要求。方便随时扩充和修改，类 Skill.md 加载机制

### D4: 状态存储 — DB 自动同步

- **日期**: 2026-07-09
- **决策**: .md 文件是内容 SSOT，DB 是状态 SSOT。启动时扫描目录自动注册。
- **备选**: A) 全存文件 B) 全存 DB C) 文件+DB 自动同步
- **原因**: 前端 toggle 需要 DB CRUD；内容变更需要文件可编辑。两层各司其职

### D5: 默认状态 — 不启用

- **日期**: 2026-07-09
- **决策**: 新增技法的全局/画像/小说开关均默认关闭
- **原因**: 防止未审核的技法自动生效

### D6: AI 筛选 — 两阶段加载

- **日期**: 2026-07-09
- **决策**: 筛选时只看 key+name+description，确认后才加载全文。最多选 5 条。
- **备选**: A) 一次性加载全文 B) 两阶段加载
- **原因**: 控制 token 成本。40+ 条概述 ≈ 2000 tokens，5 条全文 ≈ 2500 tokens

### D7: 空池/空结果处理

- **日期**: 2026-07-09
- **决策**: 空池跳过筛选直接改写；筛选结果为空也跳过
- **原因**: 无退化，不影响现有功能

### D8: 注入范围 — MVP 仅润色增强

- **日期**: 2026-07-09
- **决策**: MVP 只在润色增强时注入技法，不接入自动导演
- **备选**: A) 只润色 B) 都注入 C) 先润色后扩展
- **原因**: 润色场景可控便于验证；自动导演注入需考虑和 StyleContract 协调

### D9: 修复 StyleSummary Gap

- **日期**: 2026-07-09
- **决策**: 顺手修复润色增强不接入 StyleContract 的问题
- **原因**: 改动涉及同一文件链，分两次改等于重复劳动；技法筛选也需要感知画像风格

### D10: 画像绑定 UI — 写法引擎侧

- **日期**: 2026-07-09
- **决策**: 画像绑定在写法引擎画像编辑器中管理（与反 AI 规则面板并列）
- **备选**: A) 写法引擎侧 B) 文笔资料库侧 C) 两边都能看
- **原因**: 与反 AI 规则模式一致，用户已熟悉

### D11: 分类体系 — 六类

- **日期**: 2026-07-09
- **决策**: 句法/修辞/对话/叙事/描写/节奏六类
- **原因**: 从 40+ 篇素材中自然归纳，覆盖所有技法
