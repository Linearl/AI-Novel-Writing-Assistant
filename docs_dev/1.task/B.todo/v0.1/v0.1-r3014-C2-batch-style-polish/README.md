## 背景

当前实现中，写正文时会注入风格合约作为 prompt 约束，但只是提示，不能保证执行。写完后会自动检测和修复高风险章节（风险分≥35），但只能单章修复，无法批量处理。

本任务实现批量润色功能，允许用户：
1. 批量检测所有章节的风格问题
2. 批量修复有问题的章节
3. 实时追踪处理进度
4. 查看详细的修复报告

## 核心功能

### 1. 批量检测
- 扫描所有章节的风格问题
- 统计问题分布（高/中/低风险）
- 返回每个章节的风险分

### 2. 批量修复
- 只修复风险分 ≥ 35 的章节
- 按优先级排序（高风险先修复）
- 支持中断和继续

### 3. 进度追踪
- 显示实时进度（已处理/总数）
- 显示当前处理的章节
- 支持暂停和取消

### 4. 结果展示
- 汇总报告（修复/跳过/失败）
- 修复前后的风险分对比
- 详情查看和导出
