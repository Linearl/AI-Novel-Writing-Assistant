# REQ-2012 决策日志

| 日期 | 决策点 | 选择 | 原因 |
| ---- | ---- | ---- | ---- |
| 2026-06-26 | 新增端点 vs 扩展现有端点 | 扩展现有 `POST /repair` 端点 | 用户指导式修复本质上仍是修复，复用 SSE 管线可保持体验一致且实现成本最低 |
| 2026-06-26 | 修复模式选择 | 强制 `heavy_repair` | 用户指导是宏观方向（"加快节奏"），patch 文本替换无法满足；heavy_repair 由 LLM 全文理解后重写，更匹配 |
| 2026-06-26 | userInstruction 注入方式 | 构造合成 ReviewIssue + prompt 上下文注入 | 融入现有修复逻辑无需大改架构；prompt 注入确保 LLM 以用户方向为最高优先级 |
| 2026-06-26 | 输入上限 | 2000 字 | 与现有 description 字段上限一致，足以表达修改方向 |
| 2026-06-26 | 是否持久化 userInstruction | 不持久化 | 仅作为请求参数传递，修复完成后无需保留 |
