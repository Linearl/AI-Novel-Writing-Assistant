# REQ-7057: AI味趋势追踪

## 1. 需求背景

### 1.1 问题描述

用户在长篇小说创作中，需要了解AI生成内容的"AI味"（机械感、套路化）变化趋势，以便调整写作策略、修改提示词或进行针对性优化。当前缺少按章节追踪AI味变化的能力，用户无法量化评估优化效果。

### 1.2 现状分析

**已有功能**：
- AI味检测模块（REQ-7050）— 单章AI味评分
- 章节内容存储 — 章节历史数据
- 前端数据可视化基础 — 图表组件

**缺失功能**：
- ❌ 跨章节AI味评分趋势追踪
- ❌ 趋势变化可视化展示
- ❌ 异常点检测和提示
- ❌ 优化效果对比

### 1.3 目标用户

进行长篇小说创作的用户，特别是：
- 关注AI生成质量的用户
- 进行提示词优化的用户
- 需要量化评估写作效果的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 趋势数据存储

**描述**：存储每章的AI味评分及详细维度数据。

**规则**：
- 每章存储：总体评分、各维度评分（套路化、机械化、情感度、原创性）、检测时间
- 评分随章节生成自动记录
- 支持手动重新检测和覆盖

**验收标准**：
- [ ] 每章AI味数据自动存储
- [ ] 支持各维度独立评分
- [ ] 支持重新检测覆盖
- [ ] 数据持久化到数据库

#### FR-2: 趋势可视化

**描述**：展示AI味评分随章节变化的趋势图。

**规则**：
- 折线图展示总体评分趋势
- 多维度叠加展示
- 支持缩放和筛选
- 标注异常点（评分突变）

**验收标准**：
- [ ] 折线图展示总体趋势
- [ ] 支持维度筛选
- [ ] 支持章节范围筛选
- [ ] 异常点明显标注

#### FR-3: 异常点检测

**描述**：检测AI味评分的异常变化，提示用户关注。

**规则**：
- 检测条件：连续3章评分下降、单章评分突变（>20分）
- 异常类型：AI味上升、AI味异常下降
- 异常提示包含可能原因和建议

**验收标准**：
- [ ] 检测连续下降趋势
- [ ] 检测单章突变
- [ ] 提供可能原因分析
- [ ] 提供优化建议

#### FR-4: 优化效果对比

**描述**：支持对比不同阶段的AI味评分，评估优化效果。

**规则**：
- 支持选择两个章节范围进行对比
- 展示对比结果：平均分、变化趋势、各维度差异
- 支持导出对比报告

**验收标准**：
- [ ] 支持范围选择
- [ ] 对比结果清晰展示
- [ ] 支持导出报告

### 2.2 非功能需求

#### NFR-1: 性能

- 趋势图加载时间<2秒
- 支持1000+章节数据展示

#### NFR-2: 用户体验

- 图表交互流畅
- 支持移动端查看

## 3. 技术约束

### 3.1 架构约束

- AI味评分存储在章节关联表中
- 趋势数据可从评分记录聚合计算
- 可视化使用前端图表库（如Recharts）
- 异常检测使用简单统计方法+AI分析

### 3.2 依赖约束

- 依赖：REQ-7050（AI味检测）
- 被依赖：无

### 3.3 数据约束

- 每章AI味数据需要持久化
- 趋势数据可实时聚合计算

## 4. 验收标准

### 4.1 功能验收

- [ ] 趋势数据存储正常
- [ ] 趋势可视化正常展示
- [ ] 异常点检测正常工作
- [ ] 优化效果对比正常工作

### 4.2 性能验收

- [ ] 趋势图加载<2秒
- [ ] 支持大量章节数据

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 可视化效果人工验证

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 数据量大 | 加载慢 | 低 | 分页加载+数据聚合 |
| 图表兼容性 | 移动端显示异常 | 低 | 响应式设计+兼容测试 |
| 异常误报 | 用户困惑 | 中 | 合理阈值+人工复核 |
