# REQ-7049: 自动重新生成

## 1. 需求背景

### 1.1 问题描述

当AI生成的章节质量不合格时（字数不足、结构异常、人物不一致等），用户需要手动触发重新生成。在长篇小说创作（160章+）中，这种手动干预严重影响创作效率，且用户可能忽略质量问题继续推进。

### 1.2 现状分析

**已有功能**：
- Auto-Director 章节审核流程 — 事后审核，不阻断
- 质量检查器（REQ-7048）— 检测质量问题，但无自动修复机制
- 人工审核回调 — 依赖用户手动操作

**缺失功能**：
- ❌ 质量不合格时自动触发重新生成
- ❌ 重试次数控制和退避策略
- ❌ 重新生成前的参数调整（提高温度、更换prompt等）
- ❌ 重试过程可视化

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 长篇网文（160章+）的创作者
- 对自动创作流畅性要求高的用户
- 不希望频繁手动干预的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 自动重试触发

**描述**：当质量检查器报告章节不合格时，系统自动触发重新生成。

**规则**：
- 质量检查不通过时自动触发（可配置为手动确认）
- 最多重试3次（可配置）
- 每次重试可调整生成参数

**验收标准**：
- [ ] 质量检查不通过时自动触发重新生成
- [ ] 重试次数达到上限后停止并通知用户
- [ ] 重试配置可通过系统参数调整

#### FR-2: 重试参数调整

**描述**：每次重试时自动调整生成参数，提高重新生成成功率。

**参数调整策略**：
- 第1次重试：温度 +0.1（更随机）
- 第2次重试：温度 +0.2（更发散）
- 第3次重试：更换prompt变体（如果有备选prompt）

**验收标准**：
- [ ] 每次重试参数自动调整
- [ ] 参数调整策略可配置
- [ ] 参数调整有日志记录

#### FR-3: 重试过程追踪

**描述**：记录每次重试的详细信息，供用户查看和分析。

**记录内容**：
- 重试次数和时间
- 每次重试的质量评分
- 参数调整记录
- 最终结果（成功/失败）

**验收标准**：
- [ ] 每次重试都有详细记录
- [ ] 支持通过API查询重试历史
- [ ] 支持前端展示重试过程

#### FR-4: 重试通知

**描述**：重试过程中的关键事件通知用户。

**通知时机**：
- 开始重试时
- 重试成功时
- 重试失败（达到上限）时

**验收标准**：
- [ ] 关键事件有通知
- [ ] 通知格式清晰
- [ ] 通知级别合理（info/warn/error）

### 2.2 非功能需求

#### NFR-1: 性能

- 重试不应显著增加章节完成时间
- 重试期间不阻塞其他章节生成
- 支持并发重试（多个章节同时重试）

#### NFR-2: 可靠性

- 重试失败后保留最后一次生成结果
- 不会因为重试机制导致数据丢失
- 重试过程可中断

#### NFR-3: 兼容性

- 不影响现有章节生成流程
- 与质量检查器（REQ-7048）紧密集成
- 向后兼容现有配置

## 3. 技术约束

### 3.1 架构约束

- 重试逻辑作为章节生成流程的后处理环节
- 依赖质量检查器（REQ-7048）的检查结果
- 通过事件驱动触发重试

### 3.2 依赖约束

- 依赖：REQ-7048（实时质量检查器）
- 被依赖：无

### 3.3 数据约束

- 重试历史存储在 `ChapterRegenerationLog` 表
- 重试配置存储在系统配置表

## 4. 验收标准

### 4.1 功能验收

- [ ] 质量不合格自动触发重试
- [ ] 重试参数自动调整
- [ ] 重试历史可查询
- [ ] 重试上限控制正常

### 4.2 性能验收

- [ ] 重试不阻塞其他章节生成
- [ ] 并发重试无性能问题
- [ ] 重试过程可中断

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖（0次重试、1次成功、3次失败）

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 无限重试 | 资源浪费 | 低 | 严格限制重试次数上限 |
| 重试质量更差 | 生成结果劣化 | 中 | 参数调整策略有限，最终保留最佳结果 |
| 并发冲突 | 章节覆盖 | 低 | 事务性更新 + 版本控制 |
| 用户无感知 | 重试过程不可见 | 中 | 实时通知 + 过程可视化 |

## 6. 工作量评估

- **开发时间**：0.5天
- **测试时间**：0.5天
- **总计**：1天

**P1** - 高优先级

**理由**：
- 是v0.2自动化质量保证的核心环节
- 与REQ-7048配合形成完整的质量闭环
- 直接提升长篇创作效率
