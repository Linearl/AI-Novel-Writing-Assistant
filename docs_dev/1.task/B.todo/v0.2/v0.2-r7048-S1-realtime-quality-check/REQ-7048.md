# REQ-7048: 实时质量检查器

## 1. 需求背景

### 1.1 问题描述

当前章节生成后缺乏自动质量检查机制，用户只能手动通读整章才能发现问题。在长篇小说创作（160章+）中，这导致质量不一致问题难以及时发现，严重影响最终作品质量。

### 1.2 现状分析

**已有功能**：
- Auto-Director 章节审核流程 — 事后审核，不阻断主链
- 人工审核回调 — 依赖用户手动操作
- 生成内容保存 — 无质量评估环节

**缺失功能**：
- ❌ 章节生成后的自动化质量检查
- ❌ 多维度质量评估（字数、结构、人物、情节、AI味、一致性）
- ❌ 质量评分与阈值判定
- ❌ 质量报告自动生成

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 长篇网文（160章+）的创作者
- 对作品质量有较高要求的用户
- 希望实时了解章节质量的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 字数检查

**描述**：检查章节字数是否在合理范围内。

**规则**：
- 默认范围：3000-8000字（可配置）
- 统计实际有效字数（排除空行、标点）
- 输出字数统计结果

**验收标准**：
- [ ] 正确统计章节字数
- [ ] 字数低于下限时标记为警告
- [ ] 字数超过上限时标记为警告
- [ ] 字数配置可通过系统参数调整

#### FR-2: 结构检查

**描述**：检查章节是否包含必要的结构元素（开头、发展、结尾）。

**规则**：
- 检测段落数量是否充足（至少5段）
- 检测对话是否分布均匀
- 检测是否有完整的情节推进
- 识别过度单调的段落结构

**验收标准**：
- [ ] 正确检测段落数量
- [ ] 正确识别对话分布
- [ ] 标记结构异常（如全篇对话或全篇叙述）
- [ ] 输出结构分析结果

#### FR-3: 人物一致性检查

**描述**：检查章节中人物名称、性格、行为是否一致。

**规则**：
- 提取章节中出现的所有人物名称
- 与人物设定库交叉验证
- 检测人物名称拼写错误或变体
- 标记可能的人物行为不一致

**验收标准**：
- [ ] 正确提取人物名称
- [ ] 与人物设定库匹配
- [ ] 标记未定义的人物
- [ ] 输出人物出现统计

#### FR-4: 情节连贯性检查

**描述**：检查章节内容与前后章节的情节连贯性。

**规则**：
- 提取章节核心情节摘要
- 与前一章结尾对比，检查衔接
- 标记情节跳跃或矛盾
- 检测时间线逻辑

**验收标准**：
- [ ] 正确提取章节情节摘要
- [ ] 与前章衔接度评估
- [ ] 标记明显的情节跳跃
- [ ] 输出情节连贯性评分

#### FR-5: AI味检测集成

**描述**：调用AI味检测模块（REQ-7050）评估内容的AI生成痕迹。

**规则**：
- 集成REQ-7050的检测结果
- AI味分数超过阈值时标记为警告
- 输出AI味评分和具体问题点

**验收标准**：
- [ ] 正确集成AI味检测结果
- [ ] 超阈值时生成警告
- [ ] 输出详细的AI味报告

#### FR-6: 质量报告生成

**描述**：汇总所有检查结果，生成结构化质量报告。

**规则**：
- 输出JSON格式质量报告
- 包含各维度评分和总评分
- 包含具体问题清单和建议
- 报告存储到数据库供查询

**验收标准**：
- [ ] 生成完整的JSON质量报告
- [ ] 报告包含所有检查维度
- [ ] 报告存储到数据库
- [ ] 支持通过API查询历史报告

### 2.2 非功能需求

#### NFR-1: 性能

- 单章质量检查耗时<5秒
- 不阻塞章节生成主流程
- 支持异步检查模式

#### NFR-2: 可扩展性

- 检查维度可配置
- 阈值可调整
- 支持后续新增检查维度

#### NFR-3: 兼容性

- 不影响现有章节生成流程
- 与Auto-Director审核流程协同
- 向后兼容现有数据

## 3. 技术约束

### 3.1 架构约束

- 质量检查作为独立服务模块实现
- 异步执行，不阻塞生成流程
- 复用现有的人物、章节数据模型

### 3.2 依赖约束

- 依赖：REQ-7050（AI味检测）
- 被依赖：REQ-7049（自动重新生成）、REQ-7051（一致性监控）

### 3.3 数据约束

- 质量报告存储在新增的 `ChapterQualityReport` 表
- 检查配置存储在系统配置表

## 4. 验收标准

### 4.1 功能验收

- [ ] 所有检查维度正常工作
- [ ] 质量报告格式正确
- [ ] 配置项可调整
- [ ] 与AI味检测集成正常

### 4.2 性能验收

- [ ] 单章检查耗时<5秒
- [ ] 异步模式不阻塞生成流程
- [ ] 并发检查无性能问题

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖（空章节、超长章节、特殊字符）

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 检查耗时过长 | 阻塞生成流程 | 中 | 异步执行 + 超时控制 |
| 误判质量合格 | 质量问题遗漏 | 中 | 多维度交叉验证 + 人工复核 |
| AI味检测不准 | 误报/漏报 | 中 | 可配置阈值 + 持续调优 |

## 6. 工作量评估

- **开发时间**：0.5天
- **测试时间**：0.5天
- **总计**：1天

## 7. 优先级

**P1** - 高优先级

**理由**：
- 是v0.2质量保证体系的基础
- 为后续自动重新生成（REQ-7049）提供判断依据
- 直接影响用户体验和作品质量
