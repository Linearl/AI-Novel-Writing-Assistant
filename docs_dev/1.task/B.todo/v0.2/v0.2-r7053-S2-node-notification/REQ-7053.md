# REQ-7053: 关键节点通知

## 1. 需求背景

### 1.1 问题描述

AI创作过程中，任务完成后需要用户确认、质量检查或人工干预时，用户往往不在当前页面，导致任务长时间等待。系统缺乏主动通知机制，用户需要频繁刷新页面查看任务状态，体验不佳。

### 1.2 现状分析

**已有功能**：
- 前端轮询状态更新（TanStack Query）
- 后端事件总线（EventBus）
- 任务状态API

**缺失功能**：
- ❌ 浏览器原生通知（Web Notifications API）
- ❌ 关键节点的主动推送
- ❌ 通知权限管理
- ❌ 通知内容模板

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 创作过程中切换到其他应用的用户
- 等待AI生成结果的用户
- 需要人工审核的场景

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 浏览器通知

**描述**：当关键节点发生时，通过浏览器原生通知推送消息给用户。

**规则**：
- 使用Web Notifications API
- 需要用户授权通知权限
- 通知包含标题、内容、图标
- 点击通知跳转到对应页面

**验收标准**：
- [ ] 首次使用时请求通知权限
- [ ] 授权后能发送通知
- [ ] 未授权时静默降级，不报错
- [ ] 点击通知跳转到任务详情页

#### FR-2: 关键节点配置

**描述**：定义哪些事件触发通知，支持按事件类型配置。

**规则**：
- 支持的通知事件：任务完成、任务失败、需要人工审核、质量检查结果
- 支持按事件类型开启/关闭通知
- 支持全局开关

**验收标准**：
- [ ] 支持配置各事件的通知开关
- [ ] 支持全局通知开关
- [ ] 配置持久化到用户设置
- [ ] 默认配置合理（完成、失败开启，其他可选）

#### FR-3: 通知内容模板

**描述**：通知内容使用模板化消息，包含关键信息。

**规则**：
- 任务完成："[小说名] 第X章生成完成"
- 任务失败："[小说名] 第X章生成失败：[原因]"
- 需要审核："[小说名] 第X章等待审核"
- 质量检查："[小说名] 第X章质量评分：[分数]"

**验收标准**：
- [ ] 通知标题清晰表达事件
- [ ] 通知内容包含必要信息
- [ ] 通知文案符合UI文案规则（用户视角）

### 2.2 非功能需求

#### NFR-1: 可靠性

- 通知发送失败不影响任务执行
- 通知权限被拒绝时静默降级

#### NFR-2: 兼容性

- 支持Chrome、Firefox、Edge
- 不支持的浏览器静默降级

#### NFR-3: 用户体验

- 通知频率不过高（避免通知疲劳）
- 支持免打扰模式

## 3. 技术约束

### 3.1 架构约束

- 使用浏览器原生Web Notifications API
- 前端实现通知管理器
- 通知配置存储在用户设置中
- 后端任务状态变更触发通知事件

### 3.2 依赖约束

- 依赖：REQ-7052（后台任务管理）
- 被依赖：无

### 3.3 数据约束

- 通知配置存储在用户偏好设置中
- 通知历史不需要持久化

## 4. 验收标准

### 4.1 功能验收

- [ ] 浏览器通知正常工作
- [ ] 通知权限管理正常
- [ ] 关键节点配置正常
- [ ] 通知内容模板正常
- [ ] 全局开关正常

### 4.2 性能验收

- [ ] 通知发送不影响任务执行
- [ ] 通知延迟<1秒

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 浏览器兼容性测试

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 用户拒绝通知权限 | 功能不可用 | 中 | 提供引导和降级方案 |
| 通知频率过高 | 用户体验下降 | 中 | 合并通知+免打扰模式 |
| 浏览器兼容性 | 部分用户不可用 | 低 | 静默降级+功能检测 |

- **总计**：0.5天
