# REQ-7063: Tension Curve 冲突曲线

## 1. 需求背景

### 1.1 问题描述

当前章节冲突强度（conflictLevel）和揭示程度（revealLevel）为静态数值，用户无法直观看到全书的冲突曲线走势，也无法通过交互方式调整节奏。缺乏可视化的冲突曲线导致：
- 难以发现节奏问题（如平坦高原、高潮过迟）
- 无法按卷/按章批量调整冲突等级
- 缺乏与节拍表、角色弧光的联动视图

### 1.2 现状分析

**已有功能**：
- 章节级 `conflictLevel` 和 `revealLevel` 数据字段（Prisma schema）
- 基础节奏曲线可视化（REQ-2030 已实现 recharts 折线图）
- 冲突等级作为生成硬约束（REQ-7060 FR-5 后端部分）

**缺失功能**：
- ❌ 交互式曲线编辑（拖拽调整冲突强度）
- ❌ React Flow + d3-shape 高级可视化
- ❌ 节奏问题自动检测（平坦高原、高潮过迟等）
- ❌ 节拍表上下文联动
- ❌ 章节详情侧边栏
- ❌ 卷上下文栏

### 1.3 目标用户

- 追求精细节奏控制的小说作者
- 使用自动导演功能的用户（冲突曲线作为生成约束）
- 需要宏观把控全书节奏的创作者

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 交互式冲突强度可视化

**描述**：以 0-100 数值范围可视化全书冲突强度曲线，支持按卷/按章缩放。

**可视化要素**：
- X 轴：章节编号（支持按卷分组）
- Y 轴：冲突强度（0-100）
- 曲线：d3-shape 生成的平滑曲线
- 节点：可交互的章节节点（React Flow）

**验收标准**：
- [ ] 曲线覆盖全书所有章节
- [ ] X 轴支持按卷分组显示
- [ ] Y 轴范围 0-100
- [ ] 曲线平滑（d3-shape monotoneX 插值）
- [ ] 支持缩放和平移

**上游参考**：
- `TensionCurveFlowCanvas.tsx` — React Flow 画布
- `curveCoordinates.ts` — 坐标计算
- `tensionCurveTypes.ts` — 类型定义

#### FR-2: 拖拽编辑

**描述**：支持通过拖拽节点直接编辑章节的冲突强度值。

**交互规则**：
- 拖拽节点上下移动调整 0-100 值
- 拖拽实时更新曲线形状
- 松开后保存到后端
- 支持撤销操作

**验收标准**：
- [ ] 节点可拖拽（上下移动）
- [ ] 拖拽实时反馈（曲线跟随变化）
- [ ] 松开后自动保存
- [ ] 支持 Ctrl+Z 撤销

**上游参考**：
- `TensionCurveNodes.tsx` — 自定义可拖拽节点
- `TensionCurveEditDialog.tsx` — 精确数值编辑

#### FR-3: 节奏问题自动检测

**描述**：自动检测冲突曲线中的节奏问题并高亮显示。

**检测类型**：
| 问题码 | 描述 | 检测规则 |
|--------|------|----------|
| `flatPlateau` | 平坦高原 | 连续 5+ 章节冲突值波动 < 10 |
| `lateClimax` | 高潮过迟 | 书末 20% 才出现最高峰 |
| `earlyPeak` | 高潮过早 | 前 30% 出现全书最高峰 |
| `noTension` | 无冲突 | 连续 3+ 章节冲突值 < 20 |
| `excessiveTension` | 冲突过度 | 连续 5+ 章节冲突值 > 90 |

**验收标准**：
- [ ] 5 种问题码全部实现
- [ ] 问题区域在曲线上高亮标注
- [ ] 问题描述和修复建议可查看
- [ ] 支持手动忽略误报

**上游参考**：
- `tensionCurveAnalysis.ts` — 节奏问题检测逻辑

#### FR-4: 节拍表上下文联动

**描述**：冲突曲线与节拍表（Beat Sheet）联动，显示每个章节对应的节拍信息。

**验收标准**：
- [ ] 鼠标悬停显示节拍信息
- [ ] 节拍上下文条（BeatContextStrip）显示
- [ ] 支持从曲线跳转到节拍详情

**上游参考**：
- `TensionCurveBeatContextStrip.tsx` — 节拍上下文条

#### FR-5: 章节详情侧边栏

**描述**：点击曲线节点后打开侧边栏，显示章节详细信息和冲突强度调整。

**验收标准**：
- [ ] 点击节点打开侧边栏
- [ ] 侧边栏显示章节标题、摘要、冲突值
- [ ] 支持在侧边栏中精确调整数值
- [ ] 支持展开/折叠侧边栏

**上游参考**：
- `TensionCurveChapterDetailSidebar.tsx` — 侧边栏组件

#### FR-6: 卷上下文栏

**描述**：显示当前卷的上下文信息（卷名、章节数、卷平均冲突值）。

**验收标准**：
- [ ] 显示当前卷的基本信息
- [ ] 显示卷平均冲突值
- [ ] 支持切换卷

**上游参考**：
- `TensionCurveVolumeContextBar.tsx` — 卷上下文栏

#### FR-7: 主面板容器

**描述**：整合所有子组件的主面板，作为冲突曲线功能的入口。

**验收标准**：
- [ ] 整合画布、侧边栏、上下文栏
- [ ] 响应式布局
- [ ] 支持全屏模式

**上游参考**：
- `TensionCurvePanel.tsx` — 主面板组件

### 2.2 非功能需求

#### NFR-1: 性能

- 160 章节（16 卷 x 10 章）场景下曲线渲染 < 1s
- 拖拽交互帧率 >= 60fps
- 节奏问题检测 < 100ms

#### NFR-2: 可访问性

- 曲线支持键盘导航
- 节奏问题有文字描述（不依赖颜色区分）

#### NFR-3: 响应式

- 最小支持 1024px 宽度
- 侧边栏在窄屏下自动折叠

## 3. 技术约束

### 3.1 架构约束

- 前端组件基于 React Flow + d3-shape
- 曲线渲染使用 SVG（非 Canvas）
- 数据通过 API 从后端获取

### 3.2 依赖约束

- 需要 `react-flow-renderer` 或 `@xyflow/react`
- 需要 `d3-shape`（曲线插值）
- 与 REQ-7060 FR-5 共享冲突等级数据模型

### 3.3 数据约束

- 每章一个冲突值（0-100 整数）
- 数据存储在 NovelChapter 表的 `conflictLevel` 字段
- 前端缓存减少 API 调用

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| React Flow 兼容性 | 渲染异常 | 低 | 使用稳定版本 |
| d3-shape 曲线不平滑 | 视觉效果差 | 低 | 调整插值参数 |
| 大量章节性能 | 渲染卡顿 | 中 | 虚拟滚动 + 按卷加载 |
| 拖拽精度不足 | 用户体验差 | 中 | 支持精确数值输入 |
