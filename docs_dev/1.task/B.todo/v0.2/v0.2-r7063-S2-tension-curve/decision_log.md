# REQ-7063: Tension Curve 冲突曲线 — 决策日志

## 决策1: 可视化技术选型

**决策日期**：2026-07-14

**背景**：冲突曲线的可视化应该使用什么技术？

**选项**：
- **方案A**: React Flow + d3-shape（上游实现）
- **方案B**: 纯 recharts（REQ-2030 已有基础）
- **方案C**: Canvas 2D 自绘（最高灵活度）

**决策**：✅ 方案A

**理由**：
1. 上游已用 React Flow + d3-shape 实现完整功能，可直接移植
2. React Flow 提供节点交互（拖拽、选择、连线）能力
3. d3-shape 提供专业级曲线插值（monotoneX）
4. recharts（方案B）不支持交互式节点编辑

**影响**：
- 新增 `@xyflow/react` 和 `d3-shape` 依赖
- 需要移植上游全部 10 个文件
- 与 REQ-2030 的 recharts 曲线可能共存（不同用途）

## 决策2: 曲线渲染方式

**决策日期**：2026-07-14

**背景**：曲线应该用 SVG 还是 Canvas 渲染？

**选项**：
- **方案A**: SVG（React Flow 默认）
- **方案B**: Canvas 2D（性能更好）
- **方案C**: WebGL（最高性能）

**决策**：✅ 方案A

**理由**：
1. React Flow 默认使用 SVG，无需额外配置
2. SVG 支持 CSS 样式和 DOM 事件（鼠标悬停、点击）
3. 160 章节规模下 SVG 性能足够
4. 与上游实现一致

**影响**：
- 160 章节以内性能无问题
- 超大规模（500+）时可能需要虚拟化
- SVG 元素可直接添加 CSS 动画

**决策日期**：2026-07-14

**背景**：REQ-7060 FR-5（冲突等级曲线）与本任务包功能重叠，如何处理？

4. 职责清晰，无交叉

**影响**：
- REQ-7060 FR-5 仅实现：数据模型 + 后端约束注入
- REQ-7063 实现：完整前端 UI + 交互 + 分析

## 决策4: 章节节点交互方式

**决策日期**：2026-07-14

**背景**：章节节点的冲突值调整应该用拖拽还是滑块？

**选项**：
- **方案A**: 拖拽节点（直观但精度低）
- **方案B**: 滑块编辑（精确但不直观）
- **方案C**: 拖拽 + 双击精确编辑（平衡）

**决策**：✅ 方案C

**理由**：
1. 拖拽提供直观的视觉反馈
2. 双击弹出编辑对话框支持精确数值输入
3. 两种方式互补，满足不同场景
4. 与上游实现一致

**影响**：
- 实现拖拽交互（TensionCurveNodes）
- 实现编辑对话框（TensionCurveEditDialog）
- 拖拽松开后自动保存

## 决策5: 节奏问题检测阈值

**决策日期**：2026-07-14

**背景**：节奏问题检测的阈值如何设定？

**选项**：
- **方案A**: 使用上游固定阈值
- **方案B**: 用户可配置阈值
- **方案C**: AI 自适应阈值

**决策**：✅ 方案A

**理由**：
1. 上游阈值经过实践验证
2. 实现阶段先用固定阈值，降低复杂度
3. 后续迭代可加入配置化
4. AI 自适应阈值需要大量数据训练，暂不适用

**影响**：
- 平坦高原：连续 5 章波动 < 10
- 高潮过迟：末 20% 出现峰值
- 高潮过早：前 30% 出现峰值
- 无冲突：连续 3 章 < 20
- 冲突过度：连续 5 章 > 90
