# REQ-7058: 环境变量启动校验

## 1. 需求背景

### 1.1 问题描述

系统启动时，如果缺少必要的环境变量（如API Key、数据库连接字符串、模型配置等），服务可能启动失败或运行时才报错，导致排查困难。用户需要快速了解启动失败的原因，并得到明确的修复指引。

### 1.2 现状分析

**已有功能**：
- 环境变量读取（server/.env）
- 配置文件加载
- 基本的错误处理

**缺失功能**：
- ❌ 启动时环境变量完整性校验
- ❌ 缺失变量的快速失败和提示
- ❌ 变量格式/值的合法性校验
- ❌ 校验结果的友好展示

### 1.3 目标用户

所有部署和运行系统的开发者和运维人员，特别是：
- 首次部署的用户
- 升级后配置变更的用户
- 多环境部署的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 必要变量校验

**描述**：启动时校验所有必要环境变量是否存在。

**规则**：
- 校验变量：API_KEY、DATABASE_URL、LLM_MODEL等核心配置
- 变量缺失时立即停止启动
- 输出缺失变量列表和修复指引
- 支持必填和可选变量分类

**验收标准**：
- [ ] 必要变量缺失时服务不启动
- [ ] 输出清晰的缺失变量列表
- [ ] 提供修复指引（示例值）
- [ ] 必填和可选变量正确区分

#### FR-2: 变量格式校验

**描述**：校验环境变量值的格式和合法性。

**规则**：
- API_KEY：非空字符串，最小长度
- DATABASE_URL：有效的数据库连接字符串格式
- 端口号：有效范围（1-65535）
- 布尔值：true/false字符串

**验收标准**：
- [ ] 格式校验正确工作
- [ ] 无效值时输出具体错误信息
- [ ] 不影响有效配置的启动

#### FR-3: 启动报告

**描述**：启动时输出配置校验报告。

**规则**：
- 报告格式：分组展示（必填/可选/已配置/未配置）
- 颜色标记：绿色（已配置）、黄色（可选未配置）、红色（必填缺失）
- 报告在服务启动日志中输出

**验收标准**：
- [ ] 报告格式清晰易读
- [ ] 颜色标记正确
- [ ] 报告包含所有配置项状态

#### FR-4: 快速失败

**描述**：校验失败时快速失败，输出友好的错误信息。

**规则**：
- 错误信息包含：缺失/无效的变量名、原因、修复建议
- 错误信息格式化输出
- 退出码非0，便于脚本检测

**验收标准**：
- [ ] 错误信息友好易懂
- [ ] 包含修复建议
- [ ] 退出码正确

### 2.2 非功能需求

#### NFR-1: 性能

- 校验过程<1秒
- 不影响正常启动速度

#### NFR-2: 可维护性

- 新增变量只需添加校验规则
- 校验规则集中管理

#### NFR-3: 兼容性

- 向后兼容现有配置
- 不影响无配置文件的启动方式

## 3. 技术约束

### 3.1 架构约束

- 校验逻辑在server启动入口处执行
- 校验规则配置化（变量名、是否必填、格式、默认值）
- 校验失败阻止服务启动

### 3.2 依赖约束

- 依赖：无
- 被依赖：无

### 3.3 数据约束

- 环境变量从process.env读取
- 校验规则不持久化

### 4.1 功能验收

- [ ] 变量格式校验正常
- [ ] 启动报告正常输出
- [ ] 快速失败正常工作

### 4.2 性能验收

- [ ] 校验<1秒
- [ ] 不影响正常启动

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 误报 | 正常配置无法启动 | 低 | 校验规则准确+可配置 |
| 向后兼容 | 现有部署受影响 | 低 | 渐进式校验+警告模式 |
| 变量名变更 | 校验失效 | 低 | 集中管理+文档 |
