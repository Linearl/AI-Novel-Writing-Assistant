# REQ-7058: 环境变量启动校验 — 决策日志

## 决策1: 校验执行时机

**决策日期**：2026-07-11

**背景**：环境变量校验应该在什么时候执行？

**选项**：
- **方案A**: 服务启动时同步执行
- **方案B**: 服务启动时异步执行
- **方案C**: 首次API调用时执行

**决策**：✅ 方案A

**理由**：
1. 启动时同步执行可以快速失败
2. 不会在服务启动后才发现配置错误
3. 实现简单，逻辑清晰
4. 校验过程<1秒，不影响启动速度

**影响**：
- 在app.ts启动入口处调用校验
- 校验失败时进程退出（exit code非0）
- 校验通过后继续正常启动

---

## 决策2: 校验失败处理方式

**决策日期**：2026-07-11

**背景**：校验失败时应该如何处理？

**选项**：
- **方案A**: 阻止启动（exit code非0）
- **方案B**: 输出警告但继续启动
- **方案C**: 根据变量重要程度决定（必填阻止，可选警告）

**决策**：✅ 方案A

**理由**：
1. 快速失败，不浪费资源
2. 明确的错误信息帮助用户快速修复
3. 可选变量使用默认值，不影响启动
4. 与CI/CD脚本集成友好（exit code检测）

**影响**：
- 必填变量缺失时阻止启动
- 可选变量缺失时设置默认值并警告
- 退出码非0，便于脚本检测

## 决策3: 变量定义管理方式

**决策日期**：2026-07-11

**背景**：环境变量定义如何管理？

**选项**：
- **方案A**: 集中在envValidator.ts中定义
- **方案B**: 分散在各模块中，启动时汇总
- **方案C**: 使用JSON/YAML配置文件

**决策**：✅ 方案A

**理由**：
1. 集中管理便于维护和查阅
2. 所有变量定义在一个文件中，一目了然
3. 不需要额外的配置文件解析
4. TypeScript类型安全保障

**影响**：
- envValidator.ts中包含所有变量定义
- 新增变量只需在此文件中添加
- 文件可能较长（但变量数量有限）

---

## 决策4: 敏感信息显示方式

**决策日期**：2026-07-11

**背景**：API Key等敏感信息在报告中如何显示？

**选项**：
- **方案A**: 完全隐藏（只显示"已设置"）
- **方案B**: 部分掩码（显示前4位+***）
- **方案C**: 完全显示（调试模式）

**决策**：✅ 方案B

**理由**：
1. 部分显示帮助用户确认是否是正确的Key
2. 掩码防止日志泄露完整密钥
3. 平衡了可读性和安全性
4. 调试模式可以完全显示（可选功能）

**影响**：
- API_KEY等敏感变量显示前4位+***
- 非敏感变量正常显示
- 日志中也使用掩码格式

---

## 决策5: 报告输出格式

**决策日期**：2026-07-11

**背景**：校验报告应该如何输出？

**选项**：
- **方案A**: 彩色控制台输出（ANSI颜色码）
- **方案B**: 纯文本输出
- **方案C**: JSON格式输出

**决策**：✅ 方案A

**理由**：
1. 彩色输出直观，一眼看出问题
2. 开发环境通常支持ANSI颜色
3. 生产环境可以降级为纯文本（检测终端支持）
4. 格式清晰，信息层次分明

**影响**：
- 使用ANSI颜色码（绿色=正常，黄色=警告，红色=错误）
- Windows终端可能需要特殊处理
- 支持颜色降级（非TTY输出时）

| 失败处理 | 方案A：阻止启动 | 明确错误，不浪费资源 |
| 变量管理 | 方案A：集中定义 | 维护方便 |
| 敏感信息 | 方案B：部分掩码 | 安全性+可读性 |
