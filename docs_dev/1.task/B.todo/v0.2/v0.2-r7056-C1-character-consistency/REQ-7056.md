# REQ-7056: 跨章节人物一致性

## 1. 需求背景

### 1.1 问题描述

在长篇小说创作中（160章+），AI生成的新章节可能与已有人物设定产生矛盾。例如：角色外貌描述前后不一致、性格特征偏移、能力设定冲突、关系变化无铺垫等。这些矛盾严重影响小说质量和读者体验，人工检查160+章节的人设一致性几乎不可能。

### 1.2 现状分析

**已有功能**：
- 角色管理模块 — 角色基本信息维护
- 角色描述（character description）— 文字描述
- 前置章节上下文注入 — 生成时传入前面章节内容

**缺失功能**：
- ❌ 角色状态结构化跟踪（CharacterState表）
- ❌ 跨章节矛盾检测
- ❌ 矛盾报告生成
- ❌ 人物一致性评分

### 1.3 目标用户

进行长篇小说创作的用户，特别是：
- 创作160章+长篇网文的用户
- 多角色复杂关系网络的小说
- 需要严格人设一致性的作品

## 2. 需求定义

### 2.1 功能需求

#### FR-1: CharacterState表

**描述**：维护每个角色的结构化状态，按章节更新。

**规则**：
- 每个角色维护：外貌特征、性格特征、能力/技能、人际关系、当前状态、位置
- 每生成一章后更新CharacterState
- 保留历史状态，支持按章节查看
- 状态变更记录原因（哪一章引入的变更）

**验收标准**：
- [ ] CharacterState表包含外貌、性格、能力、关系、状态、位置字段
- [ ] 每章生成后自动更新CharacterState
- [ ] 支持按章节查看历史状态
- [ ] 状态变更记录来源章节

#### FR-2: 矛盾检测

**描述**：在生成新章节时，检测人物特征与已有状态的矛盾。

**规则**：
- 检测维度：外貌矛盾、性格矛盾、能力矛盾、关系矛盾、位置矛盾
- 矛盾严重度分级：硬矛盾（直接冲突）、软矛盾（模糊不一致）
- 检测时机：章节生成后、发布前
- 检测方法：AI结构化分析+规则匹配

**验收标准**：
- [ ] 支持5个维度的矛盾检测
- [ ] 矛盾分为硬矛盾和软矛盾
- [ ] 检测结果包含矛盾描述和位置
- [ ] 检测准确率>80%（人工复核验证）

#### FR-3: 矛盾报告

**描述**：生成矛盾检测报告，展示矛盾详情和修复建议。

**规则**：
- 报告包含：矛盾列表、严重度、位置、修复建议
- 支持按角色筛选
- 支持按严重度排序
- 支持导出为Markdown

**验收标准**：
- [ ] 报告包含所有必要信息
- [ ] 支持筛选和排序
- [ ] 导出格式正确

#### FR-4: 一致性评分

**描述**：为每个章节生成人物一致性评分（0-100）。

**规则**：
- 评分维度：外貌一致性、性格一致性、能力一致性、关系一致性
- 综合评分 = 各维度加权平均
- 评分结果可视化展示
- 低于阈值时触发警告

**验收标准**：
- [ ] 评分范围0-100
- [ ] 各维度评分独立计算
- [ ] 可视化展示评分趋势
- [ ] 低于60分时触发警告

### 2.2 非功能需求

#### NFR-1: 性能

- 矛盾检测在章节生成后异步执行
- 不阻塞章节生成流程
- 检测时间<30秒/章

#### NFR-2: 准确性

- 硬矛盾检测准确率>90%
- 软矛盾检测准确率>70%
- 误报率<20%

#### NFR-3: 可扩展性

- 支持新增矛盾检测维度
- 支持自定义矛盾规则

## 3. 技术约束

### 3.1 架构约束

- CharacterState存储在数据库中（新增表或JSON字段）
- 矛盾检测使用LLM结构化输出
- 检测结果存储在专门的矛盾记录表
- 作为后台任务异步执行

### 3.2 依赖约束

- 依赖：REQ-7052（后台任务管理）
- 被依赖：无

### 3.3 数据约束

- CharacterState需要持久化
- 矛盾记录需要保留历史
- 章节内容作为检测输入

## 4. 验收标准

### 4.1 功能验收

- [ ] CharacterState表正常维护
- [ ] 矛盾检测正常工作
- [ ] 矛盾报告正常生成
- [ ] 一致性评分正常计算

### 4.2 性能验收

- [ ] 检测不阻塞章节生成
- [ ] 检测时间<30秒/章

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 人工验证矛盾检测准确率

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 检测不准确 | 误报或漏报 | 中 | 多维度检测+人工复核 |
| 性能影响 | 章节生成变慢 | 低 | 异步执行+后台任务 |
| 数据量大 | 存储成本 | 中 | JSON字段压缩+历史清理 |

## 6. 工作量评估

- **开发时间**：1天
- **测试时间**：0.5天
- **总计**：1.5天

**理由**：
- 直接影响小说质量
- 160章+长篇的刚需功能
