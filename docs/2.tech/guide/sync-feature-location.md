# 同步功能位置指南

## "拉取世界库更新"按钮位置

### 显示条件
**只有当小说世界是从世界库导入时，才会显示同步管理部分。**

在代码中（NovelWorldManagerCard.tsx:429）：
```tsx
{novelWorld?.sourceWorldId ? (
  <div id="novel-world-sync" className="rounded-lg border border-border/70 bg-background p-4">
    // 同步管理内容
  </div>
) : null}
```

### 位置
在小说项目设置页面的"本书世界"卡片中，向下滚动找到"同步管理"部分。

### 显示流程

#### 情况1：有差异时
如果你的世界库和小说世界有差异，会显示：
```
同步管理
├── 1. 差异检查 → 检查完成（N处差异）
├── 2. 选择分区 → N个分区
├── 3. 手动同步 → 独立副本
└── 差异列表（带checkbox）
    ├── [√] 概要
    ├── [√] 规则
    ├── [√] 势力
    └── [√] 地点

按钮：
[拉取世界库更新] [推送本书修改] [关闭同步]
```

#### 情况2：无差异但未开启同步
如果当前是"独立副本"状态，会显示：
```
按钮：
[拉取世界库内容] [推送本书世界]
```

### 前提条件
1. 小说世界必须是从世界库导入的（`sourceWorldId`存在）
2. 如果小说世界是手动创建或根据主题生成的，需要先"保存为世界样本"

## 为什么你的界面上没有看到？

### 可能的原因

#### 1. 小说世界不是从世界库导入的
**检查方法**：看"本书世界"卡片顶部的Badge
- 如果显示"手动创建"或"AI生成" → 需要先保存到世界库
- 如果显示"来自世界库" → 应该有同步管理

**解决方法**：
如果显示"手动创建"或"AI生成"，需要：
1. 点击"保存为世界样本"按钮（在卡片底部）
2. 这会将当前世界保存到世界库
3. 然后就会显示同步管理部分

#### 2. 差异尚未检测
**检查方法**：看同步管理部分的"差异检查"状态
- 如果显示"等待检查" → 需要等待或刷新页面

#### 3. 同步管理部分在页面底部
**检查方法**：向下滚动
- 同步管理在"本书世界"卡片的中间位置
- 在"世界手册"和"世界资产"之后

## 完整操作流程

### 步骤1：确认小说世界来源
1. 打开小说项目设置页面
2. 找到"本书世界"卡片
3. 查看顶部Badge显示的世界来源类型

### 步骤2：如果需要，先保存到世界库
如果来源是"手动创建"或"AI生成"：
1. 在"本书世界"卡片底部找到"保存到世界库"部分
2. 点击"保存为世界样本"按钮
3. 等待保存完成

### 步骤3：找到同步管理
1. 向下滚动"本书世界"卡片
2. 找到标题为"同步管理"的部分（在世界手册和世界资产之后）
3. 这里会显示差异检查结果和同步按钮

### 步骤4：拉取世界库更新
1. 在"同步管理"部分，选择要同步的分区（勾选checkbox）
2. 点击"拉取世界库更新"按钮
3. 等待同步完成

### 步骤5：整理使用范围
1. 继续向下滚动，找到"整理本书使用范围"卡片
2. 点击"整理本书使用范围"按钮
3. 这会根据最新的世界内容重新生成使用范围
