# REQ-7050: AI味自动检测

## 1. 需求背景

### 1.1 问题描述

AI生成的小说内容常带有明显的"AI味"——过度使用某些词汇（"值得一提的是"、"在这个充满..."）、句式单调、情感表达模式化等。当前系统缺乏自动检测机制，用户需要人工识别这些问题，严重影响创作效率和最终作品质量。

### 1.2 现状分析

**已有功能**：
- 无AI味检测相关功能
- 质量检查器（REQ-7048）中预留了AI味检测接口
- 手动审核依赖用户经验

**缺失功能**：
- ❌ AI味特征自动检测
- ❌ AI味评分量化
- ❌ 超阈值自动调整生成参数
- ❌ AI味问题点定位和建议

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 追求作品自然度的创作者
- 对AI辅助创作质量有高要求的用户
- 希望系统自动优化内容质量的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: AI味特征检测

**描述**：检测文本中常见的AI生成痕迹特征。

**检测维度**：
- **词汇维度**：检测高频AI词汇（"值得一提"、"在这个充满"、"不禁"、"缓缓"等）
- **句式维度**：检测句式单调性（句子长度方差、句式重复率）
- **情感维度**：检测情感表达模式化（情感词分布、感叹号密度）
- **逻辑维度**：检测过度总结/归纳（频繁使用"总之"、"综上所述"）
- **多样性维度**：检测词汇重复率、句式变化度

**验收标准**：
- [ ] 检测常见AI高频词汇，输出命中数量和占比
- [ ] 分析句式多样性，输出句子长度方差
- [ ] 检测情感表达模式化程度
- [ ] 检测过度归纳/总结倾向
- [ ] 计算综合AI味评分（0-100，越高越AI味）

#### FR-2: AI味评分量化

**描述**：将检测结果量化为直观的评分。

**评分规则**：
- 综合评分0-100分
- 0-30分：自然度高（优秀）
- 31-60分：轻微AI味（可接受）
- 61-80分：明显AI味（需关注）
- 81-100分：重度AI味（需重写）
- 各维度独立评分

**验收标准**：
- [ ] 综合评分准确反映AI味程度
- [ ] 各维度评分独立可查
- [ ] 评分与人工判断基本一致

#### FR-3: 超阈值自动调整

**描述**：当AI味评分超过阈值时，自动调整生成参数或触发重写。

**策略**：
- 轻度超阈值（61-80分）：提高温度参数，增加随机性
- 重度超阈值（81-100分）：触发重新生成（REQ-7049），并注入反AI味prompt
- 可配置阈值和调整策略

**验收标准**：
- [ ] 轻度超阈值时自动调整温度参数
- [ ] 重度超阈值时触发重新生成
- [ ] 调整策略可通过配置修改
- [ ] 调整过程有日志记录

#### FR-4: AI味问题报告

**描述**：输出详细的AI味问题报告，帮助用户理解问题所在。

**报告内容**：
- 综合AI味评分
- 各维度评分和问题描述
- 具体问题片段定位（行号/段落）
- 改进建议

**验收标准**：
- [ ] 报告包含综合评分和各维度评分
- [ ] 报告包含具体问题片段定位
- [ ] 报告包含改进建议
- [ ] 报告存储到数据库

### 2.2 非功能需求

#### NFR-1: 性能

- 单章AI味检测耗时<3秒
- 支持异步检测模式
- 不阻塞章节生成流程

#### NFR-2: 准确性

- 综合评分与人工判断一致性>70%
- 误报率<20%
- 持续优化检测模型

#### NFR-3: 可扩展性

- 检测词汇库可配置和扩展
- 检测维度可增减
- 评分模型可调整

## 3. 技术约束

### 3.1 架构约束

- AI味检测作为独立模块实现
- 被质量检查器（REQ-7048）调用
- 检测结果影响自动重试（REQ-7049）的参数调整

### 3.2 依赖约束

- 依赖：无（独立模块）
- 被依赖：REQ-7048（质量检查器集成）、REQ-7049（参数调整依据）

### 3.3 数据约束

- AI味检测结果存储在 `ChapterQualityReport.dimensions` JSON字段
- AI味检测词汇库存储在 `AiSmellDictionary` 表

## 4. 验收标准

### 4.1 功能验收

- [ ] 所有检测维度正常工作
- [ ] 评分量化准确
- [ ] 超阈值自动调整正常
- [ ] 问题报告格式完整

### 4.2 性能验收

- [ ] 单章检测耗时<3秒
- [ ] 异步模式不阻塞生成流程
- [ ] 并发检测无性能问题

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖（无AI味内容、高度AI味内容、混合内容）

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 检测不准 | 误报/漏报 | 中 | 持续优化词汇库 + 可配置阈值 |
| 词汇库过时 | 检测效果下降 | 中 | 支持动态更新词汇库 |
| 过度检测 | 合理表达被误判 | 低 | 提供白名单机制 |
| 调整策略不当 | 生成质量下降 | 低 | 合理的参数调整范围 |

## 6. 工作量评估

- **开发时间**：1天
- **测试时间**：0.5天
- **总计**：1.5天

## 7. 优先级

**P1** - 高优先级

**理由**：
- AI味是AI辅助创作的核心痛点
- 为质量检查器（REQ-7048）提供关键维度
- 直接影响作品的自然度和可读性
