# REQ-7057: AI味趋势追踪 — 决策日志

## 决策1: 趋势数据存储方式

**决策日期**：2026-07-11

**背景**：趋势数据是实时聚合还是预计算存储？

**选项**：
- **方案A**: 实时聚合（从章节评分记录实时计算）
- **方案B**: 预计算存储（定时任务计算并存储趋势数据）

**决策**：✅ 方案A

**理由**：
1. 项目章节数量有限（160-200章），实时聚合性能足够
2. 数据一致性好，不会出现预计算数据与原始数据不一致
3. 实现简单，不需要定时任务
4. 查询响应时间<100ms（小数据量）

**影响**：
- 每次查询趋势数据需要聚合计算
- 数据量增大时可能需要缓存
- 依赖REQ-7050的评分记录表

## 决策2: 异常点检测算法

**决策日期**：2026-07-11

**背景**：如何检测AI味评分的异常变化？

**选项**：
- **方案A**: 简单规则检测（连续下降、突变阈值）
- **方案B**: 统计方法（标准差、Z-score）
- **方案C**: LLM分析

**决策**：✅ 方案A

**理由**：
1. 规则检测实现简单，结果可预测
2. 连续3章下降、单章突变20分的规则足够覆盖常见异常
3. 无需LLM调用，成本为零
4. 规则阈值可后续调整

**影响**：
- 异常检测逻辑简单
- 可能有少量误报（正常波动触发）
- 用户可以忽略误报的异常提示

## 决策3: 前端图表库选择

**决策日期**：2026-07-11

**背景**：前端趋势图使用哪个图表库？

**选项**：
- **方案A**: Recharts（React生态）
- **方案B**: Chart.js + react-chartjs-2
- **方案C**: 自实现SVG

**决策**：✅ 方案A

**理由**：
1. Recharts是React生态首选，声明式API
2. 与React 19兼容
3. 支持响应式和自定义
4. 社区活跃，文档完善

**影响**：
- 新增Recharts依赖
- 需要安装 `recharts`
- 图表组件使用JSX声明式定义

## 决策4: 对比功能实现

**决策日期**：2026-07-11

**背景**：如何实现两个章节范围的对比？

**选项**：
- **方案A**: 后端计算对比结果，返回差异数据
- **方案B**: 前端获取两个范围数据，前端计算差异

**决策**：✅ 方案A

**理由**：
1. 对比计算在后端更高效（不需要传输大量原始数据）
2. 后端可复用趋势聚合逻辑
3. 前端只需要展示对比结果
4. 支持更复杂的对比逻辑扩展

**影响**：
- 后端新增对比API
- 返回结构化对比结果（平均分、变化趋势、各维度差异）
- 前端展示对比表格和图表

## 决策5: 异常提示内容

**决策日期**：2026-07-11

**背景**：检测到异常时应该提示什么内容？

**选项**：
- **方案A**: 仅提示异常事实
- **方案B**: 提示异常事实+可能原因
- **方案C**: 提示异常事实+可能原因+修复建议

**决策**：✅ 方案C

**理由**：
1. 用户需要知道为什么会出现异常
2. 修复建议帮助用户快速定位和解决问题
3. 提升功能的实用性和用户满意度
4. 建议内容可以由LLM生成或预定义

**影响**：
- 异常点包含suggestion字段
- 部分建议预定义（如"连续下降建议检查提示词"）
- 部分建议由LLM分析生成
