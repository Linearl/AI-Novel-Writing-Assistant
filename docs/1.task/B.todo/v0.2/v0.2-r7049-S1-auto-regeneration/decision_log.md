# REQ-7049: 自动重新生成 — 决策日志

## 决策1: 重试触发方式

**决策日期**：2026-07-11

**背景**：质量不合格时如何触发重试？

**选项**：
- **方案A**: 事件驱动——监听质量检查完成事件，自动触发
- **方案B**: 同步回调——质量检查完成后直接调用重试
- **方案C**: 轮询检查——定时检查未处理的不合格章节

**决策**：方案A

**理由**：
1. 与项目现有的事件总线架构一致
2. 松耦合，质量检查器和重试管理器独立
3. 异步执行不阻塞当前流程
4. 支持多个章节并发重试

**影响**：
- 注册事件监听器
- 在事件处理器中异步执行重试
- 重试失败有独立的错误处理

## 决策2: 参数调整策略

**决策日期**：2026-07-11

**背景**：每次重试时如何调整生成参数以提高成功率？

**选项**：
- **方案A**: 每次递增温度（+0.1），增加随机性
- **方案B**: 每次递减温度（-0.1），增加稳定性
- **方案C**: 交替调整温度（递增+递减）
- **方案D**: 更换prompt变体

**决策**：方案A（温度递增为主，预留prompt变体接口）

**理由**：
1. 质量不合格往往因为过于保守/重复，增加温度可引入多样性
2. 递增策略简单可预测
3. 预留prompt变体接口，后续可扩展更复杂的策略
4. 设置最大温度上限（1.5）防止过度随机

**影响**：
- 参数调整逻辑实现为纯函数，易于测试
- 在配置中预留 `promptVariants` 字段
- 重试日志记录每次参数变化

## 决策3: 重试与原结果的关系

**决策日期**：2026-07-11

**背景**：重新生成后，新结果是覆盖原结果，还是保留所有版本供选择？

**选项**：
- **方案A**: 直接覆盖原结果
- **方案B**: 保留所有版本，选择最佳结果覆盖
- **方案C**: 保留所有版本，由用户选择

**决策**：方案B（自动选择最佳结果）

**理由**：
1. 自动化目标是减少人工干预，不应让用户选择
2. 保留所有版本可回溯，但最终只需最佳版本
3. 与Auto-Director质量门原则一致——自动处理，用户无需介入
4. 重试日志中保留所有版本的评分，便于分析

**影响**：
- 重试过程中保存所有生成内容
- 最终应用质量评分最高的版本
- 重试日志记录所有尝试的详细信息

## 决策4: 自动模式 vs 手动确认

**决策日期**：2026-07-11

**背景**：重试是完全自动还是需要用户确认？

**选项**：
- **方案A**: 完全自动（默认）
- **方案B**: 每次重试前需用户确认
- **方案C**: 可配置，默认自动

**决策**：方案C（可配置，默认自动）

**理由**：
1. 大多数场景下用户希望全自动
2. 但某些重要章节可能需要人工确认
3. 可配置提供灵活性
4. 通知机制保证用户知情

**影响**：
- 在RegenerationConfig中新增 `autoMode` 字段
- 手动模式下发送通知等待确认
- API支持手动触发重试

## 决策5: 重试结果的最终保留

**决策日期**：2026-07-11

**背景**：当3次重试都未通过质量检查时，如何处理？

**选项**：
- **方案A**: 保留最后一次生成结果
- **方案B**: 保留所有尝试中质量评分最高的结果
- **方案C**: 保留原始结果，丢弃重试结果

**决策**：方案B

**理由**：
1. 最后一次不一定是最好的
2. 保留最佳结果可以最大化内容质量
3. 即使最终不合格，也至少比原始版本有提升
4. 与"最佳结果保留"原则一致

**影响**：
- 重试过程中跟踪最佳评分
- 最终应用最佳版本
- 重试日志记录最终选择的原因

---

## 决策6: 并发控制

**决策日期**：2026-07-11

**背景**：多个章节同时需要重试时如何处理？

**选项**：
- **方案A**: 无限制并发
- **方案B**: 限制最大并发数（默认3）
- **方案C**: 串行执行，排队等待

**决策**：方案B

**理由**：
1. 无限制并发可能导致API限流
2. 串行执行效率太低
3. 限制并发数是最佳平衡点
4. 并发数可配置，适应不同环境

**影响**：
- 实现并发队列
- 在配置中添加 `maxConcurrentRetries` 字段
- 重试请求进入队列等待空闲槽位

## 总结

| 决策 | 选择 | 理由 |
|------|------|------|
| 触发方式 | 事件驱动 | 松耦合，与架构一致 |
| 参数调整 | 温度递增 | 增加随机性提高多样性 |
| 结果关系 | 自动选择最佳 | 减少人工干预 |
| 自动模式 | 可配置，默认自动 | 灵活性 |
| 最终保留 | 保留最佳结果 | 最大化内容质量 |
| 并发控制 | 限制并发数 | 平衡效率和稳定性 |
