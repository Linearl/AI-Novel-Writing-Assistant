# REQ-7059: Prompt 模板系统

## 1. 需求背景

### 1.1 问题描述

当前 Prompt 管理基于 PromptAsset + registry 的静态注册机制，所有小说共享同一套 prompt 模板。实际创作中，不同小说（不同题材、风格、目标读者）需要不同的 prompt 策略。需要一个高级模板系统，支持：
- 模板中引用动态上下文（`{{context.xxx}}` token 语法）
- 每本小说可自定义 writer prompt（覆盖官方默认版本）
- 版本化管理（保存、回滚、对比）
- 编译时诊断（检测缺失的上下文组和输入字段）
- 安全约束（必需上下文组不可移除）

### 1.2 现状分析

**已有功能**：
- PromptAsset 注册机制（registry.ts）
- PromptSlot 系统（slotTypes）
- 上下文组解析（promptContextResolution）
- 基础 prompt 执行（runStructuredPrompt）

**缺失功能**：
- 模板 token 语法（`{{context.xxx}}`、`{{input.xxx}}`、`{{slot.xxx}}`）
- 模板编译器（token 解析 + 诊断）
- 官方模板存储和管理
- 每本小说的模板覆盖（override）
- 版本化（保存多个版本 + 回滚）
- 编译诊断（缺失组、缺失字段、未知 token）
- 安全约束（WRITER_REQUIRED_CONTEXT_GROUPS 不可移除）

### 1.3 目标用户

- 需要针对不同小说调整 prompt 策略的作者
- 需要 A/B 测试不同 prompt 版本的高级用户
- 系统运维人员（需要诊断 prompt 编译问题）

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 模板 Token 语法

**描述**：支持三种 token 引用语法，用于模板中的动态内容替换。

**规则**：
- `{{context.xxx}}` — 引用上下文组（如 `{{context.book_contract}}`）
- `{{input.xxx}}` — 引用输入字段（如 `{{input.chapter_content}}`）
- `{{slot.xxx}}` — 引用 Slot 值（如 `{{slot.style_guide}}`）
- Token 使用双大括号包裹，支持内部空格
- 未知 token 会被标记为诊断警告

**验收标准**：
- [ ] 支持 `{{context.xxx}}` token 解析
- [ ] 支持 `{{input.xxx}}` token 解析
- [ ] 支持 `{{slot.xxx}}` token 解析
- [ ] 支持内部空格（`{{ context.xxx }}`）
- [ ] 未知 token 识别

#### FR-2: 模板编译器

**描述**：将模板 JSON 编译为可执行的 LangChain 消息，并提供诊断信息。

**规则**：
- 输入：PromptTemplateJson（含 messages 数组）
- 输出：LangChain BaseMessage[] + PromptTemplateDiagnostics
- 诊断包含：引用的上下文组、输入字段、Slot 键、缺失组、缺失字段、未知 token、无效消息
- 编译失败时返回诊断而不抛异常
- 模板最大长度 60000 字符（ADVANCED_TEMPLATE_MAX_CHARS）

**验收标准**：
- [ ] 模板编译为 LangChain 消息正确
- [ ] 诊断信息完整（9 个维度）
- [ ] 编译失败不抛异常
- [ ] 最大长度限制生效

#### FR-3: 官方模板管理

**描述**：管理官方（official）prompt 模板，作为所有小说的默认版本。

**规则**：
- 官方模板存储在代码中（officialTemplates.ts）
- 每个 PromptAsset 关联一个官方模板
- 官方模板有版本号（versionNo）
- 官方模板不可被用户修改
- 提供 hash 校验确保模板完整性

**验收标准**：
- [ ] 官方模板正确存储和加载
- [ ] 版本号正确管理
- [ ] hash 校验正确

#### FR-4: 小说级模板覆盖

**描述**：每本小说可以创建自定义模板，覆盖官方默认版本。

**规则**：
- 每本小说 + 每个 promptId 只有一个活跃覆盖
- 覆盖模式：official（使用官方模板）或 custom（使用自定义模板）
- 自定义模板基于官方模板修改
- 覆盖记录存储在 PromptTemplateOverride 表
- 版本存储在 PromptTemplateVersion 表

**验收标准**：
- [ ] 可创建小说级模板覆盖
- [ ] 覆盖模式切换正常（official ↔ custom）
- [ ] 不同小说的覆盖互不影响

#### FR-5: 版本化管理

**描述**：模板覆盖支持版本化，可保存、查看历史、回滚。

**规则**：
- 每次保存创建新版本（versionNo 递增）
- 每个版本记录：模板 JSON、上下文引用、编译 hash、备注
- 支持回滚到任意历史版本
- 支持查看版本对比
- 活跃版本（activeVersionId）指向当前使用的版本

**验收标准**：
- [ ] 保存创建新版本
- [ ] 版本历史列表正确
- [ ] 回滚到指定版本正常
- [ ] 版本对比可用

#### FR-6: 编译诊断

**描述**：模板编译时提供详细的诊断信息，帮助用户定位问题。

**规则**：
- 诊断维度（9 个）：
  - referencedContextGroups — 引用的上下文组
  - referencedInputFields — 引用的输入字段
  - referencedSlotKeys — 引用的 Slot 键
  - fallbackRequiredGroups — 回退的必需组
  - missingRequiredGroups — 缺失的必需组
  - missingReferencedContextGroups — 缺失的引用上下文组
  - missingInputFields — 缺失的输入字段
  - unknownTokens — 未知 token
  - invalidMessages — 无效消息
- 有 blocking 诊断时阻止保存

**验收标准**：
- [ ] 9 维诊断正确生成
- [ ] 缺失必需组正确识别
- [ ] 未知 token 正确识别
- [ ] 阻塞诊断阻止保存

#### FR-7: 安全约束

**描述**：保护关键上下文组不被用户移除。

**规则**：
- 定义 WRITER_REQUIRED_CONTEXT_GROUPS（10 个必需组）
- 用户编辑模板时，必需组不可删除
- 编译时检测必需组是否完整
- 缺少必需组时标记为 blocking 诊断

**验收标准**：
- [ ] WRITER_REQUIRED_CONTEXT_GROUPS 定义正确（10 个）
- [ ] 编辑时保护必需组
- [ ] 编译时检测必需组完整性

#### FR-8: Slot 系统 official_default 模式

**描述**：Slot 系统支持 official_default 模式，在没有自定义覆盖时使用官方默认值。

**规则**：
- Slot 值优先级：自定义覆盖 > 官方默认
- official_default 模式：当小说无覆盖时使用官方模板的 Slot 值
- Slot 值在模板编译时解析

**验收标准**：
- [ ] Slot 值优先级正确
- [ ] official_default 模式正常工作
- [ ] Slot 值在编译时正确解析

### 2.2 非功能需求

#### NFR-1: 性能

- 模板编译 < 50ms（不含 LLM 调用）
- 版本查询 < 200ms

#### NFR-2: 可靠性

- 版本数据不丢失
- 编译诊断准确
- 回滚操作原子性

## 3. 技术约束

### 3.1 架构约束

- 模板系统位于 `server/src/prompting/templates/`
- 遵循 Prompt Governance（新增产品级 prompt 在 prompting/ 注册）
- 与现有 PromptAsset + registry 系统集成
- 使用 LangChain 消息类型（SystemMessage, HumanMessage）

### 3.2 依赖约束

- 依赖现有 PromptRegistry
- 依赖现有 PromptSlot 系统
- 依赖现有 promptContextResolution
- 被依赖：章节生成、Creative Hub 等所有 LLM 调用

### 3.3 数据约束

- 新增 2 个 Prisma 模型
- 模板 JSON 最大 60000 字符
- 版本历史无硬上限（依赖清理策略）

### 4.4 Prisma 模型（上游参考）

```prisma
model PromptTemplateOverride {
  id              String   @id @default(cuid())
  scope           String   // "novel"
  novelId         String
  promptId        String   // PromptAsset ID
  basePromptVersion String // 基于的官方版本
  mode            String   @default("official") // "official" | "custom"
  activeVersionId String?  // 当前活跃版本 ID
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  versions PromptTemplateVersion[]

  @@unique([scope, novelId, promptId])
}

model PromptTemplateVersion {
  id             String   @id @default(cuid())
  overrideId     String
  versionNo      Int      // 版本号（递增）
  templateJson   String   // PromptTemplateJson 序列化
  contextRefsJson String  // PromptTemplateContextRefs 序列化
  compiledHash   String   // 编译 hash
  notes          String?  // 版本备注
  createdAt      DateTime @default(now())

  override PromptTemplateOverride @relation(fields: [overrideId], references: [id])

  @@index([overrideId, versionNo])
}
```
