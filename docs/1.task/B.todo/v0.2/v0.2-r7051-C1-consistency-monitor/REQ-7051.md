# REQ-7051: 一致性实时监控

## 1. 需求背景

### 1.1 问题描述

在长篇小说创作（160章+）中，AI生成的内容容易出现跨章节一致性问题：时间线混乱（同一天内发生不可能的事件序列）、人物行为矛盾（性格突变、前后行为不一致）、空间逻辑错误（人物瞬间移动、地理位置矛盾）等。当前系统缺乏自动监控机制，用户需要人工追踪这些问题，严重影响作品质量。

### 1.2 现状分析

**已有功能**：
- 质量检查器（REQ-7048）— 单章质量检查，无跨章节分析
- 情节连贯性检查器 — 基于关键词检测，深度有限
- 人工审核回调 — 依赖用户记忆和笔记

**缺失功能**：
- ❌ 跨章节时间线一致性检测
- ❌ 跨章节人物行为一致性检测
- ❌ 跨章节空间逻辑一致性检测
- ❌ 矛盾自动报告和定位

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 创作160章+长篇网文的作者
- 需要维护复杂世界观和人物关系的创作者
- 对作品逻辑一致性有高要求的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 时间线一致性检测

**描述**：检测跨章节的时间逻辑是否一致。

**检测维度**：
- 时间顺序：事件是否按时间顺序发生
- 时间矛盾：同一天内不可能发生的事件序列
- 时间跳跃：不合理的大幅度时间跳跃（如突然跳过重要事件）
- 季节/日期一致性：日期与季节、天气等描述是否匹配

**验收标准**：
- [ ] 正确检测时间顺序矛盾
- [ ] 正检检测不合理的时间跳跃
- [ ] 标记时间矛盾的具体章节和位置
- [ ] 输出时间线可视化数据

#### FR-2: 人物行为一致性检测

**描述**：检测跨章节人物行为和性格是否一致。

**检测维度**：
- 性格一致性：人物行为是否符合其设定性格
- 技能一致性：人物展示的能力是否与其设定一致
- 关系一致性：人物之间的关系变化是否有合理铺垫
- 背景一致性：人物背景信息是否前后一致

**验收标准**：
- [ ] 检测人物性格突变
- [ ] 检测能力设定矛盾
- [ ] 检测背景信息不一致
- [ ] 标记矛盾的具体章节和位置

#### FR-3: 空间逻辑一致性检测

**描述**：检测跨章节的空间逻辑是否一致。

**检测维度**：
- 地理位置：人物位置是否合理（不会瞬间移动）
- 场景连贯：场景转换是否有过渡
- 空间约束：封闭空间内的人物进出是否合理

**验收标准**：
- [ ] 检测人物位置矛盾
- [ ] 检测不合理的场景转换
- [ ] 标记空间矛盾的具体章节和位置

#### FR-4: 矛盾报告和可视化

**描述**：将检测到的矛盾生成结构化报告。

**报告内容**：
- 矛盾类型和严重程度
- 涉及的章节和具体位置
- 矛盾描述和修复建议
- 一致性趋势图表

**验收标准**：
- [ ] 报告包含所有检测到的矛盾
- [ ] 每个矛盾有严重程度分级
- [ ] 报告包含修复建议
- [ ] 支持一致性趋势可视化

### 2.2 非功能需求

#### NFR-1: 性能

- 单章一致性检查耗时<10秒（需要读取前几章数据）
- 支持增量检查（仅检查新增章节）
- 不阻塞章节生成流程

#### NFR-2: 准确性

- 矛盾检测准确率>70%
- 误报率<30%
- 支持人工修正和反馈

#### NFR-3: 可扩展性

- 矛盾检测规则可配置
- 支持后续新增检测维度
- 与现有质量检查系统集成

## 3. 技术约束

### 3.1 架构约束

- 一致性监控作为独立服务模块实现
- 需要读取前几章内容进行对比分析
- 异步执行，不阻塞生成流程

### 3.2 依赖约束

- 依赖：REQ-7048（质量检查器框架）
- 被依赖：无

### 3.3 数据约束

- 矛盾报告存储在 `ConsistencyViolation` 表
- 时间线数据从章节内容中提取并缓存
- 需要访问章节内容、人物设定等数据

## 4. 验收标准

### 4.1 功能验收

- [ ] 时间线检测正常工作
- [ ] 人物行为检测正常工作
- [ ] 空间逻辑检测正常工作
- [ ] 矛盾报告格式完整

### 4.2 性能验收

- [ ] 单章检查耗时<10秒
- [ ] 增量检查效率优于全量检查
- [ ] 异步模式不阻塞生成流程

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖（单章/多章/无矛盾/有矛盾场景）

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 检测不准 | 误报/漏报 | 高 | 多规则交叉验证 + 可配置阈值 |
| 性能问题 | 检查耗时过长 | 中 | 增量检查 + 缓存策略 |
| 数据不足 | 首章无法检查 | 低 | 至少2章才启动检查 |
| 矛盾误判 | 合理情节被标记 | 中 | 提供"合理"标记和反馈机制 |

## 6. 工作量评估

- **开发时间**：1天
- **测试时间**：0.5天
- **总计**：1.5天

## 7. 优先级

**P1** - 高优先级

**理由**：
- 长篇小说一致性的核心痛点
- 无其他工具能自动解决此问题
- 技术风险主要在检测准确性，可迭代优化
