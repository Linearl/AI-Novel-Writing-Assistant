# REQ-7052: 后台任务管理

## 1. 需求背景

### 1.1 问题描述

长篇小说创作涉及大量AI生成任务（章节生成、角色设定、世界观构建等），单个任务可能耗时数分钟。当前系统中，长任务执行期间用户界面会被阻塞或需要持续等待，无法进行其他操作（如浏览已生成内容、修改设定、开始新的创作任务等）。这严重影响了创作效率和用户体验。

### 1.2 现状分析

**已有功能**：
- Novel Side Effect Job — 任务级别的副作用处理
- 前端 TanStack Query 数据获取 — 异步请求管理
- 后端任务队列基础 — 基本的任务调度

**缺失功能**：
- ❌ 任务后台执行管理（提交、查询、取消）
- ❌ 任务暂停/恢复能力
- ❌ 任务状态实时同步到前端
- ❌ 用户在任务执行期间的自由操作支持

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 同时进行多个创作任务的用户
- 需要中断AI生成并稍后继续的用户
- 创作过程中需要切换工作内容的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 后台任务提交

**描述**：用户可将耗时操作提交为后台任务，立即获得任务ID，不阻塞当前操作。

**规则**：
- 提交后立即返回任务ID和初始状态
- 支持的任务类型：章节生成、角色设定生成、世界观生成、全书执行等
- 任务参数随提交请求保存
- 同一项目下最多同时执行5个任务

**验收标准**：
- [ ] 提交任务后立即返回任务ID
- [ ] 返回状态包含pending/running/completed/failed/paused
- [ ] 同一项目超过5个并发任务时拒绝并提示
- [ ] 任务参数完整保存

#### FR-2: 任务状态查询

**描述**：支持查询单个任务状态和项目下所有任务列表。

**规则**：
- 查询返回：任务ID、类型、状态、进度（0-100%）、开始时间、预估剩余时间
- 支持按状态筛选（进行中、已完成、已失败等）
- 支持SSE实时推送状态变更

**验收标准**：
- [ ] 查询单个任务返回完整状态信息
- [ ] 查询列表支持状态筛选和分页
- [ ] SSE连接建立后能实时接收状态更新
- [ ] 断线重连后能获取最新状态

#### FR-3: 任务暂停/恢复

**描述**：支持暂停运行中的任务，以及恢复已暂停的任务。

**规则**：
- 暂停：任务在当前步骤完成后停止，不再执行后续步骤
- 恢复：从暂停点继续执行，不重复已完成步骤
- 暂停/恢复操作在3秒内生效
- 仅running状态可暂停，仅paused状态可恢复

**验收标准**：
- [ ] running状态任务可暂停
- [ ] paused状态任务可恢复
- [ ] 恢复后从断点继续，不重复已完成步骤
- [ ] pending和completed状态不支持暂停/恢复

#### FR-4: 任务取消

**描述**：支持取消运行中或已暂停的任务，清理相关资源。

**规则**：
- 取消后任务状态变为cancelled
- 已生成的部分结果保留
- 释放任务占用的资源
- cancelled状态不可恢复

**验收标准**：
- [ ] running/paused任务可取消
- [ ] 取消后状态变为cancelled
- [ ] 部分生成结果保留可查看
- [ ] 资源正确释放

### 2.2 非功能需求

#### NFR-1: 性能

- 任务状态查询响应时间<100ms
- SSE状态推送延迟<500ms
- 暂停/恢复操作3秒内生效

#### NFR-2: 可靠性

- 服务重启后未完成任务可恢复
- 任务执行失败时保留错误信息和上下文
- 部分完成的结果不丢失

#### NFR-3: 可观测性

- 任务执行过程可追踪（日志）
- 关键指标可监控（任务成功率、平均执行时间）

## 3. 技术约束

### 3.1 架构约束

- 任务状态存储在数据库中（SQLite/PostgreSQL）
- 使用Express服务端事件（SSE）实现实时状态推送
- 任务执行器需支持暂停点（checkpoint）机制
- 后台任务管理器作为独立服务模块

### 3.2 依赖约束

- 依赖：无
- 被依赖：REQ-7053（关键节点通知）、REQ-7056（跨章节人物一致性）

### 3.3 数据约束

- 任务状态记录需要持久化到数据库
- 任务参数和结果需要序列化存储
- 状态变更历史需要记录

## 4. 验收标准

### 4.1 功能验收

- [ ] 任务提交后立即返回，不阻塞
- [ ] 任务状态查询正常
- [ ] SSE实时状态推送正常
- [ ] 任务暂停/恢复正常工作
- [ ] 任务取消正常工作
- [ ] 并发任务数量限制生效

### 4.2 性能验收

- [ ] 状态查询<100ms
- [ ] SSE推送延迟<500ms
- [ ] 暂停/恢复3秒内生效

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖（并发限制、状态流转）

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 暂停点实现复杂 | 开发周期延长 | 中 | 分阶段实现，先支持简单暂停 |
| SSE连接断开 | 状态推送丢失 | 低 | 支持断线重连和状态补偿 |
| 并发控制冲突 | 资源竞争 | 中 | 数据库级别锁+状态机校验 |
| 服务重启丢失任务 | 任务丢失 | 低 | 任务状态持久化到数据库 |

## 6. 工作量评估

- **开发时间**：0.75天
- **测试时间**：0.25天
- **总计**：1天

## 7. 优先级

**P1** - 高优先级

**理由**：
- 直接影响创作效率，长任务阻塞是核心痛点
- 是v0.2用户体验提升的关键功能
- 技术复杂度适中，可快速实现
