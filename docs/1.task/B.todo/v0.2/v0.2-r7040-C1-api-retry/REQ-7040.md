# REQ-7040: API失败自动重试

## 1. 需求背景

### 1.1 问题描述

当前LLM API调用失败时（网络超时、429限流、服务不可用等），系统直接抛出错误，用户需要手动重试。这在长篇小说创作中（160章+）会导致频繁阻塞，严重影响用户体验。

### 1.2 现状分析

**已有功能**：
- SQLite数据库重试（withSqliteRetry）- 针对瞬态数据库错误
- Novel Side Effect Job指数退避重试 - 针对任务级别
- 结构化输出策略降级 - 针对输出格式兼容性

**缺失功能**：
- ❌ LLM API调用的传输层重试（网络错误、429限流等）
- ❌ 重试次数可配置
- ❌ 429 Retry-After头解析

### 1.3 目标用户

所有使用系统进行长篇小说创作的用户，特别是：
- 生成160章+网文的用户
- 使用不稳定网络环境的用户
- 使用有限流API的用户

## 2. 需求定义

### 2.1 功能需求

#### FR-1: 指数退避重试

**描述**：当LLM API调用因网络错误失败时，系统自动重试，采用指数退避算法。

**规则**：
- 最多重试3次（可配置）
- 退避时间：1秒 → 2秒 → 4秒
- 仅重试可恢复错误（网络超时、502/503/504、429限流）
- 不重试不可恢复错误（401/403认证失败、400请求错误）

**验收标准**：
- [ ] 网络超时时自动重试3次
- [ ] 每次重试间隔符合指数退避算法
- [ ] 重试成功后正常返回结果
- [ ] 重试3次仍失败后抛出错误

#### FR-2: 429限流智能重试

**描述**：当收到429 Too Many Requests响应时，解析Retry-After头，等待指定时间后重试。

**规则**：
- 解析Retry-After头（秒数或HTTP日期）
- 等待Retry-After指定的时间（最多60秒）
- 如果没有Retry-After头，使用指数退避
- 等待后自动重试

**验收标准**：
- [ ] 正确解析Retry-After头
- [ ] 等待指定时间后重试
- [ ] 没有Retry-After头时使用指数退避
- [ ] 最大等待时间不超过60秒

#### FR-3: 重试次数可配置

**描述**：重试次数和退避参数可通过配置调整。

**配置项**：
- `maxRetries`: 最大重试次数（默认3）
- `baseDelayMs`: 基础延迟时间（默认1000ms）
- `maxDelayMs`: 最大延迟时间（默认60000ms）

**验收标准**：
- [ ] 支持通过配置调整重试次数
- [ ] 支持通过配置调整退避参数
- [ ] 默认配置合理（3次重试）

#### FR-4: 重试日志记录

**描述**：每次重试都记录详细日志，便于排查问题。

**日志内容**：
- 重试次数
- 错误类型和消息
- 等待时间
- 重试结果

**验收标准**：
- [ ] 每次重试都有日志记录
- [ ] 日志包含必要的调试信息
- [ ] 日志级别合理（warn for retry, error for failure）

### 2.2 非功能需求

#### NFR-1: 性能

- 重试逻辑不应显著增加正常调用的延迟
- 重试等待期间不应阻塞其他请求

#### NFR-2: 可观测性

- 重试事件应可通过日志追踪
- 关键指标可监控（重试次数、成功率）

#### NFR-3: 兼容性

- 不影响现有LLM调用逻辑
- 向后兼容现有配置

## 3. 技术约束

### 3.1 架构约束

- 必须在 `invokeStructuredLlmDetailed` 层实现（方案A）
- 必须在策略降级之后重试
- 复用现有的 `StructuredOutputError` 分类（transport_error可重试）

### 3.2 依赖约束

- 依赖：无
- 被依赖：REQ-7041（模型备用切换）、REQ-7042（错误分类器）

### 3.3 数据约束

- 重试配置存储在Provider Secret表
- 重试日志存储在应用日志

## 4. 验收标准

### 4.1 功能验收

- [ ] 网络超时自动重试3次
- [ ] 429限流自动解析Retry-After并重试
- [ ] 重试次数可配置
- [ ] 重试日志完整
- [ ] 重试成功后正常返回

### 4.2 性能验收

- [ ] 正常调用延迟增加<100ms
- [ ] 重试等待期间不阻塞其他请求

### 4.3 测试验收

- [ ] 单元测试覆盖率>80%
- [ ] 集成测试通过
- [ ] 边界用例覆盖（0次重试、1次重试、3次重试）

## 5. 风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 重试风暴 | API限流加剧 | 中 | 设置最大重试次数和最大等待时间 |
| 日志过多 | 日志存储成本 | 低 | 仅记录关键重试事件 |
| 配置错误 | 重试行为异常 | 低 | 配置校验和默认值 |

## 6. 工作量评估

- **开发时间**：1.5天
- **测试时间**：0.5天
- **总计**：2天
