# 决策记录

## 2026-07-01 需求分析

### D-001：采用分级恢复而非硬失败

| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 硬失败（当前） | 简单直接 | 模型无法自我纠正 |
| B: 分级恢复（MiMo Code） | 给模型机会纠正 | 实现复杂 |

**决策**：采用 B（分级恢复）
**理由**：参考 MiMo Code 的成熟设计，给模型自我纠正的机会可以减少不必要的截断

### D-002：最大恢复次数

| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 1 次 | 快速失败 | 可能过于激进 |
| B: 2 次（MiMo Code） | 平衡恢复和失败 | - |
| C: 3 次 | 更多恢复机会 | 可能浪费 token |

**决策**：采用 B（2 次）
**理由**：与 MiMo Code 保持一致，平衡恢复机会和 token 消耗

### D-003：检测算法选择

| 选项 | 优点 | 缺点 |
|------|------|------|
| A: n-gram 重复率（当前） | 实现简单 | 对段落循环不敏感 |
| B: 连续重复块检测（MiMo Code） | 对段落循环更敏感 | 实现复杂 |

**决策**：采用 B（连续重复块检测）
**理由**：对"段落循环"模式更敏感，检测更准确

### D-004：分词策略

| 选项 | 优点 | 缺点 |
|------|------|------|
| A: 字符级（当前） | 简单 | 中文处理不友好 |
| B: 中日韩分词（MiMo Code） | 中文友好 | 需要额外处理 |

**决策**：采用 B（中日韩分词）
**理由**：项目主要处理中文内容，需要更好的分词支持
