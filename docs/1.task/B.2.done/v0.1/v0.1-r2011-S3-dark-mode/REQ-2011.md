## 1. 背景与问题

### 1.1 现状

系统仅有亮色主题。`tailwind.config.ts` 已配置 `darkMode: "class"`（来自 shadcn/ui 脚手架），但：

- `index.css` 只定义了 `:root` 亮色 CSS 变量，无 `.dark` 类变量
- 未安装 `next-themes` 或任何主题管理库
- 无主题切换 UI
- 无系统主题检测

### 1.2 用户场景

1. 夜间写作需要暗色背景减轻视觉疲劳
2. 长时间盯着亮屏导致眼睛不适
3. 用户希望跟随系统主题自动切换

### 1.3 关键文件

| 文件 | 作用 |
| ---- | ---- |
| `client/tailwind.config.ts` | 已配置 `darkMode: "class"` |
| `client/src/index.css` | CSS 变量定义（仅亮色） |
| `client/src/components/layout/AppLayout.tsx` | 主布局，主题 Provider 挂载点 |
| `client/src/components/layout/Sidebar.tsx` | 左侧导航栏 |
| `client/src/components/layout/Navbar.tsx` | 顶部导航栏 |

## 2. 目标与范围

### 2.1 目标

1. 实现暗色主题，覆盖所有页面和组件
2. 支持三种模式：亮色 / 暗色 / 跟随系统
3. 主题切换 UI 放在导航栏或设置页
4. 用户偏好持久化

### 2.2 In Scope

**依赖：**
- 安装 `next-themes`

**CSS：**
- `index.css` 新增 `.dark` 类的 CSS 变量

**组件：**
- ThemeProvider 挂载
- 主题切换按钮/下拉菜单
- 系统主题检测

### 2.3 Out of Scope

- 自定义主题色（仅提供亮/暗两套）
- 每个组件的主题适配审查（依赖 Tailwind dark: 类自动适配）
- 桌面端 Electron 主题同步

## 3. 需求详情

### 3.1 暗色 CSS 变量

`index.css` 新增 `.dark` 类：

```css
.dark {
  --background: 222.2 84% 4.9%;
  --foreground: 210 40% 98%;
  --card: 222.2 84% 4.9%;
  --card-foreground: 210 40% 98%;
  --popover: 222.2 84% 4.9%;
  --popover-foreground: 210 40% 98%;
  --primary: 210 40% 98%;
  --primary-foreground: 222.2 47.4% 11.2%;
  --secondary: 217.2 32.6% 17.5%;
  --secondary-foreground: 210 40% 98%;
  --muted: 217.2 32.6% 17.5%;
  --muted-foreground: 215 20.2% 65.1%;
  --accent: 217.2 32.6% 17.5%;
  --accent-foreground: 210 40% 98%;
  --destructive: 0 62.8% 30.6%;
  --destructive-foreground: 210 40% 98%;
  --border: 217.2 32.6% 17.5%;
  --input: 217.2 32.6% 17.5%;
  --ring: 212.7 26.8% 83.9%;
}
```

色值参考 shadcn/ui 官方暗色主题。

### 3.2 ThemeProvider

安装 `next-themes`，在 `AppLayout` 中挂载：

```tsx
import { ThemeProvider } from "next-themes";

<ThemeProvider attribute="class" defaultTheme="system" enableSystem>
  {children}
</ThemeProvider>
```

- `attribute="class"`：通过 `<html class="dark">` 切换
- `defaultTheme="system"`：默认跟随系统
- `enableSystem`：启用系统主题检测

### 3.3 主题切换 UI

在导航栏或设置页提供切换入口：

**方案A：导航栏快捷切换**（推荐）

在 `Navbar.tsx` 右侧区域增加主题切换按钮：

```
[☀️ / 🌙 / 💻] 三态切换按钮
```

- ☀️ 亮色模式
- 🌙 暗色模式
- 💻 跟随系统

**方案B：设置页配置**

在设置页增加主题选择 Radio：

```
外观
○ 亮色
○ 暗色
○ 跟随系统
```

建议两个入口都保留：导航栏快捷切换 + 设置页配置。

### 3.4 持久化

`next-themes` 自动将用户选择保存到 `localStorage`：

```
Key: theme
Value: "light" | "dark" | "system"
```

页面加载时自动恢复。

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 部分组件可能未使用 CSS 变量，暗色下不可读 | 全局审查关键页面，补充 `dark:` 类 |
| 第三方组件（Plate 编辑器等）可能不支持暗色 | 逐步适配，不在首期覆盖全部 |
| 暗色主题下图片/图标对比度不足 | 仅调整背景和文字色，图片暂不处理 |
