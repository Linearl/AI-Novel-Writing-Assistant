# REQ-2036 决策日志

## 决策记录

### D1: 存储方案选择

**日期**: 2026-07-02
**决策**: 使用文件系统存储，每条反馈一个独立文件夹
**备选方案**:
1. 数据库存储（Prisma/SQLite）
2. 文件系统 + 数据库索引
3. 纯文件系统

**理由**:
- 参考项目 data_platform 采用文件系统存储，验证可行
- 文件系统存储便于查看和调试
- 反馈数据量不大，无需数据库
- 支持附件自然存储

**风险**: 文件系统查询性能不如数据库
**缓解**: 反馈数量有限，性能可接受

---

### D2: 反馈文件夹命名规则

**日期**: 2026-07-02
**决策**: 采用 `{userId}_{timestamp}` 格式
**备选方案**:
1. UUID
2. 自增 ID
3. `{userId}_{timestamp}`

**理由**:
- 包含用户 ID 便于快速识别
- 包含时间戳便于排序
- 参考项目采用此格式

---

### D3: 反馈内容格式

**日期**: 2026-07-02
**决策**: 使用 Markdown + YAML frontmatter
**备选方案**:
1. JSON
2. Markdown + frontmatter
3. 纯文本

**理由**:
- Markdown 便于人类阅读
- frontmatter 便于程序解析
- 参考项目采用此格式

---

### D4: 速率限制策略

**日期**: 2026-07-02
**决策**: 每小时最多 10 次提交
**理由**:
- 防止滥用
- 参考项目采用类似策略

---

### D5: 附件限制

**日期**: 2026-07-02
**决策**: 最多 5 张图片，每张最大 5MB
**理由**:
- 控制存储空间
- 图片为主，便于截图反馈
