# REQ-2016 自动导演失败状态重试入口缺失

> 版本：v1.0
> 创建时间：2026-06-26T22:50:00+08:00
> 最后更新：2026-06-26T22:50:00+08:00

---

## 1. 问题描述

自动导演任务进入 `failed` 状态后，自动导演面板无直接重试入口。用户只能通过 TaskDrawer 手动重试，或通过 API/CLI 干预。

## 2. 根因

- **服务端**：`DirectorDashboardViewBuilder` 对 failed 模式不生成 `retry` action
- **客户端**：`resolveDashboardAction()` 将 `resume_from_checkpoint` 映射为打开 TaskDrawer 而非触发重试

## 3. 修复方案

### 3.1 服务端：DashboardView 增加 retry action

在 `DirectorDashboardViewBuilder.buildActions()` 的 `failed` 分支中，将 primary action 改为 `retry` 类型：

```typescript
if (mode === "failed") {
  return {
    primaryAction: action("retry", "重试任务", "primary"),
    secondaryActions: [
      action("resume_from_checkpoint", "从最近进度恢复", "secondary"),
      action("open_task_center", "查看执行详情", "secondary"),
    ],
  };
}
```

### 3.2 客户端：resolveDashboardAction 处理 retry

在 `NovelAutoDirectorProgressPanel.resolveDashboardAction()` 中新增 `retry` case：

```typescript
if (dashboardAction.type === "retry") {
  return {
    label: retryPending ? "重试中..." : dashboardAction.label,
    onClick: onRetry,
    variant: "default" as const,
    disabled: retryPending,
  };
}
```

### 3.3 客户端：resume_from_checkpoint 触发恢复

将 `resume_from_checkpoint` 的映射从 `onOpenTaskCenter` 改为触发 resume 重试：

```typescript
if (dashboardAction.type === "resume_from_checkpoint") {
  return {
    label: retryPending ? "恢复中..." : dashboardAction.label,
    onClick: onRetryWithResume,
    variant: "outline" as const,
    disabled: retryPending,
  };
}
```

### 3.4 props 传递

`NovelAutoDirectorProgressPanel` 需要新增 `onRetry` 和 `onRetryWithResume` props，从 `NovelEdit.tsx` 传入现有的 retry mutations。

## 4. 涉及文件

| 文件 | 变更 |
|------|------|
| `server/src/services/novel/director/projections/DirectorDashboardViewBuilder.ts` | failed 模式增加 retry action |
| `client/src/pages/novels/components/NovelAutoDirectorProgressPanel.tsx` | resolveDashboardAction 处理 retry + resume_from_checkpoint |
| `client/src/pages/novels/NovelEdit.tsx` | 传递 retry props 到 progress panel |
