# REQ-3017 创建页面路径选择卡片

## 1. 任务概述

### 1.1 背景

当前创建小说页面（NovelCreate）直接展示所有表单字段和多个辅助按钮（AI自动导演开书、素材导入、快速预览等），用户需要自己寻找和理解这些功能。这导致：

1. 新用户不知道从哪里开始
2. AI自动导演开书和手动创建被分开，用户体验不连贯
3. 快速预览功能的位置不明确

### 1.2 目标

在创建小说页面顶部新增路径选择卡片，统一AI自动导演开书、素材导入、手动填写三种路径，让用户在第一步就选择适合自己的创作方式。

### 1.3 价值

- **简化用户决策**：明确的三选一路径，降低选择困难
- **统一入口**：无论走哪条路径，最终都填充同一组字段（Novel表）
- **提升体验**：从"选择路径" → "填充字段" → "创建小说"的清晰流程

---

## 2. 功能需求

### 2.1 路径选择卡片

**位置**：创建小说页面顶部，在快速预览卡片下方

**内容**：
```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  选择你的创作方式：                                          │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ [A. 我有初步想法，AI自动导演开书]                       │  │
│  │    从想法出发，AI帮你生成完整设定                         │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ [B. 我有完善想法，从素材导入]                           │  │
│  │    导入万字长文，AI帮你解析为结构化字段                    │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ [C. 都不需要，我要手动填写]                             │  │
│  │    逐步填写各字段                                      │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**交互逻辑**：
- 路径A：点击后直接打开 NovelAutoDirectorDialog 对话框
- 路径B：点击后在下方显示素材导入框（MaterialParseDialog）
- 路径C：点击后在下方显示字段表单（NovelBasicInfoForm）
- 路径选择后，下方动态显示对应内容

### 2.2 动态内容区

**路径A - AI自动导演开书**：
- 打开 NovelAutoDirectorDialog 对话框
- 用户输入想法 → AI生成候选方案 → 用户选择 → 自动填充所有字段
- 对话框完成后，自动填入表单字段，用户可继续微调

**路径B - 素材导入**：
- 显示 MaterialParseDialog 导入区
- 用户上传/粘贴长文本 → AI解析 → 显示预览 → 用户确认 → 填充字段
- 导入完成后，用户可继续编辑字段

**路径C - 手动填写**：
- 提示文字："可以开始手动填写"
- 显示 NovelBasicInfoForm 表单
- 用户逐个填写字段

### 2.3 页面布局

```
页面布局从上到下：

┌─────────────────────────────────────────────────────┐
│  [快速预览卡片]                                       │ ← 独立功能，始终显示
│  一句话灵感 → 生成3个候选方案                           │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  [路径选择卡片]                                       │ ← A/B/C 三个选项
│  [A] [B] [C]                                         │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  [动态内容区]                                         │ ← 根据路径选择变化
│  路径A → 对话框（自动填充后显示表单）                   │
│  路径B → 素材导入框                                   │
│  路径C → 表单                                         │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  [创建小说] 按钮                                       │ ← 始终显示，title填写后可点击
└─────────────────────────────────────────────────────┘
```

## 3. 非功能需求

### 3.1 用户体验

- 路径选择卡片要突出显示，引导用户首先选择
- 选中路径后有明显视觉反馈（高亮/边框）
- 动态内容区切换要流畅，无卡顿

### 3.2 兼容性

- 保持现有功能不变（快速预览、AI自动导演、素材导入等）
- 路径选择是UI层面的改变，不影响底层数据流

### 3.3 可维护性

- 路径选择逻辑与现有组件解耦
- 便于后续新增其他路径

---

## 4. 数据流

### 4.1 统一目标

无论走哪条路径，最终都填充 Novel 表的这些字段：

```typescript
interface NovelBasicFormState {
  title: string;
  description: string;
  worldSetting: string;
  characters: string;
  outline: string;
  styleTone: string;
  readerChannelPreference: 'male' | 'female' | 'neutral';
  narrativePov: 'first' | 'third_limited' | 'third_omniscient';
  pacePreference: 'fast' | 'medium' | 'slow';
  estimatedChapterCount: number;
  // ...其他字段
}
```

### 4.2 路径数据流

**路径A**：
```
用户输入想法 → NovelAutoDirectorDialog
→ AI生成候选方案
→ 用户选择方案
→ 自动填充 basicForm 所有字段
→ 用户微调
→ 点击"创建小说"
```

**路径B**：
```
用户导入素材 → MaterialParseDialog
→ AI解析为结构化字段
→ 预览结果
→ 用户确认
→ 填充 basicForm 对应字段
→ 用户继续编辑
→ 点击"创建小说"
```

**路径C**：
```
直接显示表单
→ 用户逐个填写字段
→ 点击"创建小说"
```
