## D-002: 认证方案选择

**日期**: 2026-07-01

**决策**: 静态 API Token 认证

**备选方案**:
1. 静态 API Token — 本地单用户应用最简单方案
2. JWT 认证 — 适合未来多用户场景
3. localhost 白名单 + Token — 双重防护

**理由**: 本项目是本地单用户应用，静态 Token 方案最简单有效。首次启动生成 token 存 .env，前端请求拦截器自动附加。

## D-003: 循环引用解耦方案

**日期**: 2026-07-01
