### D005：持久化策略
- **时间**：2026-07-11 22:30
- **决策**：基础版本不实现设置持久化
- **依据**：
  - 优先实现核心功能
  - 持久化是可选增强
  - 避免增加复杂度
- **影响**：刷新页面后设置恢复默认值；后续可添加 localStorage 缓存

### D006：移动端适配
- **时间**：2026-07-11 22:30
- **决策**：本次不实现移动端适配
- **依据**：
  - 产品主要面向桌面端用户
  - 移动端适配需要额外的响应式设计
  - 留作后续优化
- **影响**：仅在桌面端（1440px+）测试布局

### D007：字体方案
- **时间**：2026-07-11 22:30
- **决策**：使用系统字体，不实现自定义字体上传
- **依据**：
  - 系统字体加载性能最佳
  - 中文字体文件体积大，上传会增加复杂度
  - 提供 6 种常见系统字体已足够
- **影响**：字体列表固定，但用户仍有选择空间

### D008：性能优化策略
- **时间**：2026-07-11 22:30
- **决策**：采用 React.memo + useMemo + useCallback 优化
- **依据**：
  - 控制面板频繁交互
  - 需要避免不必要的重渲染
  - 使用 React 标准性能优化方案
- **影响**：确保实时预览的流畅度
