## 1. 背景与问题

`characterTools.ts` 目前只暴露了 base character library 的两个工具：`list_base_characters` 和 `get_base_character_detail`。没有查询 novel 级角色弧光、时间线、动态、关系演化的工具。

数据库中已有丰富的角色数据：
- `Character.arcStart/Midpoint/Climax/End` — 弧光四阶段规划
- `CharacterTimeline` — 按章节有序的事件记录
- `CharacterRelationStage` — 版本化的关系演化（trust/intimacy/conflict/dependency 分数）
- `CharacterDynamicsService.getOverview()` — 全角色动态概览（含缺席风险）
- `CharacterState` — 按 snapshot 的情绪/目标状态

但这些数据对 agent（包括 narrative_advisor）完全不可见，需要新增 4 个 read-only 工具。

---

## 2. 目标与范围

### 2.1 目标

1. 新增 4 个 read-only agent tool，让 narrative_advisor 能查询角色弧光数据
2. 复用现有 `CharacterDynamicsService` 等服务层，不新增 service
3. 工具注册到 `toolRegistry.ts`，权限配置到 `approvalPolicy.ts`

### 2.2 In Scope

**后端**：
- `server/src/agents/tools/characterTools.ts`：新增 4 个工具定义
- `server/src/agents/toolRegistry.ts`：注册新工具
- `server/src/agents/approvalPolicy.ts`：Planner/Reviewer 角色权限

### 2.3 Out of Scope

- 不新增前端可视化（REQ-2032 承接）
- 不修改现有 `list_base_characters` / `get_base_character_detail`
- 不新增 Prisma 查询（复用现有 service 层）
- 不修改 narrative_advisor 的 workflow（后续迭代再集成）

## 3. 需求详情

### 3.1 get_character_arc

查询某角色的弧光规划与事件时间线。

**输入**：`{ novelId: string, characterId: string }`
**输出**：
```json
{
  "characterId": "...",
  "name": "林墨",
  "arc": {
    "arcStart": "普通学生，对修仙一无所知",
    "arcMidpoint": "获得传承，开始觉醒",
    "arcClimax": "与师门决裂，独自面对天劫",
    "arcEnd": "飞升成仙，但仍怀念凡间"
  },
  "currentState": "...",
  "timeline": [
    { "chapterOrder": 1, "title": "第一章", "event": "偶遇仙人" },
    { "chapterOrder": 5, "event": "获得传承" }
  ]
}
```

### 3.2 get_character_dynamics_overview

查看全角色动态概览（复用 `CharacterDynamicsService.getOverview()`）。

**输入**：`{ novelId: string, chapterOrder?: number }`
**输出**：`DynamicCharacterOverview`（含各角色缺席风险、阵营分布、关系摘要）

### 3.3 get_character_relation_evolution

查看某对角色的关系演化时间线。

**输入**：`{ novelId: string, characterIdA: string, characterIdB: string }`
**输出**：
```json
{
  "characterA": "林墨",
  "characterB": "苏瑶",
  "stages": [
    {
      "stageLabel": "初识",
      "stageSummary": "同门师兄妹，互有好感",
      "trustScore": 60,
      "intimacyScore": 30,
      "chapterOrder": 1
    },
    {
      "stageLabel": "信任危机",
      "trustScore": 20,
      "conflictScore": 80,
      "chapterOrder": 15
    }
  ]
}
```

### 3.4 get_character_states_by_chapter

查看某角色按章节的情绪/目标变化序列。

**输入**：`{ novelId: string, characterId: string }`
**输出**：
```json
{
  "characterId": "...",
  "name": "林墨",
  "states": [
    { "chapterOrder": 1, "emotion": "好奇", "stressLevel": 20, "currentGoal": "了解修仙" },
    { "chapterOrder": 10, "emotion": "焦虑", "stressLevel": 75, "currentGoal": "突破瓶颈" }
  ]
}
```

## 4. 验收标准

- [ ] 4 个工具均可通过 `getAgentToolDefinition()` 获取
- [ ] 工具的 `category` 为 `"read"` 或 `"inspect"`
- [ ] 工具的 `riskLevel` 为 `"low"`
- [ ] Planner 和 Reviewer 角色可调用
- [ ] 类型检查通过（`pnpm typecheck`）
- [ ] 相关测试通过（`pnpm --filter @ai-novel/server test:planner`）

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 某些数据可能为空（角色未设 arcStart 等） | 输出中允许 null 字段，工具描述中注明 |
| get_character_dynamics_overview 可能返回大量数据 | 限制返回角色数量上限（如 20），或只返回核心角色 |
