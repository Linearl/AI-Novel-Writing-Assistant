# REQ-7003 Server 超大文件拆分方案设计

> 状态：待补充（开发阶段产出）

## 1. 分批策略

按功能域分 5 批，每批独立可验证。批次间无强依赖，但建议按顺序执行以降低风险。

## 2. 通用拆分原则

### 2.1 导出保持不变

拆分后的原始文件通过 `export { ... } from './internal/...'` 重导出，保证所有现有 import 路径不受影响。

### 2.2 拆分粒度

- 按**职责**拆分（不按行数机械切割）
- 每个子模块有明确的单一职责
- 避免循环依赖（A → B 不能同时 B → A）

### 2.3 命名规范

- 子模块放在原文件同级的子目录中
- 原文件保留为 facade（重导出）
