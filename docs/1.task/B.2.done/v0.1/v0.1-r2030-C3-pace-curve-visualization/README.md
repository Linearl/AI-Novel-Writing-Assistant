# REQ-2030 节奏曲线可视化与调整 (Pace Curve Visualization)

> 创建日期：2026-06-30
> 目标版本：v0.1
> 状态：✅ 已完成

---

## 1. 任务概述

### 1.1 需求来源

REQ-2029 narrative_advisor 后续迭代。现有 `conflictLevel`（冲突强度 0-100）和 `revealLevel`（信息揭示 0-100）已存储在 `VolumeChapterPlan`（规划）和 `Chapter`（已写）上，但全无图表可视化，用户无法直观看到全书节奏曲线，也无法调整未撰写章节的节奏参数。

### 1.2 核心内容

1. 后端：新增节奏数据聚合 API，返回各章节的 conflictLevel / revealLevel / pacingScore
2. 前端：新增节奏曲线图表组件（折线图），横轴为章节序号，纵轴为分数，按卷分组着色
3. 前端：支持对未开始撰写的章节直接调整 conflictLevel / revealLevel（拖拽或输入）

### 1.3 前置条件

- 无外部依赖；现有 Prisma schema 字段已就绪，无需数据库迁移
