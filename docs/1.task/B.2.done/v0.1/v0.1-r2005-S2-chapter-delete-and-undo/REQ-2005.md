# REQ-2005 步骤6新建章节删除与误操作恢复

> 状态：⏳ 进行中（完成后改为 ✅ 完成）

| 字段 | 内容 |

## 1. 背景与问题

### 1.1 现状

步骤6（章节执行区）的"新建章节"按钮创建一个空壳章节（无 taskSheet、无 sceneCards、无卷规划映射）。创建后：

- 章节出现在左侧队列中
- 无法删除 — 没有删除按钮
- 无法撤销 — 没有 undo 机制

### 1.2 用户场景

1. 用户误点了"新建章节"，出现一个多余的空章节
2. 用户新建章节后发现应该回到步骤5通过卷规划创建
3. 用户新建了多个测试章节，想要清理

### 1.3 当前限制

- `Chapter` 表的删除只在 `VolumeChapterSyncService` 的 `applyDeletes: true` 路径中存在
- 该路径仅在步骤5同步时触发，且需要卷规划中有对应的删除标记
- 手动新建的章节因不在卷规划中，同步时会被标记为 `delete_candidate` 但默认不删除
- 前端没有章节删除 API 调用入口

---

## 2. 目标与范围

### 2.1 目标

1. 步骤6中选中章节后，提供"删除章节"按钮
2. 删除前需要用户确认（防止二次误操作）
3. 删除后提供 undo 恢复能力（软删除 + 撤销窗口）
4. 已有正文内容的章节删除时给出更强的警告

### 2.2 In Scope

**后端：**
- 软删除 API：`DELETE /novels/:novelId/chapters/:chapterId`（标记 `deletedAt`）
- 恢复 API：`POST /novels/:novelId/chapters/:chapterId/restore`
- 查询已删除章节：`GET /novels/:novelId/chapters?includeDeleted=true`

**前端：**
- 章节选中后显示"删除章节"按钮
- 删除确认对话框（区分空章节 vs 有内容章节）
- Undo toast 通知（删除后 10 秒内可撤销）
- 删除后自动选中相邻章节

### 2.3 Out of Scope

- 物理删除（本期只做软删除）
- 批量删除
- 删除后自动同步卷规划（删除的是步骤6的章节，不影响步骤5卷规划）
- 删除历史审计日志 UI

## 3. 需求详情

### 3.1 删除按钮

WHEN 用户在步骤6左侧队列中选中一个章节
THE SYSTEM SHALL 在右侧 AI 执行台或章节详情区域显示"删除章节"按钮。

按钮行为：
- 空章节（无 content、无 taskSheet）：点击后弹出简单确认对话框
- 有内容章节（有 content 或 taskSheet）：点击后弹出警告对话框，提示内容将丢失

### 3.2 删除确认对话框

WHEN 用户点击"删除章节"
THE SYSTEM SHALL 弹出确认对话框：

**空章节：**
```
标题：删除章节
内容：确定删除「{chapter.title}」？此章节为空章节。
按钮：[取消] [删除]
```

**有内容章节：**
```
标题：删除章节
内容：确定删除「{chapter.title}」？此章节包含 {wordCount} 字正文。
      删除后可在 10 秒内撤销。
按钮：[取消] [删除]
```

### 3.3 软删除

WHEN 用户确认删除
THE SYSTEM SHALL：
- 设置 `deletedAt` 为当前时间（不物理删除）
- 从左侧队列中移除该章节
- 自动选中相邻章节（优先下一个，无下一个则选上一个）
- 显示 undo toast

### 3.4 Undo 恢复

WHEN 章节被删除后
THE SYSTEM SHALL 显示 toast 通知：

```
已删除「{chapter.title}」    [撤销]
```

- toast 持续 10 秒
- 点击"撤销"调用恢复 API，章节重新出现在队列中
- 10 秒后 toast 自动消失，章节仍可通过 API 恢复（软删除保留）

### 3.5 恢复 API 行为

WHEN 调用 `POST /novels/:novelId/chapters/:chapterId/restore`
THE SYSTEM SHALL：
- 清除 `deletedAt` 字段
- 恢复章节到原位置（按 `order` 排序）
- 返回恢复后的章节数据

### 3.6 列表过滤

默认情况下，章节列表不显示已软删除的章节。查询参数 `includeDeleted=true` 可包含已删除章节（用于管理/恢复场景）。

---

## 4. 验收标准

- [ ] 步骤6选中章节后，AI 执行台区域显示"删除章节"按钮
- [ ] 空章节删除时显示简单确认对话框
- [ ] 有内容章节删除时显示带字数警告的确认对话框
- [ ] 确认删除后章节从队列中消失
- [ ] 删除后显示 undo toast，持续 10 秒
- [ ] 点击"撤销"后章节恢复到队列中
- [ ] 删除后自动选中相邻章节
- [ ] 已删除章节不出现在默认列表中
- [ ] 恢复 API 正确清除 deletedAt
- [ ] 软删除不影响卷规划数据

---

## 5. 风险与约束

| 风险 | 缓解 |
| ---- | ---- |
| 软删除章节可能干扰卷规划同步 | 同步逻辑过滤 deletedAt 非空的章节 |
| 用户删除后刷新页面，undo toast 消失 | 软删除数据保留，可通过 API 恢复 |
| 删除有自动导演任务关联的章节 | 删除前检查是否有活跃任务，有则提示先取消任务 |

---

## 6. 关联与边界

- 与 `VolumeChapterSyncService` 的关系：同步时应忽略 `deletedAt` 非空的章节
- 与自动导演的关系：删除章节前应检查是否有活跃的导演任务关联
- 与步骤5的关系：步骤6的删除不影响步骤5的卷规划条目
