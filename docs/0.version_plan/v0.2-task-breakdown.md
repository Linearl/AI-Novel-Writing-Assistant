## M1: 异常自动处理（5 天）

### REQ-7040: API 失败自动重试

**模块**：异常自动处理  
**优先级**：P1  
**预估工时**：1.5 天  
**依赖**：无

#### 任务目标

实现 API 调用失败的自动重试机制，使用指数退避算法，最多重试 3 次，用户无需手动干预。

#### 功能需求

1. **指数退避重试**
   - 第 1 次重试：等待 1 秒
   - 第 2 次重试：等待 2 秒
   - 第 3 次重试：等待 4 秒
   - 超过 3 次：标记为失败，触发备用模型切换

2. **失败类型识别**
   - 网络超时
   - API 限流（429 错误）
   - 服务不可用（500/502/503）
   - 认证失败（401/403）→ 不重试，直接通知用户

3. **重试日志**
   - 记录每次重试的时间、原因、结果
   - 便于排查问题

#### 技术实现要点

```typescript
// 伪代码示例
async function callWithRetry(apiCall: () => Promise<T>, maxRetries = 3): Promise<T> {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await apiCall();
    } catch (error) {
      if (isRetryableError(error) && i < maxRetries - 1) {
        const delay = Math.pow(2, i) * 1000; // 1s, 2s, 4s
        await sleep(delay);
        continue;
      }
      throw error;
    }
  }
}
```

#### 验收标准

- [ ] API 失败时自动重试，最多 3 次
- [ ] 使用指数退避算法（1s → 2s → 4s）
- [ ] 重试过程对用户透明（只看到最终结果）
- [ ] 重试日志记录完整
- [ ] 认证失败不重试，直接通知用户

#### 测试场景

1. **场景 1：网络超时**
   - 模拟 API 超时
   - 验证自动重试 3 次
   - 验证最终成功或失败

2. **场景 2：API 限流**
   - 模拟 429 错误
   - 验证等待后重试
   - 验证最终成功

3. **场景 3：认证失败**
   - 模拟 401 错误
   - 验证不重试，直接通知用户

### REQ-7041: 模型备用切换

**模块**：异常自动处理  
**优先级**：P1  
**预估工时**：1 天  
**依赖**：REQ-7040

#### 任务目标

实现主模型失败时自动切换到备用模型，切换过程不中断生成，用户无感知。

#### 功能需求

1. **模型配置**
   - 支持配置主模型和备用模型列表
   - 支持配置多个备用模型（按优先级排序）

2. **自动切换条件**
   - 主模型连续失败 3 次（REQ-7040 重试后仍失败）
   - 主模型响应质量差（质量分 < 阈值）
   - 主模型不可用（服务维护等）

3. **切换逻辑**
   - 按优先级尝试备用模型
   - 切换过程不中断当前生成
   - 记录切换原因和时间

4. **切换通知**
   - 页面内提示"已切换到备用模型"
   - 不阻塞用户操作

#### 技术实现要点

```typescript
// 伪代码示例
class ModelSwitcher {
  private primaryModel: string;
  private fallbackModels: string[];
  
  async callWithFallback(prompt: string): Promise<string> {
    // 尝试主模型
    try {
      return await this.callModel(this.primaryModel, prompt);
    } catch (error) {
      // 主模型失败，尝试备用模型
      for (const fallback of this.fallbackModels) {
        try {
          const result = await this.callModel(fallback, prompt);
          this.notifyUser(`已切换到备用模型: ${fallback}`);
          return result;
        } catch (e) {
          continue;
        }
      }
      throw new Error('所有模型都失败');
    }
  }
}
```

#### 验收标准

- [ ] 支持配置主模型和备用模型列表
- [ ] 主模型失败时自动切换备用模型
- [ ] 切换过程不中断生成
- [ ] 用户可配置备用模型优先级
- [ ] 切换过程有通知提示

#### 测试场景

1. **场景 1：主模型失败**
   - 模拟主模型连续失败 3 次
   - 验证自动切换到第一个备用模型
   - 验证生成继续

2. **场景 2：备用模型也失败**

### REQ-7042: 错误分类器

**模块**：异常自动处理  
**优先级**：P1  
**预估工时**：1.5 天  
**依赖**：REQ-7040

#### 任务目标

实现错误自动分类，根据错误类型采取不同的处理策略，提高异常处理的精准度。

#### 功能需求

1. **错误分类表**
   ```
   | 错误类别 | 示例 | 处理方式 |
   |----------|------|----------|
   | 可恢复 | API 限流、网络超时、服务暂时不可用 | 自动重试 |
   | 需配置 | API Key 错误、模型不存在、配置缺失 | 通知用户修复配置 |
   | 需人工 | 内容审核拒绝、安全问题、业务逻辑错误 | 通知用户处理 |
   | 系统错误 | 数据库错误、内存不足、磁盘空间不足 | 记录日志，通知开发者 |
   ```

2. **错误码映射**
   - HTTP 状态码映射（429 → 可恢复，401 → 需配置）
   - 业务错误码映射（自定义错误码）
   - 未知错误 → 默认处理

3. **自动处理逻辑**
   - 可恢复 → 触发 REQ-7040 的重试机制
   - 需配置 → 显示配置修复指引
   - 需人工 → 显示问题描述和建议操作
   - 系统错误 → 记录日志，显示友好错误消息

4. **用户友好的错误消息**
   - 不暴露技术细节
   - 提供可操作的建议
   - 示例："API Key 无效，请在设置中检查并更新 API Key"

#### 验收标准

- [ ] 错误自动分类准确率 >90%
- [ ] 可恢复错误自动触发重试
- [ ] 需配置错误显示清晰的修复指引
- [ ] 需人工错误显示问题描述和建议
- [ ] 错误消息用户友好，不暴露技术细节

   - 验证自动分类为"可恢复"

```typescript
// 数据库表设计
CREATE TABLE checkpoints (
  id UUID PRIMARY KEY,
  task_id UUID NOT NULL,
  chapter_number INT,
  state JSONB,  -- 完整状态快照
  created_at TIMESTAMP,
  FOREIGN KEY (task_id) REFERENCES tasks(id)
);

// 伪代码示例
class CheckpointService {
  async saveCheckpoint(taskId: string, chapterNumber: number, state: any) {
    await db.checkpoints.create({
      taskId,
      chapterNumber,
      state: compress(state),
      createdAt: new Date()
    });
    
    // 清理旧检查点
    await this.cleanupOldCheckpoints(taskId, 5);
  }
  
  async getLatestCheckpoint(taskId: string) {
    return await db.checkpoints.findFirst({
      where: { taskId },
      orderBy: { createdAt: 'desc' }
    });
  }
}
```

   - 验证人物状态一致性

#### 技术实现要点

```typescript
// 伪代码示例
class ResumeService {
  async checkAndResume() {
    // 检查是否有未完成的任务
    const pendingTask = await this.findPendingTask();
    
    if (pendingTask) {
      // 找到最近的检查点
      const checkpoint = await this.getLatestCheckpoint(pendingTask.id);
      
      if (checkpoint) {
        // 从断点恢复
        await this.resumeFromCheckpoint(pendingTask, checkpoint);
        this.notifyUser(`从第 ${checkpoint.chapterNumber + 1} 章继续`);
      }
    }
  }
  
  async resumeFromCheckpoint(task: Task, checkpoint: Checkpoint) {
    // 恢复状态
    const state = decompress(checkpoint.state);
    
    // 从下一章开始生成
    const startChapter = checkpoint.chapterNumber + 1;
    
    // 继续生成
    await this.generateChapters(task, startChapter, state);
  }
}
```

- [ ] 恢复过程对用户透明

### REQ-7046: 批量任务队列

**模块**：长篇生成优化  
**优先级**：P1  
**预估工时**：2 天  
**依赖**：REQ-7044, REQ-7045

#### 任务目标

实现批量任务队列，支持 300+ 章排队自动执行，任务间自动衔接。

#### 功能需求

1. **任务分解**
   - 自动将 300 章分解为 30 个批次（每批 10 章）
   - 每个批次作为一个子任务
   - 批次间有依赖关系（顺序执行）

2. **任务队列管理**
   - 任务队列可视化
   - 支持任务暂停、恢复、取消
   - 支持任务优先级调整
   - 支持任务重试

3. **批次调度**
   - 按顺序执行每个批次
   - 每个批次自动检查质量
   - 不合格自动重新生成（最多 3 次）
   - 每批次完成后保存检查点

4. **并发控制**
   - 控制同时运行的任务数（避免 API 限流）
   - 支持配置并发数（默认 1）
   - 任务间自动衔接

5. **进度追踪**
   - 显示整体进度（第 X 批次/共 Y 批次）
   - 显示每个批次的状态
   - 显示预计剩余时间

#### 技术实现要点

```typescript
// 数据库表设计
CREATE TABLE tasks (
  id UUID PRIMARY KEY,
  novel_id UUID NOT NULL,
  parent_task_id UUID,  -- 父任务（可选）
  status ENUM('pending', 'running', 'completed', 'failed', 'paused'),
  total_chapters INT,
  completed_chapters INT,
  current_chapter INT,
  batch_number INT,
  total_batches INT,
  quality_score DECIMAL,
  error_message TEXT,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

// 伪代码示例
class TaskQueueService {
  async createBatchTask(novelId: string, totalChapters: number) {
    // 分解为批次
    const batchSize = 10;
    const totalBatches = Math.ceil(totalChapters / batchSize);
    
    // 创建父任务
    const parentTask = await this.createTask(novelId, totalChapters, totalBatches);
    
    // 创建子任务
    for (let i = 0; i < totalBatches; i++) {
      await this.createTask(novelId, batchSize, i + 1, parentTask.id);
    }
    
    // 开始执行
    await this.executeNextBatch(parentTask.id);
  }
  
  async executeNextBatch(parentTaskId: string) {
    const nextBatch = await this.getNextBatch(parentTaskId);
    
    if (nextBatch) {
      await this.executeBatch(nextBatch);
      
      // 执行完成后，自动执行下一个批次
      await this.executeNextBatch(parentTaskId);
    } else {
      // 所有批次完成
      await this.completeTask(parentTaskId);
    }
  }
}
```
